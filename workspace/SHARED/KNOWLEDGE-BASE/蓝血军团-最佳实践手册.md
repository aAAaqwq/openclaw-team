## 第一部分：全网对标分析与架构论证

### 1.1 对标三个参考系

#### 参考系A：AGI Super Team（GitHub开源 — 成熟的多Agent实战体系）
- **角色定义**：`agents/<id>/` 下8文件（多了IDENTITY/WORKFLOW/TOOLS/USER）
- **Skill索引**：`skills/README.md` 一级目录索引（727个skill），Agent通过 `TOOLS.md` 引用
- **核心机制**：「Work-to-Skill」——每次完成任务后评估是否值得封装为skill
- **最佳实践**：`SKILL.md` 使用轻量frontmatter（name/description），不存执行代码，只在被调用时加载
- **限制**：文件偏多（8个 vs 官方标准6-7个），部分角色文件有冗余

#### 参考系B：OpenClaw 官方推荐（正统架构）
- **角色定义**：`workspace/` 根目录7文件（AGENTS/SOUL/TOOLS/IDENTITY/USER/HEARTBEAT/BOOTSTRAP）
- **Skill存储**：`~/.openclaw/skills/` 全局目录 + `workspace/skills/` 项目级目录
- **控制**：`agents.list[].skills` 配置每个agent可用的skill白名单
- **最佳实践**：Skill Creator 强调「Concise is Key」+ 轻量 frontmatter 元数据

#### 参考系C：ClawHub 社区（社区共享生态）
- **Skill规范**：每个skill独立目录，含SKILL.md（frontmatter+body）
- **发现机制**：ClawHub平台搜索+评分，Agent按需安装
- **社区skill数量**：数万个，按域名/标签分类

#### 2.3.2 角色内SKILL.md的轻量清单模式

每个角色的 `SKILL.md` 使用「轻量清单模式」：

```markdown
# 【角色名】SKILL.md — 技能清单 v1.0

## 模式说明
本文件是技能清单，不是技能内容。
- 清单 = 定义"什么场景用什么技能"（~50行）
- 内容 = 实际实现在 `workspace/skills/<技能名>/` 目录

工作流程：
```
需要技能 → 读本清单 → 找到对应skill → 按需加载 → 执行 → 释放
```

## 技能清单

### 🎨 核心技能（本角色独有）
| 技能名 | 存放位置 | 调用场景 |
|--------|---------|---------|
| xxx | workspace/skills/xxx/ | xxx |

| 技能名 | 存放位置 | 调用场景 |
|--------|---------|---------|
| yyy | workspace/skills/yyy/ | yyy |

## 技能分类索引（按优先级）
P0（启动即用）：技能A / 技能B
P1（高频调用）：技能C / 技能D
P2（按需调用）：技能E / 技能F
```

#### 2.3.3 SKILL.md的标准frontmatter

每个skill的SKILL.md必须包含标准frontmatter（参考ClawHub和AGI ST）：

```yaml
---
name: skill-name
description: 一句话描述（必须明确何时触发、解决什么问题）
domain: [分类标签1, 分类标签2]  # 用于skills-index.md自动归类
author: 创建者
version: 1.0.0
license: MIT
trigger: 触发条件/场景说明
---
```

#### 2.3.4 Skill复用协议

| 场景 | 操作 |
|------|------|
| 角色需要技能 | 读自己SKILL.md → 在`workspace/skills/`找到 → 按需加载 |

| 已有正确skill | 直接使用，不重复创建 |
| ClawHub有好skill | `openclaw skills install <name>` → 添加到索引 |
| 修正bug/优化 | 直接改对应skill目录的SKILL.md，更新version号 |
