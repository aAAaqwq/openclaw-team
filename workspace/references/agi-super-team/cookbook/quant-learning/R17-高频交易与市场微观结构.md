## 一、市场微观结构基础

### 1.1 订单簿结构

```
订单簿 (Order Book):

    卖单 (Asks)          买单 (Bids)
    ──────────────────── ────────────────────
    价格    数量    累计    价格    数量    累计
    60100   0.5BTC  2.5    59900   0.8BTC  3.2
    60050   0.3BTC  2.0    59850   1.0BTC  2.4
    60020   0.2BTC  1.7    59800   0.6BTC  1.4
    60010   0.5BTC  1.5    59750   0.5BTC  0.8
    60005   1.0BTC  1.0    59700   0.3BTC  0.3
    ──────────────────── ────────────────────
         Ask 方向 ↑           ↓ Bid 方向

    买卖价差 (Spread) = Best Ask - Best Bid
                        = 60005 - 59700 = 305 USDT
```

### 1.2 关键微观结构指标

| 指标 | 计算方式 | 说明 |
|------|----------|------|
| **买卖价差** | Best Ask - Best Bid | 流动性成本 |
| **深度** | Best ±N 个价位的总量 | 市场吸收能力 |
| **订单不平衡** | (Bid_Vol - Ask_Vol) / Total | 方向预测 |
| **成交率** | 成交量 / 下单量 | 执行质量 |
| **VWAP偏离** | 实际成交价 vs VWAP | 交易成本 |

## 二、订单簿分析

### 2.1 实时订单簿数据

```python
import ccxt
import numpy as np
import pandas as pd
import time

class OrderBookAnalyzer:
    """订单簿分析器"""

    def __init__(self, exchange, symbol='BTC/USDT'):
        self.exchange = exchange
        self.symbol = symbol

    def get_orderbook(self, limit=50):
        """获取订单簿"""
        ob = self.exchange.fetch_order_book(self.symbol, limit=limit)
        return ob

    def calc_spread(self, ob):
        """计算买卖价差"""
        best_bid = ob['bids'][0][0]
        best_ask = ob['asks'][0][0]
        spread = best_ask - best_bid
        spread_pct = spread / best_bid * 100
        mid_price = (best_bid + best_ask) / 2

        return {
            'best_bid': best_bid,
            'best_ask': best_ask,
            'spread': spread,
            'spread_pct': spread_pct,
            'mid_price': mid_price,
        }

    def calc_depth_imbalance(self, ob, levels=10):
        """
        计算订单簿不平衡度
        范围: -1 (全部卖单) 到 +1 (全部买单)
        """
        bid_vol = sum(b[1] for b in ob['bids'][:levels])
        ask_vol = sum(a[1] for a in ob['asks'][:levels])
        total = bid_vol + ask_vol

        imbalance = (bid_vol - ask_vol) / total
        return imbalance

    def calc_vwap_levels(self, ob, side='bids'):
        """计算各深度的VWAP"""
        levels = ob[side]
        cumulative_vol = 0
        cumulative_amount = 0
        vwap_levels = []

        for price, vol in levels:
            cumulative_vol += vol
            cumulative_amount += price * vol
            vwap = cumulative_amount / cumulative_vol
            vwap_levels.append({
                'price': price,
                'volume': vol,
                'cum_volume': cumulative_vol,
                'vwap': vwap,
            })

        return pd.DataFrame(vwap_levels)

    def estimate_market_impact(self, order_size, side='buy'):
        """
        估算市场冲击
        order_size: 订单数量(BTC)
        side: 'buy' 或 'sell'
        """
        ob = self.get_orderbook()
        levels = ob['asks'] if side == 'buy' else ob['bids']

        remaining = order_size
        total_cost = 0
        levels_used = 0

        for price, vol in levels:
            fill = min(remaining, vol)
            total_cost += fill * price
            remaining -= fill
            levels_used += 1

            if remaining <= 0:
                break

        if remaining > 0:
            print(f"⚠️ 流动性不足，剩余 {remaining:.6f} BTC 无法成交")

        avg_price = total_cost / order_size
        best_price = levels[0][0]
        impact_pct = (avg_price - best_price) / best_price * 100

        return {
            'avg_price': avg_price,
            'best_price': best_price,
            'impact_pct': impact_pct,
            'levels_used': levels_used,
        }
```

### 2.2 订单流不平衡策略

```python
def order_flow_imbalance_signal(ob_series, window=20):
    """
    订单流不平衡信号
    ob_series: 连续的订单簿不平衡度序列
    """
    # 不平衡度 > 0.2 → 买压 → 短期看涨
    # 不平衡度 < -0.2 → 卖压 → 短期看跌
    avg_imbalance = ob_series.rolling(window).mean()

    signal = pd.Series(0, index=ob_series.index)
    signal[avg_imbalance > 0.2] = 1
    signal[avg_imbalance < -0.2] = -1

    return signal
```

## 四、做市策略

### 4.1 Avellaneda-Stoikov 做市

```python
class AvellanedaStoikov:
    """
    Avellaneda-Stoikov 做市策略
    经典最优做市模型
    """

    def __init__(self, gamma=0.1, sigma=0.01, T=1.0, k=1.5):
        """
        gamma: 风险厌恶系数
        sigma: 波动率
        T: 时间范围
        k: 订单流强度参数
        """
        self.gamma = gamma
        self.sigma = sigma
        self.T = T
        self.k = k

    def optimal_quotes(self, S, q, t):
        """
        计算最优买卖报价
        S: 当前中间价
        q: 当前持仓
        t: 剩余时间 (0-1)
        """
        # 预留价格 (Reservation Price)
        reservation = S - q * self.gamma * self.sigma**2 * (self.T - t)

        # 最优买卖价差
        delta = self.gamma * self.sigma**2 * (self.T - t) + \
                2 / self.gamma * np.log(1 + self.gamma / self.k)

        spread = delta / 2

        # 根据持仓调整偏移
        bid = reservation - spread / 2 - q * self.gamma * self.sigma**2 * (self.T - t) / 2
        ask = reservation + spread / 2 - q * self.gamma * self.sigma**2 * (self.T - t) / 2

        # 简化版：
        bid = S - delta / 2
        ask = S + delta / 2

        # 持仓调整
        inventory_skew = q * self.gamma * self.sigma**2 * (self.T - t) / 2
        bid -= inventory_skew
        ask -= inventory_skew

        return {
            'bid': bid,
            'ask': ask,
            'spread': ask - bid,
            'mid': (bid + ask) / 2,
            'skew': inventory_skew,
        }
```

### 4.2 网格做市

```python
class GridMarketMaker:
    """简单网格做市"""

    def __init__(self, spread_bps=5, order_size=0.001, refresh_interval=5):
        """
        spread_bps: 价差（基点，1bp=0.01%）
        order_size: 每单大小
        refresh_interval: 刷新间隔（秒）
        """
        self.spread = spread_bps / 10000
        self.size = order_size
        self.interval = refresh_interval

    def calc_quotes(self, mid_price, inventory=0, max_inventory=0.1):
        """
        计算报价
        inventory: 当前持仓(BTC)
        max_inventory: 最大允许持仓
        """
        # 基础价差
        half_spread = mid_price * self.spread / 2

        # 持仓调整（偏移报价以减少库存）
        inventory_skew = (inventory / max_inventory) * half_spread

        bid = mid_price - half_spread - inventory_skew
        ask = mid_price + half_spread - inventory_skew

        # 持仓限制调整
        bid_size = self.size
        ask_size = self.size

        if inventory >= max_inventory:
            bid_size = 0  # 不再挂买单
        elif inventory <= -max_inventory:
            ask_size = 0  # 不再挂卖单

        return {
            'bid': bid, 'bid_size': bid_size,
            'ask': ask, 'ask_size': ask_size,
        }
```

## 五、A股高频限制

### 5.1 监管限制

| 限制 | 内容 |
|------|------|
| **报备要求** | 程序化交易需向券商报备 |
| **撤单率限制** | 撤单率不得过高 |
| **申报速率** | 每秒申报笔数有上限 |
| **异常交易** | 禁止幌骗(Spoofing)、分层(Layering) |

### 5.2 A股高频可行性

- A股传统高频（微秒级）几乎不可能（T+1 + 交易所限制）
- 更适合**中低频量化**（分钟级~日级）
- 日内策略主要依靠底仓T+0或ETF
