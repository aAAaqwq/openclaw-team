## 三、Phase 2: 回测（2-3个月）

### 3.1 策略开发清单

| 阶段 | A股策略 | BTC策略 |
|------|---------|---------|
| **入门** | 双均线交叉 | 动量策略 |
| **进阶** | 多因子选股 | 网格交易 |
| **高级** | 指数增强 | 期现套利 |

### 3.2 回测验证标准

| 指标 | 最低标准 | 理想标准 |
|------|----------|----------|
| 夏普比率 | > 1.0 | > 2.0 |
| 最大回撤 | < 20% | < 10% |
| Calmar比率 | > 0.5 | > 1.5 |
| 胜率 | > 45% | > 55% |
| 盈亏比 | > 1.5 | > 2.0 |
| 样本外衰减 | < 30% | < 15% |

### 3.3 回测检查清单

```
□ 是否使用了未来函数？
□ 是否包含退市股票（A股）？
□ 交易成本是否合理？
    □ A股: 佣金万2.5 + 印花税千1 + 滑点0.1%
    □ BTC: Maker 0.02% + Taker 0.05%
□ 是否做了Walk-Forward测试？
□ 是否有样本外验证？
□ 最大单笔成交量是否限制在当日成交量的5%以内？
□ 调仓频率是否考虑了T+1（A股）？
□ 因子信号是否滞后一天执行？
```

### 4.3 模拟盘评估

```
模拟盘运行至少30天，重点评估:
□ 策略收益 vs 回测收益差异（应 < 20%）
□ 最大回撤是否在可接受范围
□ 实际滑点 vs 回测假设
□ 系统稳定性（是否有技术故障）
□ 风控是否正常触发
```

## 五、Phase 4: 实盘（3-6个月）

### 5.1 实盘启动检查

| 检查项 | 要求 |
|--------|------|
| 策略回测通过 | 夏普>1, 最大回撤<20% |
| 模拟盘运行 | 30天+，无重大问题 |
| 风控系统 | 止损/止盈/仓位限制 |
| 监控告警 | 异常交易/亏损告警 |
| 日志系统 | 所有交易记录 |
| 应急方案 | 停电/断网/宕机处理 |

### 5.2 逐步加仓

```
阶段1 (月1-2): 投入总资金的 10-20%
    → 验证实盘表现
    → 对比回测和实盘差异

阶段2 (月3-4): 投入总资金的 30-50%
    → 确认策略稳定性
    → 微调参数

阶段3 (月5-6): 投入总资金的 60-80%
    → 全面运行
    → 持续优化

阶段4 (6月+): 投入总资金的 80-95%
    → 不建议满仓
    → 保留应急资金
```

## 七、避坑指南

### 7.1 新手常见错误

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| **跳过回测直接实盘** | 亏损 | 先回测3个月+ |
| **过度优化参数** | 过拟合 | Walk-Forward |
| **忽视交易成本** | 策略无效 | 加入实际成本 |
| **满仓操作** | 爆仓 | 分批建仓 |
| **不止损** | 大亏损 | 严格止损 |
| **频繁换策略** | 无所适从 | 坚持一个策略3个月 |
| **追求高收益** | 高风险 | 先追求稳定 |

### 7.2 实战心态

1. **年化20%就很优秀**：巴菲特年化约20%
2. **先活下来**：保本比赚钱重要
3. **小资金试错**：亏了不心疼的金额
4. **接受亏损**：亏损是交易的一部分
5. **持续学习**：市场在变，策略也要变
