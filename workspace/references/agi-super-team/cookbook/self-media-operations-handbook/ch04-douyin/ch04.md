# 第4章 · 抖音 & 短视频运营

**本章核心问题**

- 抖音算法到底在奖励什么行为？

- 如何用竞品分析工具批量采集抖音数据？
- 每日内容生产的标准 SOP 是什么？

## 4.1 抖音/TikTok 算法底层逻辑

### 原理说明

抖音和 TikTok 的算法逻辑完全不同于小红书的"收藏驱动"。抖音的核心指标是**完播率**——用户有没有看完你的视频。

算法按以下优先级排名：

| 指标 | 权重 | 含义 |
|------|------|------|
| 完播率 | 最高 | 用户是否看完整个视频 |
| 重播率 | 高 | 用户是否看了第二遍 |
| 分享率 | 高 | 用户是否发给朋友 |
| 评论率 | 中高 | 用户是否留言互动 |
| 关注转化 | 中 | 视频是否带来新粉丝 |

这个排名决定了所有创作决策：**你的每一个脚本、每一帧画面、每一句文案，都必须服务于提升完播率。**

完播率的两个关键时点：

- **3秒留存率**：71%的观众在前3秒决定是否继续看。这个数字决定你的视频能不能进入下一级流量池。
- **黄金完播线**：短视频的"黄金完播"是60秒内看到85%以上。低于50%完播率的视频很难获得推荐。

### 实操步骤

**第一步：确定视频长度策略**

| 长度 | 适用内容 | 完播率特征 |
|------|---------|-----------|
| 7-15秒 | 梗、一句话干货、视觉冲击 | 最高完播率 |
| 30-60秒 | 教程、故事、观点 | 完播率和信息量最佳平衡 |
| 1-3分钟 | 深度分析、完整教程 | 完播率低但粉丝质量高 |

初期建议以30-60秒为主——既能传递足够的信息量，又能保持高完播率。

**第二步：选择内容支柱**

围绕定位建立3-5个内容支柱。以"AI工具"为例：
- 支柱1：AI工具实测（"我用XX做了YY，结果ZZ"）
- 支柱2：AI提效技巧（"3步用AI搞定XX"）
- 支柱3：AI行业观点（"90%的AI Agent是噱头"）
- 支柱4：副业变现案例（"用AI月入X万的真实路径"）

**第三步：确定发布节奏**

| 模式 | 频率 | 适用阶段 |
|------|------|---------|
| 起步期 | 1条/天 | 前30天 |
| 增长期 | 1-2条/天 | 30-90天 |
| 稳定期 | 1条/天 + 定期系列 | 90天+ |

最佳发布时间：6-9点、12-14点、19-23点（目标用户时区）。测试后发现你的账号的高流量时段后，固定在那个时段发。

### 参数配置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 视频比例 | 9:16 竖版 | 抖音默认 |
| 视频时长 | 30-60秒（初期） | 完播率和信息量平衡 |
| 标题 | ≤30字 | 封面+标题组合传达核心信息 |
| 话题 | 3-5个 | 2个垂直+1个宽泛+1个热门 |
| 封面/缩略图 | 3:4 或 1:1 | 显示在视频封面位置 |
| AIGC标注 | 必须 | 片头或片尾标注"内容由AI辅助创作" |

### 常见问题

**Q：为什么有些10秒视频比60秒视频播放量高？**
A：完播率。10秒视频的完播率天然高，算法会优先推。但10秒视频很难传递深度信息，涨粉效果弱。30-60秒是涨粉效率最高的区间。

**Q：要不要买流量（DOU+）？**
A：初期不要。先用自然流量测试内容质量，找到2-3个高完播率的内容模型后，再用DOU+放大。没有数据支撑的买流量等于烧钱。

**Q：抖音和小红书的内容能复用吗？**
A：可以，但必须改格式。小红书图文→抖音需要变成视频脚本。小红书正文→抖音需要口语化改写。直接搬运会被限流。

### 避坑指南

**坑1：前3秒没有Hook**
"大家好，我是XX，今天给大家分享..."——这种开场在抖音上等于自杀。用户在前3秒就划走了。

**坑2：视频太长说太多**
60秒的视频塞了5个观点，每个都浅尝辄止。不如60秒只讲透1个观点，完播率和互动率都会更高。

**坑3：忽视AIGC标注**
抖音平台对AI生成内容有明确标注要求。不标注可能导致限流甚至下架。片头加一句"内容由AI辅助创作"就足够了。

## 4.2 短视频脚本创作

### 原理说明

短视频脚本不是"写一段话然后念出来"。短视频脚本是一个**时间预算系统**——每一秒都必须有明确的任务，要么在传递信息，要么在制造情绪，要么在推动节奏。

通用结构是三段式：**Hook（0-3秒）→ Body（3-50秒）→ CTA（最后3-5秒）**

Hook 的唯一任务是"停止滑动"。Body 的任务是"让用户看完"。CTA 的任务是"让用户做下一步动作"。

### 实操步骤

**第一步：写 Hook（最关键的一步）**

Hook 类型和使用场景：

| 类型 | 公式 | 示例 |
|------|------|------|
| 冲突/反常识 | "所有人都X，但他们错了" | "都说AI会抢饭碗，但我靠AI涨了3倍工资" |
| 数字+结果 | "X天/月做到Y" | "30天用AI从0做到月入2万" |
| POV/代入 | "你正在经历X的时候" | "半夜两点还在加班的你，看看这个" |
| 悬念 | "看完了你就知道为什么" | "这个AI工具用完之后我再也不想回去了" |
| 直接给结论 | "XX就是YY" | "最好的AI编程工具不是Cursor" |

Hook 要点：≤10个字，直接上结论或冲突，口语化，不要书面语。

**第二步：写 Body**

Body 的核心原则是**15秒一个小高潮**。每隔15秒，必须有一个让用户"哦！原来如此"的瞬间。

三种 Body 模板：

**模板A：教程型**
```
SETUP (3-8秒): "很多人不知道XX其实可以这样用..."
POINT 1 (8-20秒): "第一步，打开XX，选择YY..."
POINT 2 (20-35秒): "第二步，最关键的一步——ZZ..."
POINT 3 (35-45秒): "最后一步，看效果..."
```

**模板B：故事型**
```
CONTEXT (3-10秒): "3个月前，我还在XX..."
CONFLICT (10-30秒): "直到有一天，YY..."
TURN (30-40秒): "然后我发现了ZZ..."
LESSON (40-50秒): "我学到的最大教训是..."
```

**模板C：观点/测评型**
```
STATEMENT (3-15秒): "说说我对XX的真实看法："
EVIDENCE (15-40秒): "第一...第二...第三..."
CONCLUSION (40-50秒): "所以我的结论是..."
```

**第三步：写 CTA**

| CTA类型 | 示例 | 适用场景 |
|---------|------|---------|
| 关注引导 | "关注我，每天分享AI实战" | 通用 |
| 评论引导 | "你用过哪个？评论区告诉我" | 工具测评 |
| 收藏引导 | "码住，下次肯定用得上" | 教程干货 |
| 系列预告 | "明天更新第2集，关注不迷路" | 系列内容 |

**第四步：标注字幕设计**

脚本中标注哪些词需要在视频中用大字突出（关键数字、核心结论、品牌名）。字幕设计直接影响信息的视觉层次感。

### 参数配置

| 参数 | 标准 | 说明 |
|------|------|------|
| 脚本字数 | 220-260字/60秒 | 口语化节奏 |
| Hook字数 | ≤10字 | 必须能一眼看完 |
| 小高潮间隔 | 每15秒 | 保持用户注意力 |
| 口语化检查 | 念出来不卡壳 | 必须大声朗读一遍 |
| AI痕迹 | 0个 | 不用"值得注意的是/此外/与此同时" |

### 常见问题

**Q：脚本要不要逐字写？**
A：初期建议逐字写。熟练后可以只写要点，对着要点自由发挥。但不管写不写逐字稿，最终成品必须是口语化的——念出来自然、不卡壳。

**Q：口播好还是不出镜好？**
A：口播的信任感更强（用户看到真人），但制作成本更高。初期可以用"不出镜+AI配音+屏幕录制"的方式，降低制作门槛。等找到高互动的内容模型后，再升级到口播。

**Q：如何让脚本听起来不像念稿？**
A：三个方法：1) 加入口语词（"说实话"、"讲真"）；2) 长短句交替，不要每句都差不多长；3) 加入不确定性和个人态度（"我觉得"、"不一定对"）。

### 避坑指南

**坑1：Hook太长**
超过10个字的Hook在手机屏幕上显示不完，用户还没看完就划走了。Hook必须≤10字，一眼看完。

**坑2：Body平铺直叙**
"第一个...第二个...第三个..."这种平铺没有节奏感。每15秒要有一个小高潮——一个意外事实、一个具体数字、一个情绪转折。

**坑3：CTA太硬或太软**
太硬："关注我！点赞！收藏！"——三个指令同时下达，用户反而什么都不做。太软：完全没有CTA——用户看完就划走了。一个视频只给一个CTA指令。

### 示例

**主题：AI编程工具实测**

**Hook（3秒）**：
"我被AI取代了3个外包，但工资涨了"
（冲突+数字+结果，9个字）

**Body（47秒）**：
"3个月前我还在手写代码，一个API接口写两天。现在呢？（SETUP - 5秒）

第一步，用Cursor。把需求扔进去，它直接生成80%的代码，我改改就能用。（POINT 1 - 12秒）

第二步，用AI做code review。每次提交之前让Claude审一遍，bug率降了一半。（POINT 2 - 12秒）

第三步，最狠的一步——我用AI替代了3个初级外包。不是因为他们不好，是因为AI真的更快。（POINT 3 - 13秒）

说点真话：AI不会淘汰程序员，但会用AI的程序员会淘汰不会用的。"（TURN - 5秒）

**CTA（5秒）**：
"你用什么AI编程工具？评论区聊聊，我去试试"

## 4.3 竞品视频分析与数据采集

### 原理说明

抖音是纯 JS 渲染的 SPA（单页应用），不能用常规的 HTTP 请求抓取内容。必须使用浏览器工具渲染页面，才能获取视频列表和文案。

竞品分析的目的不是"抄"，而是回答：**这个赛道里什么内容能火？火的规律是什么？我能在哪里做出差异化？**

### 实操步骤

**第一步：用浏览器抓取视频列表**

```
# 打开竞品账号主页
browser(action="open", profile="openclaw", targetUrl="https://www.douyin.com/用户主页")

# 等页面渲染，确认视频列表出现
browser(action="screenshot", ...)

# 提取视频链接
browser(action="snapshot", ...)
# 从 aria tree 中提取 link[href=/video/XXXXXXXXXXXXXXXXX]
```

**第二步：批量提取视频文案**

通过 douyin-mcp 工具并发提取视频语音转文字（每条30-90秒），同时发起所有请求，不要串行等待：

```bash
# 阿里云百炼版本
DASHSCOPE_API_KEY="<key>" mcporter call douyin-mcp.extract_douyin_text \
  share_link="https://www.douyin.com/video/<video_id>"
```

**第三步：文案整理与分析**

对每条视频：
1. 去除口语化表达（语气词、重复、填充词）
2. 保留核心信息和论点
3. 书面化呈现，不做摘要压缩

最后跨视频归纳：主线逻辑、核心主题、一致的风格特征。

**第四步：提取爆款规律**

分析高互动视频的共性：
- Hook 类型分布（冲突型占比多少？数字型呢？）
- 视频长度分布（高互动视频集中在哪个时长？）
- 话题标签规律
- 发布时间规律

### 参数配置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 采集视频数 | 20-30条 | 保证统计意义 |
| 并发提取数 | 最多10条同时 | 避免触发限流 |
| 单条提取超时 | 90秒 | API 响应较慢 |
| 分析频率 | 每周1次 | 跟踪赛道变化 |

### 常见问题

**Q：抖音登录弹窗怎么办？**
A：直接读取 snapshot，视频列表通常仍可在 aria tree 中获取。第一条视频可能需要登录，从第二条起往往正常。

**Q：文案提取失败了怎么办？**
A：先用 `parse_douyin_video_info` 获取下载链接，再手动处理。或者换一个时间段重试，API 限流通常是临时的。

### 避坑指南

**坑1：串行提取文案**
逐条提取20个视频需要20×90秒=30分钟。并发提取只需90秒。并发是必须的。

**坑2：只分析头部大号**
头部大号的内容策略不一定适合你。同时分析中腰部账号（1-10万粉），他们的内容更接地气，参考价值更大。

## 4.4 每日内容生产 SOP

### 原理说明

与小红书类似，抖音每日内容生产也需要 SOP 化。但抖音多了"视频脚本"和"口语化改写"两个环节，整体流程比图文更复杂。

### 实操步骤

**Step 1：选题（15分钟）**

1. brave-search 搜索抖音 AI 热点
2. 7角度竞争分析法选3个选题
3. 5维评分法选标题（≤30字）

**Step 2：内容创作（每条30分钟）**

每条内容需要同时产出：
- 60秒视频脚本（220-260字）
- 图文版本描述（≤200字，作为备用发布格式）
- 3-5个话题标签

**Step 3：去AI痕迹（10分钟）**

用 humanizer 处理脚本。抖音比小红书更口语化：
- 加入口语词："说实话"、"讲真"、"我跟你讲"
- 长短句交替，不要每句差不多长
- 加入个人态度和不确定性
- 消除所有书面化表达

**Step 4：封面/缩略图生成（每条15分钟）**

提示词由视频主题驱动，严禁纯风格模板。

```bash
uv run ~/.openclaw/skills/relay-image-gen/scripts/relay_image_gen.py \
  -p "提示词" -f "输出路径" -a "3:4" -r "1k"
```

**Step 5：质量检查（每条10分钟）**

- [ ] 标题 ≤30字
- [ ] 脚本 220-260字
- [ ] 有Hook（≤10字，冲突/数字/悬念）
- [ ] Body每15秒一个小高潮
- [ ] 有CTA（一个指令）
- [ ] AI痕迹 <5%
- [ ] AIGC已标注
- [ ] 封面跟内容强关联
- [ ] 无违禁词

**Step 6：保存与发布**

保存到 `~/clawd/docs/daily-content/{YYYY-MM-DD}/douyin/`，然后发布：

```bash
python3 ~/clawd/skills/douyin-smart-publish/scripts/publish.py \
  --article ~/clawd/docs/daily-content/2026-04-15/douyin/article.md \
  --cover ~/clawd/docs/daily-content/2026-04-15/douyin/cover.jpg \
  --decision draft
```

### 参数配置

| 步骤 | 时间预算 | 关键工具 |
|------|---------|---------|
| 选题 | 15分钟 | brave-search + 竞争分析法 |
| 脚本创作 | 30分钟/条 | 三段式模板 + humanizer |
| 封面生成 | 15分钟/条 | relay-image-gen |
| 质量检查 | 10分钟/条 | 检查清单 |
| **总计** | **~3.5小时/3条** | |

### 常见问题

**Q：抖音和小红书的选题可以共用吗？**
A：选题可以共用，但内容形式必须完全重做。小红书是图文+300-800字正文，抖音是60秒视频脚本。同一个选题，两边的内容几乎需要重写。

**Q：视频脚本写完了怎么变成视频？**
A：有几种路径：1) 口播+手机拍摄（最简单）；2) AI数字人（即梦/HeyGen，参考第6章）；3) 屏幕录制+AI配音（工具教程类最适合）。选择哪种取决于你的内容类型和制作能力。

## 本章小结

**4.1节**的核心：抖音算法奖励完播率。71%的观众在前3秒决定是否继续看。视频长度以30-60秒为最佳区间，兼顾信息量和完播率。

**4.2节**的核心：短视频脚本 = Hook（≤10字，冲突/数字/悬念） + Body（每15秒一个小高潮） + CTA（一个指令）。三种模板：教程型、故事型、观点型。脚本必须口语化——念出来不卡壳。

**4.3节**的核心：竞品分析必须用浏览器工具（抖音是纯JS SPA），并发提取文案。分析收藏/点赞比找到高实用价值内容，从中提取爆款规律。

**4.4节**的核心：每日SOP = 选题15分钟 + 脚本30分钟/条 + 去AI痕10分钟 + 封面15分钟/条 + 检查10分钟。3条约3.5小时。

**短视频运营的最终标准：用户划到你的视频，前3秒停下来，看完不划走，看完还点了个关注。做到这三步，流量自然来。**
