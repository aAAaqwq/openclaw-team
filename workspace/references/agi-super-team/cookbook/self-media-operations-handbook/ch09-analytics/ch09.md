# 第9章：数据分析与竞品研究

**本章核心问题**

- 为什么感觉型运营会把账号越做越偏？
- 竞品分析到底在分析什么，不该分析什么？
- 电商竞品研究如何落到标题、视觉、评论、市场盲区？
- Google Analytics该看哪些核心指标，怎么读出问题？

## 9.1 竞品分析：不是抄别人，是看清战场

### 原理说明

做自媒体和做产品一样，最大的对手往往不是同行，而是**用户什么都不做**。

很多人把竞品分析理解成“抄爆款标题”“学别人封面排版”“模仿选题”。这叫低水平跟随，不叫分析。真正的竞品分析要回答四个问题：

1. 用户如果不看你，会看谁？
2. 用户如果不看任何账号，会用什么替代方案解决问题？
3. 你的优势到底是内容、分发、人格，还是商业模式？
4. 竞争对手最强的地方，恰好暴露了什么弱点？

按照 Competitive Analysis 的框架，竞品不只包括直接同行，还包括：

- **直接竞争者**：同赛道、同平台、同受众的账号
- **间接竞争者**：不同平台但争夺同一注意力的人
- **替代方案**：搜索引擎、微信群、朋友推荐、线下经验
- **状态 quo**：用户选择“不看内容、不做决定”

最常见的错误，是把竞品分析做成“功能/栏目对照表”。这没用。用户不因为你比别人多一个栏目就关注你，用户因为你**更快解决问题、更可信、更容易理解**才会停留。

### 实操步骤

**第一步：扩展竞品集合**

不要只列3个同行账号。至少按以下结构列出10个对象：

| 类型 | 数量 | 示例 |
|------|------|------|
| 直接竞品 | 3-5个 | 同领域同平台头部账号 |
| 间接竞品 | 2-3个 | 其他平台同主题创作者 |
| 替代方案 | 2-3个 | 搜索、问朋友、行业社群 |
| 不行动 | 1个 | 用户拖延/放弃解决问题 |

**第二步：建立五维比较表**

用统一维度比较，不凭感觉：

| 维度 | 说明 | 观察点 |
|------|------|--------|
| 定位 | 它服务谁、解决什么问题 | 简介、标题、内容结构 |
| 内容结构 | 它怎么讲 | 短平快/深度长文/故事化 |
| 分发能力 | 它怎么获客 | 搜索、推荐、社群、私域 |
| 信任建立 | 为什么用户信它 | 数据、案例、身份、表达 |
| 商业模式 | 它靠什么赚钱 | 广告、咨询、课程、带货 |

**第三步：找出非对称优势**

竞争中最有价值的不是“我比他多一点”，而是“他很难复制我”。

典型非对称优势包括：
- 你有真实一线案例，对方只有信息搬运
- 你有强分发渠道，对方只有内容能力
- 你有清晰的人格和表达风格，对方是标准化模板号
- 你有行业数据，对方只有观点

**第四步：设计回应策略，而不是盲目跟随**

竞品发了爆款，不代表你要跟着发。先判断：

- 这是用户真实需求，还是平台偶然奖励？
- 这是对方擅长领域，还是你也能做好的类型？
- 这是短期热点，还是长期资产？

只有当三者都成立，才值得跟进。

### 参数配置

| 参数项 | 推荐设置 | 说明 |
|--------|----------|------|
| 竞品样本数 | 8-10个 | 少于8个容易误判 |
| 分析周期 | 月度一次 | 热点赛道可双周一次 |
| 比较维度 | 固定5维 | 统一口径便于复盘 |
| 重点观察内容 | 近30天Top 10内容 | 看最新打法，不看过时内容 |
| 结论输出 | 3个机会 + 3个风险 | 避免只看机会不看风险 |

### 常见问题

**Q：竞品分析会不会导致自己越来越像别人？**
会，如果你只是复制表层。不会，如果你分析的是对方的“结构”和“优势来源”。学结构，不抄表达；学分发逻辑，不抄选题皮。

**Q：小账号有必要做竞品分析吗？**
更有必要。小账号最怕在错误方向上苦干三个月。竞品分析的意义不是追头部，而是少走弯路。

### 避坑指南

| 错误 | 问题 | 解决方案 |
|------|------|---------|
| 只看头部账号 | 得到的全是不可复制经验 | 加入腰部账号和新锐账号 |
| 只看爆款 | 忽略了账号整体稳定性 | 同时看最近30条的平均数据 |
| 功能式对比 | 看不见真正优势 | 增加分发、信任、商业模式维度 |
| 看到竞品动作就跟 | 容易被带节奏 | 先判断是否符合你的定位 |

### 示例

假设你做“AI办公效率”账号：

- 直接竞品A：天天发“10个AI工具合集”，流量高但评论浅
- 直接竞品B：专门做真实工作流拆解，更新少但收藏高
- 替代方案：用户直接搜“小红书 AI 办公工具推荐”
- 状态 quo：用户继续手工做，不主动学习新工具

分析结论：
1. 工具合集赛道拥挤，内容越来越同质化
2. 工作流实战内容少，但收藏率和转化率高
3. 你的机会不是再发一个“50个AI工具”，而是发“一个真实岗位如何用3个AI工具提升效率”

## 9.2 电商竞品研究：拆Listing，而不是看热闹

### 原理说明

电商竞品研究不能停留在“看价格、看销量”。真正有价值的是拆四层：

1. **文案逻辑**：标题、卖点、关键词怎么组织
2. **视觉逻辑**：主图、详情图、对比图怎么打动用户
3. **评论逻辑**：用户为什么买、为什么骂
4. **市场盲区**：还有什么需求没被满足

E-commerce Competitor Analyzer 给出了一个很实用的四维框架：

- **The Brain**：文案构建逻辑与词频
- **The Face**：视觉资产设计思路
- **The Voice**：评论定量与定性分析
- **The Pulse**：市场位置与盲区扫描

这套框架特别适合做内容选题，因为商品详情页本质上就是一页高转化内容。你拆得越细，越知道用户真正关心什么。

### 实操步骤

**第一步：收集竞品样本**

针对Amazon、Temu、Shopee等平台，至少选10个SKU，按价格带分层：

- 低价带 3个
- 主流价格带 4个
- 高价带 3个

这样可以避免只看头部产品，忽略不同价格段的策略差异。

**第二步：做基础信息表**

| 字段 | 说明 |
|------|------|
| SKU / ASIN | 产品唯一标识 |
| 标题 | 看关键词布局 |
| 价格 | 看定价区间 |
| 评分 | 看口碑天花板 |
| 评论数 | 看市场成熟度 |
| 卖点摘要 | 看核心主张 |

**第三步：按四维框架拆解**

1. **文案构建逻辑**
   - 高频词出现了哪些？
   - 标题是强调功能、结果还是场景？
   - Bullet Points里卖点排序是什么？

2. **视觉资产设计**
   - 主图是白底标准图还是场景图？
   - 详情页重点放参数、对比还是使用场景？
   - 是否大量使用“before/after”或图示化表达？

3. **评论分析**
   - 好评最常提什么？
   - 差评集中在哪些问题？
   - 评论里有没有出现用户自发创造的卖点？

4. **市场盲区扫描**
   - 哪些需求高频出现但卖家没强调？
   - 哪些抱怨在多个竞品上重复出现？
   - 是否存在“高评论需求、低供应表达”的空白点？

**第四步：转化为内容策略**

竞品研究最后不是为了做表，而是为了产出内容：

- 可做选题：用户最常问的问题
- 可做标题：用户最常说的抱怨/需求
- 可做封面：竞品详情页中最有效的视觉表达
- 可做脚本：基于评论区的真实用户语言

### 参数配置

| 参数项 | 推荐设置 | 说明 |
|--------|----------|------|
| 样本量 | 10-20个SKU | 太少不具代表性 |
| 评论抽样 | 每个SKU前100条 | 足够看出情绪模式 |
| 价格带覆盖 | 低/中/高三层 | 避免只看一个区间 |
| 输出维度 | 4维固定 | 保持长期可比较 |
| 复盘频率 | 每月一次 | 竞品策略变化快 |

### 常见问题

**Q：没有工具，能手工做竞品分析吗？**
可以，但效率低。手工分析5个SKU已经很累。建议先用表格抓基础信息，再重点拆2-3个核心竞品。

**Q：评论太多，怎么抓重点？**
不要全读。按“高赞评论、最新差评、带图评论、1星-3星评论”四类优先读，最容易看出真实问题。

### 避坑指南

| 错误 | 后果 | 解决方案 |
|------|------|---------|
| 只看价格和销量 | 得不到可执行结论 | 增加文案、视觉、评论、盲区四维分析 |
| 只看头部爆款 | 容易误判大众需求 | 同时覆盖中腰部产品 |
| 只读好评 | 看不到产品缺陷 | 差评往往更有价值 |
| 分析完不落地 | 报告变废纸 | 每次输出至少3个内容选题 |

### 示例

假设你在分析一款“颈椎按摩仪”：

- 标题高频词：portable、heat、deep tissue、pain relief
- 主图策略：1张产品白底 + 2张办公室/家中使用场景 + 1张对比图
- 差评集中：电池续航短、力度不稳定、材质廉价感
- 盲区：用户频繁提“久坐办公室肩颈酸”，但竞品详情页很少做“办公室真实使用场景”

可直接转化的内容选题：
1. 《我测了6款颈椎按摩仪，办公室党真正需要的不是“力度大”》
2. 《买按摩仪最容易踩的3个坑：续航、贴合度、噪音》
3. 《为什么大家都在讲“热敷”，却没人讲上班时到底好不好用》

## 9.3 Google Analytics：看见流量背后的行为真相

### 原理说明

GA的价值不是告诉你“今天有多少访问”，而是告诉你：**这些访问是从哪来的、在看什么、为什么离开、哪里发生了转化**。

Google Analytics最常见的误区，是只盯着Sessions和Users。访问量本身没有意义，关键是质量。1000个低质量访问，不如100个高转化访问。

分析GA，优先看四类指标：

| 指标类型 | 关键指标 | 作用 |
|---------|---------|------|
| 流量规模 | Sessions, Users, New Users | 看基本盘 |
| 互动质量 | Engagement Rate, Avg Session Duration, Bounce Rate | 看内容是否有吸引力 |
| 来源质量 | Source/Medium, Channel Grouping | 看流量从哪里来、值不值 |
| 转化结果 | Conversions, Event Count, CVR | 看业务是否真正增长 |

### 实操步骤

**第一步：先看来源结构，不要先看总量**

先拉出最近30天的 Source/Medium 报表。问自己三个问题：

1. 哪个渠道带来最多流量？
2. 哪个渠道转化率最高？
3. 哪个渠道跳出率最高？

经常出现的情况是：
- 社交流量很多，但停留时间短
- 搜索流量不算大，但转化率高
- 邮件/私域流量少，但质量最好

这意味着你的增长重点不该只放在“增流量”，而是放在“把高质量流量渠道做大”。

**第二步：看落地页与退出页**

Landing Page告诉你用户是从哪里进入的，Exit Page告诉你用户在哪走掉了。

重点观察：
- 哪些页面带来最多新用户？
- 哪些页面进入量大但转化差？
- 哪些页面退出率高得异常？

如果某篇文章流量高但转化低，大概率是：
- 标题吸引错了人
- 内容满足了阅读，但没有引导下一步
- 页面加载慢或CTA太弱

**第三步：看转化漏斗事件**

把关键行为定义为事件：

- page_view
- scroll_90
- click_cta
- lead_submit
- purchase

然后看漏斗：

```
1000 page_view
→ 600 scroll_90
→ 120 click_cta
→ 20 lead_submit
→ 5 purchase
```

如果scroll很多、click很少，说明内容还行但CTA弱；如果click多、submit少，说明表单或落地页有问题。

**第四步：做周期对比，而不是只看单点**

最少做两组对比：
- 本月 vs 上月
- 本月同期 vs 去年同期（如果有历史数据）

目的不是看绝对数，而是看趋势和异常。比如：
- Sessions涨了30%，但Conversions没涨 → 流量质量下降
- 搜索流量持平，Direct暴涨 → 可能是某次私域分发有效
- 平均停留时间下降 → 内容质量或页面体验出了问题

### 参数配置

| 参数项 | 推荐设置 | 说明 |
|--------|----------|------|
| 分析周期 | 最近30天 + 对比上30天 | 兼顾时效和趋势 |
| 核心指标数 | 8-12个 | 太多会淹没重点 |
| 渠道拆分粒度 | Source/Medium | 足够细又不太乱 |
| 关键事件数 | 4-6个 | 覆盖主要漏斗节点 |
| 复盘频率 | 每周简报 + 每月深度复盘 | 周看波动，月看结构 |

### 常见问题

**Q：Bounce Rate高一定是坏事吗？**
不一定。资讯类、工具类、单页回答型内容，本来就是“看完就走”。要结合页面目标判断。如果页面目标是让用户读完一篇文章，那么高跳出不一定坏；如果目标是引导注册/购买，那就要警惕。

**Q：流量上涨但转化没涨，先改哪？**
优先查流量来源和CTA。大概率不是“量不够”，而是“来的人不对”或“引导太弱”。

### 避坑指南

| 错误 | 问题 | 解决方案 |
|------|------|---------|
| 只看Sessions/Users | 只能看到表面热闹 | 同时看转化和互动质量 |
| 不设事件 | 无法分析漏斗 | 先定义4-6个关键事件 |
| 不做对比 | 看不出趋势 | 固定做周期对比 |
| 看报表不行动 | 数据没有业务意义 | 每次复盘输出3条动作建议 |

### 示例

最近30天GA数据显示：

| 渠道 | Sessions | Engagement Rate | CVR |
|------|----------|----------------|-----|
| Organic Search | 4,200 | 68% | 4.2% |
| Social | 7,800 | 34% | 0.9% |
| Direct | 1,600 | 72% | 6.1% |
| Email | 900 | 81% | 8.4% |

结论：
1. 社交流量最大，但质量最差，不能只追播放量
2. 搜索和邮件流量虽然小，但转化高，应继续加码SEO和私域
3. 如果预算有限，优先做“高意图内容 + 邮件/私域承接”

## 9.4 章节小结

本章的核心是：**别再靠感觉运营，开始用数据和竞品看清现实。**

**第一，竞品分析不是模仿，是理解竞争结构。** 你的对手不只是同行，还包括用户的惯性、替代方案和“不做决定”。

**第二，电商竞品研究要拆四层。** 文案、视觉、评论、市场盲区，缺一不可。只看价格和销量，只会得到一堆无用结论。

**第三，GA的重点不是流量多少，而是流量质量。** 哪个渠道带来高质量访问，哪个页面漏掉了用户，哪个环节阻断了转化——这些才是真问题。

**行动清单：**

1. 今天：列出你所在赛道的8-10个竞争对象，按5维框架比较
2. 本周：挑10个电商竞品SKU，至少拆出3个可做内容的盲点
3. 本周：在GA里拉出最近30天的渠道质量表，不再只看总流量
4. 本月：建立固定复盘机制，每次复盘输出3条可执行动作
