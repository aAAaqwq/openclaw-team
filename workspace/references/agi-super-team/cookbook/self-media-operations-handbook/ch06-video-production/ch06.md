# 第 6 章 · 视频制作全流程

> 本章涵盖：脚本创作 → AI视频生成 → 数字人制作 → 后期处理（剪辑/字幕/配音）→ 多模态合成
>
> 涉及工具：aigc-director、relay-video-gen、jimeng-storyboard、jimeng-digital-human、heygen-avatar-lite、video-merge-send、video-subtitles、video-caption-burner、ffmpeg-video-editor、minimax-tTS、multimodal-gen 等。

## 6.1 原理说明：视频制作的三层架构与工作流设计

自媒体视频制作本质上是**内容、形式、交付**三层解耦再重组的过程。每层都有独立的质量标准和优化空间，分层治理是保证视频质量的核心方法论。

**第一层：内容层——脚本与分镜**

内容层决定了视频是否有价值、能否留住观众。脚本烂，再好的画面也救不回来。

这一层的核心挑战是「信息密度」与「情感节奏」的平衡：太多干货会让观众疲惫，太少干货则浪费了观众的注意力。优秀的口播脚本遵循「钩子-问题-方案-案例-CTA」五段式结构，每个段落都有明确的时长预算和情绪目标。

分镜拆解是将宏观剧本落地为微观操作单元的过程。每个镜头的本质是一个「信息-情感」单元：数字人说出什么台词，同时做出什么动作/表情，镜头如何运动。分镜质量直接决定了数字人生成效率和最终视频的流畅度。

**第二层：生成层——AI视频生成**

生成层是AI介入最深的环节。2025年后，AI已能独立完成从脚本到分镜到视频生成的完整链路，你不再需要摄影棚和演员。但AI生成有其独特规律：

- **数字人视频**（即梦/HeyGen）：擅长口播类内容，核心是台词+动作+音色三位一体。优势是低成本、批量生产，劣势是长镜头时嘴型同步质量下降。
- **场景视频**（relay-video-gen）：擅长视觉冲击力强的内容，如产品展示、动画演示。核心是Prompt工程——用英文描述镜头语言能获得更好的生成效果。
- **复杂剧情**（aigc-director）：适合有叙事逻辑的长视频，6阶段工作流确保每个环节质量可控，但本地部署门槛较高。

AI生成的「不确定性」是最大的质量风险：同一Prompt可能生成风格完全不同的结果。因此，生成层的核心策略是：建立视觉规范、控制随机性、保留人工审核节点。

**第三层：交付层——剪辑合成与发布适配**

交付层是视频从「半成品」变为「可发布产品」的关键步骤。包括：多个片段合并、字幕烧录、横竖版转换、BGM混音、封面制作等。

这一层的核心挑战是「平台差异」：同一个视频在抖音、YouTube、B站的体验可能完全不同。竖屏视频在横屏场景下体验差，字幕字号在小屏幕手机上可能看不清，音频电平在嘈杂环境下可能听不清——所有这些问题都必须在交付层解决。

**三层之间的数据流转规范：**

```
分镜脚本（JSON/文本）
    ↓
视频片段（MP4，分辨率960×960/1080×1920/1920×1080）
    ↓
合并视频（MP4，统一分辨率1920×1080）
    ↓
字幕烧录（MP4，内嵌字幕轨道）
    ↓
平台适配（MP4，按平台规格输出多个版本）
```

每层之间的产物必须遵循固定格式规范，否则会引发后续环节的兼容性问题。比如即梦数字人输出的分辨率为960×960，但如果直接和1920×1080的片段合并，画面会变形。必须在合并前统一分辨率。

## 6.2 实操步骤：五阶段工作流

### 阶段一：脚本创作与分镜拆解

**Step 1：生成口播剧本**

根据主题和目标时长生成完整剧本。剧本结构遵循自媒体黄金模板：

```
钩子（0-5秒）→ 问题/痛点（5-20秒）→ 解决方案（20-60秒）→ 案例/数据（60-120秒）→ CTA（最后10秒）
```

参考指令（发给AI助手）：
> 生成一个约90秒的口播视频剧本，主题：「中小企业为什么要做私域流量」，目标用户：传统制造业老板。风格：实战派，有具体数据，不说空话。

**Step 2：用 jimeng-storyboard 拆解分镜脚本**

将完整剧本按语义段落拆分为若干镜头，每个镜头对应即梦数字人界面的一个生成单元。

关键参数：
- 每镜头台词：**15-36字**（中文口播约3字/秒，对应5-12秒时长）
- 硬限制：单镜头超过36字/12秒必须拆分
- Hook镜头台词≤20字，CTA镜头≤25字
- 动作描述格式：`[镜头类型]，[表情/动作]，[镜头语言]`

分镜输出模板：
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎬 分镜脚本 — [主题名称]
总时长：约90秒 | 共8个镜头
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【镜头1】⏱️ 约10s
📢 角色说：老板，你还在靠平台流量做生意吗？
🎭 动作描述：正面中景，数字人严肃地看向镜头

【镜头2】⏱️ 约12s
📢 角色说：平台流量越来越贵，获客成本三年翻了三倍。
🎭 动作描述：镜头缓慢推近，数字人竖起食指强调
...
```

**Step 3：确认分镜后，按序号命名视频文件**
命名规范：`01_hook.mp4`、`02_problem.mp4`、`03_solution.mp4`、`04_cta.mp4`

### 阶段二：AI视频生成

根据视频类型选择对应工具。

#### 2A. 数字人口播视频（即梦平台）

**前置条件：** 浏览器已启动（`browser start profile=openclaw`），即梦账号已登录。

**Step 1：导航到数字人页面**
```
browser navigate url=https://jimeng.jianying.com/ai-tool/home?type=digitalHuman&workspace=0
```
等待2秒，截图确认左下角显示用户头像（已登录）。

**Step 2：选择或上传角色**
- 有默认角色：直接使用已保存的角色图片
- 无默认角色：点击「角色」→「上传人物图片」（支持真人或动漫风格）→ 等待3秒确认显示

上传操作：
```bash
mkdir -p /tmp/openclaw/uploads
cp <图片路径> /tmp/openclaw/uploads/avatar.png
browser upload selector=".lv-popover-open input.file-input-OfqonL" paths=["/tmp/openclaw/uploads/avatar.png"]
```

**Step 3：选择音色**
点击角色名标签 → 在音色面板中选择目标音色。即梦音色按「全部音色/我的音色/收藏」分类，支持性别、年龄、语言、声音特点筛选。

**Step 4：填入台词和动作描述**
点击编辑区域 → 输入「角色说」内容 → 换行后输入「动作描述」
> 注意：即梦编辑区是 contenteditable div，需用 `browser act kind=type` 输入，而非标准 input

**Step 5：生成视频**
确认底部橙色生成按钮已激活（非disabled）→ 点击生成 → 页面跳转至生成队列 → 等待1-3分钟

**Step 6：下载视频（关键闭环）**
1. 注入下载拦截器：
```javascript
() => {
  window.__downloadUrls = [];
  const origFetch = window.fetch;
  window.fetch = async function(...args) {
    const url = typeof args[0] === 'string' ? args[0] : args[0]?.url;
    if (url && (url.includes('.mp4') || url.includes('video') || url.includes('dreamnia'))) {
      window.__downloadUrls.push(url);
    }
    return origFetch.apply(this, args);
  };
  return 'interceptor ready';
}
```
2. 点击视频条目旁的展开按钮 → 打开详情弹窗
3. 点击「下载」按钮
4. 读取拦截器获取CDN直链（`https://v3-dreamnia.jimeng.com/...` 格式）
5. curl下载：
```bash
curl -L -o "/tmp/openclaw/jimeng-videos/01_hook.mp4" "<CDN直链URL>"
```

**Step 7：批量多镜头生成**
对每个镜头重复 Step 4→5→6，每个镜头前清空编辑区：
```
browser act kind=click ref=<textbox>
browser act kind=press key=Meta+a
browser act kind=press key=Backspace
```

> **HeyGen替代方案**：HeyGen支持API调用（需Creator计划，$29/月），适合需要更高质量数字人且用量较大的场景。API调用流程：列出可用角色和音色 → 提交生成任务 → 轮询任务状态 → 下载视频。HeyGen的优势在于音色自然度和角色丰富度，劣势是需要付费且不支持中文界面操作。

#### 2B. 场景视频生成（Relay模式）

**使用 relay-video-gen**进行多模型自动回退的视频生成：

```bash
uv run ~/.openclaw/skills/relay-video-gen/scripts/relay_video_gen.py \
  -p "A rotating blue globe with stars in background, cinematic lighting" \
  -f globe.mp4 -d 5 -a 16:9
```

模型回退顺序：Gemini Veo 3.1 → 可灵 Kling → minimax/video-01 → grok-video-3。任意模型失败自动切换下一模型，无需人工干预。

#### 2C. 本地AIGC导演工作流（aigc-director）

适合复杂剧情视频的本地部署方案，包含六个完整阶段：

```
剧本生成 → 角色/场景设计 → 分镜设计 → 参考图生成 → 视频生成 → 后期剪辑
```

每个阶段有强制停点（stage_completed/waiting_in_stage），必须展示产物给用户确认后才能继续，确保质量可控。详见 aigc-director skill 文档。

### 阶段三：后期处理

#### 3.1 视频合并

使用 video-merge-send skill 合并多个片段：

```bash
python3 skills/video-merge-send/scripts/merge_videos.py \
  -d /tmp/openclaw/jimeng-videos/ \
  -o /tmp/openclaw/output/final_merged.mp4 \
  --transition fade \
  --transition-duration 0.5
```

或指定文件列表：
```bash
python3 skills/video-merge-send/scripts/merge_videos.py \
  -i clip1.mp4 clip2.mp4 clip3.mp4 \
  -o output.mp4 \
  --transition fade
```

> 视频片段分辨率不同时，脚本自动统一为第一个视频的分辨率。命名带序号确保合并顺序正确。

#### 3.2 字幕生成与烧录

**方式A：自动转写生成字幕**（video-subtitles skill）

```bash
# 自动检测语言并转写
./scripts/generate_srt.py video.mp4 --srt

# 指定语言（中文）
./scripts/generate_srt.py video.mp4 --srt --lang zh

# 直接烧录进视频
./scripts/generate_srt.py video.mp4 --srt --burn

# 翻译+烧录
./scripts/generate_srt.py video.mp4 --srt --burn --translate en
```

字幕风格：42字/行上限，2行最大，自然断句位置（标点+停顿），白色字体带黑色描边，底部居中。

**方式B：直接烧录已有字幕**（video-caption-burner skill）

当字幕文件已存在（如SRT格式），直接烧录：
```bash
ffmpeg -y -hide_banner -i "video.mp4" -vf "subtitles='subtitles.srt'" "video_subtitled.mp4"
```

#### 3.3 FFmpeg专业剪辑

**裁剪片段：**
```bash
ffmpeg -y -hide_banner -i "video.mp4" -ss 00:01:21 -to 00:01:35 -c copy "clip.mp4"
```

**改变分辨率：**
```bash
# 720p
ffmpeg -y -hide_banner -i "video.mp4" -vf "scale=1280:720" -c:a copy "video_720p.mp4"

# 4K
ffmpeg -y -hide_banner -i "video.mp4" -vf "scale=3840:2160" -c:a copy "video_4k.mp4"
```

**调整比例（加黑边）：**
```bash
# TikTok竖屏 9:16
ffmpeg -y -hide_banner -i "video.mp4" -vf "scale=1080:1920:force_original_aspect_ratio=decrease,pad=1080:1920:(ow-iw)/2:(oh-ih)/2:black" -c:a copy "video_tiktok.mp4"

# Instagram方形 1:1
ffmpeg -y -hide_banner -i "video.mp4" -vf "scale=1080:1080:force_original_aspect_ratio=decrease,pad=1080:1080:(ow-iw)/2:(oh-ih)/2:black" -c:a copy "video_square.mp4"
```

**压缩文件：**
```bash
# 平衡模式（推荐）
ffmpeg -y -hide_banner -i "video.mp4" -c:v libx264 -crf 23 -preset medium -c:a aac -b:a 128k "video_compressed.mp4"

# 高压缩（小文件）
ffmpeg -y -hide_banner -i "video.mp4" -c:v libx264 -crf 28 -preset fast -c:a aac -b:a 96k "video_small.mp4"
```

**加水印：**
```bash
# 右下角
ffmpeg -y -hide_banner -i "video.mp4" -i "logo.png" -filter_complex "overlay=W-w-10:10" "video_watermarked.mp4"
```

**提取音频：**
```bash
ffmpeg -y -hide_banner -i "video.mp4" -vn -acodec libmp3lame "video.mp3"
```

**配音混音**（MiniMax TTS → 混音）：
```python
# 1. 生成配音
await synthesize(model="speech-2.8-hd", voice_id="female-yujie",
                 text="你好世界", api_key="...", output_path="tts.mp3")

# 2. 混音（配音叠加原音，背景音乐降低到15%）
ffmpeg -y -hide_banner \
  -i original_video.mp4 \
  -i tts.mp3 \
  -i background_music.mp3 \
  -filter_complex "[1:a]volume=1.5[v1];[2:a]volume=0.15[v2];[v1][v2]amix=inputs=2:duration=longest[aout]" \
  -map 0:v -map "[aout]" \
  output_with_voiceover.mp4
```

## 6.4 常见问题（Q&A）

**Q1：即梦数字人视频有水印怎么办？**
即梦非会员下载的视频带平台水印。解决方案：①开通即梦会员去除水印；②使用FFmpeg裁剪去掉水印区域（通常在右下角）；③升级到HeyGen API方案（质量更高且无水印）。

**Q2：多片段视频合并后音画不同步怎么办？**
大多数情况是因为片段分辨率或帧率不一致。先用FFmpeg统一所有片段的分辨率和帧率（推荐统一为1920×1080，30fps），再合并。如果仍然不同步，用`-async 1`参数强制对齐音频。

**Q3：字幕生成后时间轴不准怎么处理？**
Whisper转写的字幕时间轴在中文视频上可能出现0.5-2秒偏差。可用SRT编辑器手动微调，或使用LLM分析语音内容后重新对齐时间轴。专业场景建议人工校对关键信息。

**Q4：AI生成的视频片段风格不统一怎么办？**
建立角色和场景的视觉规范文档：包括人物外观描述、配色方案、镜头运动风格等，在每次生成时附加强制一致的提示词前缀。即梦数字人的角色图片固定后，同一角色生成的所有片段风格天然一致。

**Q5：即梦数字人生成的视频人物嘴型对不上台词怎么办？**
这是即梦平台的已知局限，尤其在长台词和快速语速时更明显。解决方案：①将长台词拆成多个短镜头（每个≤12秒）；②在动作描述中加入「正面中景」等限制条件；③切换到HeyGen（嘴型同步质量更好）。

**Q6：视频在某些平台上传后画面被裁切怎么办？**
上传前必须了解各平台的裁切规则。抖音/快手通常裁切上下；YouTube缩放适配。最可靠的做法：提前用FFmpeg输出多个比例版本（16:9、9:16、1:1），按平台分发对应版本。核心信息不要放在可能被裁切的画面边缘区域。

**Q8：即梦生成的数字人视频人物形象每次都有细微差异怎么办？**
同一角色图片在不同次生成时可能出现肤色微调、面部比例略有变化等问题。根本原因是AI生成的随机性。解决方案：①固定使用同一张高质量角色图片（分辨率≥1024×1024，光线均匀，正面照）；②在动作描述中加入明确的形象约束词（如「同一个人物，相同的肤色和发型」）；③生成后快速浏览，差异明显的片段重新生成。

**Q9：视频声音和字幕不同步怎么办？**
常见原因：Whisper转写时间轴本身有偏差，或者即梦的嘴型生成有延迟。解决方案：①用FFmpeg检查音视频轨道时间戳是否一致：`ffprobe -show_streams video.mp4 | grep start_time`；②如不一致，用FFmpeg重新混流：`ffmpeg -y -i video.mp4 -c copy -video_track_timescale 15360 fixed.mp4`；③字幕不同步时，用SRT编辑器按0.5秒步进微调。

**Q10：MiniMax TTS生成的配音听起来「太机器」怎么办？**
①选择更高级的模型（`speech-2.8-hd` > `speech-2.6-hd` > `speech-02-hd`），HD版本音质更好；②在文本中加入情感标记（部分模型支持停顿、语调变化符号）；③将配音与原视频中的环境音混合，降低「纯录音棚」的冷感；④考虑用真人录音替代关键视频的配音。

**Q11：视频发布后画质下降严重怎么办？**
平台二次压缩是画质下降的主要原因。解决方案：①上传前确保视频码率足够高（H.264编码，CRF≤23，码率≥4Mbps）；②避免多次转码（不要上传已经压缩过的视频再导出）；③YouTube建议上传原始质量文件（4K/1080p），平台会自动适配；④抖音建议上传720p或1080p，文件大小控制在200MB以内效果最佳。

**Q12：建筑视频、产品展示等特殊内容用什么工具最合适？**
建筑漫游视频、产品360度展示等属于专业视觉类内容，可使用aigc-director进行本地工作流生成（支持角色+场景+分镜的完整控制），或使用relay-video-gen生成场景片段后用FFmpeg拼接加Ken Burns效果（缓慢缩放+平移）。这类内容对画面稳定性和光线一致性要求高，不建议使用数字人工具。

**Q13：如何判断一条视频是否需要重新制作？**
触发返工的条件：①字幕错别字超过3个；②音画不同步超过1秒；③关键信息被裁切掉；④分辨率低于720p；⑤内容审核被拒绝（需修改违规内容后重新烧录）。返工成本通常为首次制作的20-40%，所以前期的分镜和脚本审核非常重要。

**Q14：横屏视频要转成竖屏发布，有什么要注意的？**
竖屏转横屏会导致大量画面内容被裁切，这是最常见的发布事故。正确做法：①制作时就确定目标平台，不确定时就做竖屏（竖屏内容可以加黑边变成横屏，横屏内容竖屏时几乎无法看）；②必须转换时，用FFmpeg的`crop`参数指定裁切区域，确保主体在画面中心：`ffmpeg -y -i video.mp4 -vf "crop=ih*9/16:ih:iw/2-ih*9/32:0" vertical.mp4`；③不要相信平台的「自动裁切」功能，必ず手动确认。

**Q15：视频生成需要等待很长时间，有没有办法加速？**
①即梦数字人通常1-3分钟，比HeyGen（5-30分钟）快，适合快速迭代；②场景视频用relay-video-gen的快速模式（`-m veo3.1-fast`），牺牲少量质量换取速度；③批量生成时准备多个平台的账号，错峰使用API；④建筑视频等长片段考虑分段生成后合并，而非一次生成整段。

## 6.5 避坑指南

### 坑1：分镜脚本不标准化导致合并混乱

**错误做法**：用自然语言描述镜头，让AI自由发挥，结果台词长短不一、动作描述模糊。
```
❌ "第一个镜头随便说点什么开场，第二个镜头展开讲讲"
```

**正确做法**：严格执行分镜规范，台词字数+时长双约束。
```
✅ 【镜头1】⏱️ 约10s
   📢 角色说：老板，你还在靠平台流量做生意吗？（18字）
   🎭 动作描述：正面中景，数字人严肃地看向镜头
```

### 坑2：数字人角色和音色每次重新选择

**错误做法**：每个镜头都重新上传角色、选择音色，浪费时间且可能产生细微差异。
**正确做法**：第一次选择后保存为默认配置，后续直接使用。即梦在单次会话中角色和音色会保持选中状态。

### 坑3：直接合并不同规格的视频片段

**错误做法**：直接合并1080p片段和720p片段，导致画面放大缩小、文件体积异常。
**正确做法**：合并前先统一规格，或在合并脚本中依赖第一个片段的规格（脚本自动统一）。

### 坑4：字幕遮挡关键画面

**错误做法**：将字幕固定烧录在画面底部中央，覆盖了人物脸部或产品。
**正确做法**：烧录前检查关键帧，确认字幕区域无重要信息。可用FFmpeg的`margin_v`参数调整字幕垂直位置。

### 坑5：视频比例不匹配平台要求

**错误做法**：上传竖屏视频到YouTube，或横屏视频到抖音，平台自动裁切导致内容丢失。
**正确做法**：制作时即规划多平台分发，一次制作输出多个比例版本。

### 坑6：CDN直链过期导致无法下载

**错误做法**：生成视频后没有立即下载，CDN URL过期（通常几小时）后无法获取。
**正确做法**：视频生成完成后立即下载，不要拖延。即梦会员可获得更长的URL有效期。

### 坑7：API Key未配置导致工作流中断

**错误做法**：开始生成后才发现API Key缺失，中断整个流程。
**正确做法**：工作流启动前检查所有必需API Key：
```bash
cat ~/.env | grep -E "DASHSCOPE_API_KEY|ARK_API_KEY|KLING_ACCESS_KEY|MINIMAX_API_KEY"
```

yt-dlp --write-comments --skip-download "视频URL"

```python
# quality_check.py
import subprocess
import json

def check_video_quality(video_path):
    """检查视频质量指标"""
    # 1. 检查分辨率
    cmd = ['ffprobe', '-v', 'error', '-select_streams', 'v:0',
           '-show_entries', 'stream=width,height', '-of', 'json', video_path]
    result = subprocess.run(cmd, capture_output=True, text=True)
    info = json.loads(result.stdout)
    width = info['streams'][0]['width']
    height = info['streams'][0]['height']
    
    # 2. 检查时长
    cmd = ['ffprobe', '-v', 'error', '-show_entries', 'format=duration',
           '-of', 'default=noprint_wrappers=1:nokey=1', video_path]
    result = subprocess.run(cmd, capture_output=True, text=True)
    duration = float(result.stdout.strip())
    
    # 3. 检查文件大小
    import os
    size_mb = os.path.getsize(video_path) / (1024 * 1024)
    
    # 质量判定
    issues = []
    if width < 1080 or height < 1080:
        issues.append(f"分辨率过低: {width}x{height}")
    if duration < 15:
        issues.append(f"时长过短: {duration:.1f}秒")
    if size_mb > 500:
        issues.append(f"文件过大: {size_mb:.1f}MB")
    
    return {
        'path': video_path,
        'resolution': f"{width}x{height}",
        'duration': f"{duration:.1f}s",
        'size': f"{size_mb:.1f}MB",
        'issues': issues,
        'pass': len(issues) == 0
    }

# 批量检查
import glob
for video in glob.glob('/tmp/openclaw/batch_videos/release/*.mp4'):
    result = check_video_quality(video)
    status = '✅' if result['pass'] else '❌'
    print(f"{status} {result['path']}")
    if result['issues']:
        for issue in result['issues']:
            print(f"  ⚠️  {issue}")
```

## 6.9 章节小结

本章建立了自媒体视频制作的完整五阶段工作流：

1. **脚本创作与分镜拆解**：剧本结构标准化，分镜脚本格式化（台词字数+时长双约束），命名规范化
2. **AI视频生成**：口播视频用即梦/HeyGen数字人，场景视频用relay-video-gen，复杂剧情用aigc-director本地工作流
3. **后期处理**：video-merge-send合并片段，video-subtitles+video-caption-burner处理字幕，FFmpeg完成所有格式转换
4. **配音与TTS**：MiniMax TTS生成高质量配音，FFmpeg混音实现人声+BGM合成
5. **多平台适配**：一次制作，输出多个比例版本，按平台分发

**关键数据指标**：
- 全流程耗时：从主题到发布约2-3小时（AI辅助）
- 成本：免费工具为主（仅即梦/HeyGen会员费用）
- 质量底线：字幕必烧录、分镜必标准化、多平台必适配
