# 第1章：内容战略与选题

**本章核心问题**

- 为什么选题决定了一篇文章80%的命运？
- 如何用竞争分析找到真正有空间的选题方向？
- 选题库怎么建才能持续产出、不陷入创意枯竭？
- 创意从哪来、怎么批量产出？

---

## 1.1 选题竞争分析：不做红海里的炮灰

### 原理说明

选错题就像在错误的战场上打仗——再努力也是输。

自媒体内容市场中，80%的内容在争夺同一批选题。"AI写作技巧"、"小红书运营干货"、"跨境电商入门"——这些方向已经拥挤到单篇内容的平均曝光趋向于零。不是内容质量的问题，是赛道本身的问题。

竞争分析的本质不是"看看别人在写什么"，而是**用数据判断：这个方向还有没有差异化空间、值不值得投入**。一个正确的选题，即使写作水平一般，也可能获得大量曝光；一个红海选题，即使写得再好，也只是在噪音里添加了一点声音。

竞争分析的核心框架是**7角度分类法**：

| 角度 | 特征 | 信号含义 |
|------|------|---------|
| **入门科普** | "什么是XXX"、"XXX入门指南" | 市场教育期，需求旺盛但内容同质化严重 |
| **深度测评** | 对比多个产品/工具，给出具体数据 | 门槛较高，差异化空间大 |
| **个人经验** | "我做了X个月的真实感受" | 读者信任度高，但容易被质疑是软文 |
| **数据报告** | 引用第三方数据，做趋势分析 | 权威性强，但获取数据成本高 |
| **争议观点** | 反共识、挑战主流认知 | 传播性最强，但风险最高 |
| **商业变现** | "我是怎么通过XXX月入X万" | 点击率最高，但内容质量参差不齐 |
| **工作流实战** | 具体工具组合、操作步骤、截图 | 转化率高，但制作成本也高 |

当某一角度的占比超过30%，说明这个角度已经拥挤。除非你有极强的差异化切入点，否则绕开。

### 实操步骤

**第一步：搜索代表性内容（工具链）**

按顺序调用搜索工具，获取宏观视角：

```
1. Perplexity → "当前[关键词]领域的热门话题和趋势"
   → 获取宏观概览，了解整个领域的热度走向

2. Exa（语义搜索）→ 搜索"[关键词]"相关高质量文章
   → 获取竞品内容列表，按相关性排序

3. Brave Search → 补充关键词变体搜索
   → 覆盖长尾和细分方向，避免遗漏

4. web_fetch → 抓取Top 5文章内容
   → 做深度分析：哪些角度已有人覆盖、覆盖质量如何
```

**第二步：按7角度分类统计**

对搜索到的20篇代表性内容，逐篇标注所属角度，统计各角度占比。

**第三步：判断拥挤与空白**

- 某角度占比>30% → **红海，慎入**
- 某角度占比<10%且有实际内容需求 → **空白机会**
- 某角度无人覆盖但搜索量可观 → **高价值蓝海**

**第四步：给出3个差异化切入点**

每个切入点必须包含：
- 具体角度（不是泛泛的"换个角度"）
- 一个可直接使用的标题建议
- 简短的差异化理由

### 参数配置

| 参数项 | 推荐设置 | 说明 |
|--------|----------|------|
| 搜索内容数量 | Top 20篇 | 太少无统计意义，太多耗时 |
| 角度分类精度 | 归入一个主角度 | 一篇文章可能有多重角度，取最主要的一个 |
| 拥挤阈值 | >30%占比 | 超过此值建议绕开 |
| 空白确认条件 | 占比<10%+搜索量>1000/月 | 两个条件同时满足才视为空白 |
| 差异化切入点 | 每个方向≥1个标题 | 不能只给方向，必须给标题 |

### 常见问题

**Q：竞争分析要花多长时间？**
完整分析一个选题方向约需30-45分钟（搜索20分钟+分类统计10分钟+切入点输出15分钟）。这个时间永远值得花——花1小时分析，可能节省你一周的写作时间。

**Q：如果搜索发现所有方向都拥挤怎么办？**
两个选择：① 找到拥挤角度里的细分变量（如"AI写作"→"AI写小红书的爆款标题"）；② 找到多个角度的交叉点（如"数据报告"+"争议观点"的组合）。

**Q：竞争分析的结果会过时吗？**
内容竞争格局变化很快。建议对核心选题每季度重新分析一次，热点话题每月复盘一次。平台算法调整也会影响竞争态势。

## 1.2 选题库构建：从灵感漂流到系统化输出

### 原理说明

没有选题库的创作者，永远在截止日期前临时抱佛脚。临时找的题，要么是之前想过的老梗，要么是热度已过的昨日黄花。

选题库的本质是**把创意的捕获和存储分离**。你在刷手机、看新闻、跟人聊天时产生的灵感——这些东西一闪而过，不记录就会永久消失。选题库的作用就是：随时随地捕获，一段时间后统一筛选，把最优质的灵感转化为可执行的内容计划。

选题库的运转分为三个阶段：

```
捕获期 → 沉淀期 → 筛选期
(随时记录)   (每周整理)   (每月/每周规划时)
```

很多人失败在第一步：想建选题库，但不知道记什么。核心原则只有一条：**记录问题，不记录答案**。当你看到一个有趣的现象、一个没解答的疑问、一个让你愣住的数据——那就是一个选题的种子。

### 实操步骤

**第一步：建立捕获系统（随时进行）**

用手机备忘录或笔记软件，建立一个始终可快速访问的"选题临时区"。记录格式：

```
[日期] [一句话描述问题/现象] — [来源：刷到什么/听到什么/自己想到]
```

示例：
```
2026-04-10 为什么深圳的咖啡店越开越多但赚钱的越来越少 — 看到一篇行业报道
2026-04-11 小红书评论区越来越像客服窗口 — 自己刷到的感受
2026-04-12 00后找工作真的更看重"不内耗"吗 — 知乎热榜
```

**第二步：每周一次结构化整理**

每周固定一个时间（推荐周六或周日），把一周的临时灵感整理到选题库：

```
选题库分类维度（每个选题填写以下字段）：

标题候选：[至少3个标题方向]
核心观点：[一句话说清楚你要表达什么]
目标读者：[谁会对这个话题感兴趣]
竞争预判：[简单判断：红海/中性/蓝海]
数据支撑：[有没有具体数据可以引用]
预计字数：[短篇1500/中篇3000/长篇5000]
预计制作成本：[1-3分，低/中/高]
预计效果：[流量潜力 1-5分]
优先级：[P0/P1/P2/P3]
```

**第三步：每月一次战略筛选**

每月做选题会时，从选题库里筛选下一阶段的内容计划。筛选标准：

- **流量潜力**：搜索量够大吗？这个话题有人搜吗？
- **差异化空间**：我的角度跟别人有什么不同？
- **制作成本**：我能承受这个内容的制作成本吗？
- **账号契合度**：这个选题跟我的账号定位一致吗？

**第四步：维护选题库的三个原则**

| 原则 | 说明 | 执行方法 |
|------|------|---------|
| 宁多勿少 | 库存量至少是月产能的3倍 | 每周补充，不设上限 |
| 定期淘汰 | 超过3个月未执行的选题降级或删除 | 每月一次 |
| 标签管理 | 给选题打标签便于检索 | 标签：热点/ evergreen/ 测评/ 观点/ 教程/ 行业 |

### 参数配置

| 参数项 | 推荐设置 | 说明 |
|--------|----------|------|
| 临时灵感区容量 | 不设上限 | 捕获第一，整理第二 |
| 选题库库存量 | 月产能×3以上 | 少于3个月产能=创意枯竭预警 |
| 整理频率 | 每周一次 | 太频繁打断创作节奏 |
| 月度筛选频率 | 每月一次 | 决定下月内容计划 |
| 过期处理 | 3个月未执行降级或删除 | 过期选题通常已失去时效性 |
| 标签数量 | 5-8个核心标签 | 太多等于没有标签 |

### 常见问题

**Q：选题库里放多少选题才够用？**
起步阶段至少20个，理想状态50个以上。但更重要的是**周转率**——再大的库存如果从不执行也是死水。宁可10个选题高频轮转，也不要50个选题全部积压。

**Q：热点选题和常青选题怎么平衡？**
建议比例：60%常青选题（SEO价值高，长期有搜索量）+ 40%热点选题（借势流量，但时效性强）。没有常青选题的账号，停止更新就等于从零开始。

**Q：选题库建立初期从哪找灵感？**
四个高效来源：① 评论区——读者问的问题是最好的选题来源；② 竞品账号的爆款——爆款是验证过的需求；③ 行业数据报告——数据是观点的弹药；④ 自己的真实经历——你踩过的坑、总结过的经验最有说服力。

## 1.3 创意头脑风暴：批量产出高质量选题

### 原理说明

创意不是灵感突发，是系统化生产的结果。

"想不出来"、"创意枯竭"、"不知道写什么"——这些问题的根源不在于缺乏灵感，而在于**没有建立创意生产的正确流程**。顶级创意人不是在等待灵感，而是在用一套方法持续产出。

创意头脑风暴的核心方法论是**约束激发创意**。完全开放的命题（如"你想写什么"）反而让人无法下手。正确的做法是：给定一个约束条件，然后在约束范围内穷尽可能性。

约束可以来自：
- **内容角度约束**：从7角度里选一个
- **格式约束**：必须是对比测评、必须是清单体、必须是故事开头
- **受众约束**：写给刚入行3个月的新手看
- **数据约束**：必须有3个以上具体数据支撑
- **反共识约束**：必须挑战一个主流观点

### 实操步骤

**第一步：选定创意锚点**

在选题库中选一个待开发的选题，作为本次头脑风暴的锚点。

**第二步：围绕锚点进行发散提问**

对锚点提出以下问题，强制自己回答（哪怕答案一开始很烂）：

```
- 这个话题的反面是什么？
- 如果只给读者一句话，最重要的话是什么？
- 读者最常见的误解是什么？
- 有什么数据可以支撑/反驳这个观点？
- 如果我是新手，我会问什么问题？
- 行业内的人最不愿意说出口的真相是什么？
- 这个话题跟什么完全不相关的领域有相似结构？
```

**第三步：执行交叉组合**

将不同问题的答案进行组合：

```
问题A的答案 × 问题B的答案 = 新角度
```

**第四步：从组合中提炼标题方向**

每个组合出一个标题候选，共输出6-12个候选标题。

**第五步：用5维标题评分筛选**

对候选标题按以下维度打分（满分10分）：

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 点击欲望 | 30% | 看到标题是否想点进去 |
| 信息密度 | 20% | 是否传递了实质性信息 |
| 清晰度 | 15% | 读者能否一眼看懂 |
| 差异化 | 20% | 和同类标题是否有明显区别 |
| 正文匹配度 | 15% | 标题是否准确反映正文内容 |

总分 = 点击欲望×0.3 + 信息密度×0.2 + 清晰度×0.15 + 差异化×0.2 + 正文匹配×0.15。选出Top 3。

### 参数配置

| 参数项 | 推荐设置 | 说明 |
|--------|----------|------|
| 单次头脑风暴时长 | 20-30分钟 | 时间过长效率递减 |
| 候选标题数量 | 6-12个 | 太少没选择，太多难决策 |
| 发散问题数量 | 6-8个 | 覆盖不同思考维度 |
| 组合交叉次数 | 不限 | 鼓励大胆组合 |
| 标题评分阈值 | Top 3 | 最终保留3个最优标题 |
| 头脑风暴频率 | 每周1-2次 | 配合选题库整理节奏 |

### 常见问题

**Q：头脑风暴时总想不出好创意怎么办？**
不是能力问题，是约束不够。尝试给自己加限制：① "这个标题里必须包含一个数字"；② "这个标题必须用反义词"；③ "这个标题必须让读者感到不适/惊讶"。约束越具体，创意越容易涌现。

**Q：批量产出的创意质量参差不齐怎么办？**
正常的。头脑风暴阶段的目标是**数量，不是质量**。先用粗糙的方式把12个标题全写出来，然后再筛选。筛选阶段用5维评分客观判断，不是凭感觉。

**Q：一个人做头脑风暴效率太低，能团队协作吗？**
可以。团队头脑风暴的变体是**"6-3-3法则"**：6个人，每人先独立想3个方向，然后每人用3分钟介绍自己的3个方向，其他人自由补充。关键原则：任何人提出想法时，其他人只能记录和补充，不能当场否定。

## 1.4 各平台选题策略差异

### 原理说明

同一个话题，在不同平台的价值完全不同。

不是所有选题都值得全平台分发，也不是所有平台都适合你的选题。选对平台，是选题策略的最后一环。

选平台看三个维度：**受众匹配度**（你的目标读者在不在这个平台）、**内容形式匹配度**（你的内容能否在这个平台最优呈现）、**流量特性**（这个平台的流量是搜索驱动还是推荐驱动）。

| 平台 | 流量特性 | 最佳选题类型 |
|------|---------|------------|
| 微信公众号 | 搜索+朋友圈传播 | 深度长文、行业分析、干货教程 |
| 小红书 | 推荐算法驱动 | 真实体验、对比测评、攻略清单 |
| 知乎 | 搜索驱动 | 专业分析、问答型、争议话题 |
| B站 | 推荐+搜索 | 深度视频脚本、教程系列 |
| 微博 | 热点驱动 | 实时评论、行业快评、热点回应 |
| 抖音/视频号 | 推荐算法驱动 | 娱乐化、情绪化、视觉冲击 |

### 实操步骤

**第一步：判断选题的流量属性**

- 如果选题有**搜索需求**（读者会主动搜索"如何..."、"XX对比XX"）→ 优先公众号/知乎/小红书
- 如果选题有**传播潜力**（观点足够争议或共鸣）→ 优先微博/抖音/视频号
- 如果选题需要**深度理解**（读者需要一步步学习）→ 优先B站/公众号长文

**第二步：平台适配微调**

同一选题在不同平台，微调的维度：

```
公众号版：加入更多数据和分析，适合深度阅读
小红书版：强调个人真实体验，标题加emoji，控制在500字以内
知乎版：增加逻辑论证，引用权威来源，使用小标题
B站版：拆解为视频脚本，开头3秒钩子，结尾引导互动
```

**第三步：分发优先级排序**

根据选题特性和平台资源，制定分发顺序：

```
首发平台（最优呈现形式）
↓ (1-7天后)
二发平台（适配改写版本）
↓ (持续)
长尾平台（SEO优化版本，永久保留）
```

### 参数配置

| 参数项 | 推荐设置 | 说明 |
|--------|----------|------|
| 单篇内容最多分发平台数 | 3个 | 太多稀释精力，质量下降 |
| 首发与二发间隔 | 1-7天 | 太短影响首发数据，太长失去时效 |
| SEO版本保留时长 | 永久 | 长尾搜索流量持续积累 |
| 平台适配改写时间 | 每平台30-60分钟 | 太简略影响效果，太久效率低 |

## 1.5 章节小结

本章解决了自媒体创作最前置的问题：**写什么**。

核心方法论只有三句话：

**第一，选题决定生死。** 一篇内容能否爆，80%在选题时就已决定。不要在红海赛道里用质量对抗流量——找到有差异化空间的切入点，比提升写作技巧更高效。

**第二，选题库是创作的弹药库。** 把创意的捕获和筛选分离。平时随手记录灵感，每周整理，每月规划。没有选题库的创作者，永远在最后一刻随机选择——随机选择的结果必然平庸。

**第三，创意可以系统化生产。** 灵感不是等来的，是用约束激发、用交叉组合、用评分筛选生产出来的。每个创作者都应该建立自己的选题库和头脑风暴流程。

**行动清单：**

1. 今天：用7角度竞争分析法，分析你下一个要写的选题方向
2. 本周：建立你的选题库，把过去积累的所有灵感整理入库
3. 本周：执行一次20分钟的头脑风暴，围绕一个核心选题产出6个以上标题
4. 本月：根据各平台特性，为你的核心选题制定分发优先级计划

---

*写作风格注记：方法论务实，数据具体。竞争分析有工具链，选题库有字段模板，头脑风暴有约束框架。读完就能执行，执行就有效果。*
