# 第3章 · 小红书运营

**本章核心问题**

- 为什么小红书的流量逻辑跟抖音、公众号完全不同？
- 爆款标题有哪些经过验证的公式？
- 如何系统化生产图文系列内容？
- 每日内容生产流的标准 SOP 是什么？
- 如何用 CLI 工具做竞品分析和热点研究？

## 3.1 小红书平台底层逻辑

### 原理说明

小红书不是短视频平台，不是公众号，不是朋友圈。它是一个**消费决策引擎**——用户打开小红书的核心动机是"买东西之前搜一搜"或"找个方案"。

这个本质决定了内容策略：小红书爆款内容的核心不是娱乐性，而是**可收藏性**。用户在你的内容里看到了值得"存下来以后用"的信息，就会收藏。收藏率越高，平台认为内容越有价值，给的流量越大。

平台有四个核心互动信号，每个信号的权重不同：

| 互动类型 | 权重 | 含义 |
|---------|------|------|
| 收藏 | 最高 | "以后用得着"——内容有实用价值 |
| 评论 | 高 | 触发讨论——内容有争议性或共鸣感 |
| 点赞 | 中 | "觉得不错"——浅层认可 |
| 分享 | 中高 | "发给朋友看"——社交货币属性 |

一个关键指标：**收藏/点赞比**。

| 比值 | 分类 | 含义 |
|------|------|------|
| >40% | 工具型 | 教程、清单、模板——用户存下来反复用 |
| 20-40% | 认知型 | 有启发性但不一定存 |
| <20% | 娱乐型 | 看完就忘 |

做小红书，优先追求"工具型"内容，收藏率天然高。

### 实操步骤

**第一步：明确账号定位**

回答三个问题：
1. 你能给用户解决什么具体问题？
2. 你的信息差在哪里？（你知道的、用户不知道的）
3. 用户收藏你的内容之后，会在什么场景下翻出来用？

**第二步：选择内容类型**

小红书四种高互动内容类型：
- **种草安利型**：产品测评、好物推荐（"XX也太绝了吧"）
- **避坑指南型**：踩坑经历+正确方法（"血泪教训"）
- **教程攻略型**：步骤清晰、可直接操作（"3分钟学会"）
- **情感共鸣型**：引发共鸣的故事+解决方案（"XX的痛谁懂"）

**第三步：建立内容矩阵**

围绕定位，建立3-5个内容支柱（pillar），每个支柱下面持续生产。比如定位"AI效率工具"：
- 支柱1：AI工具实测（工具横评、使用体验）
- 支柱2：AI提效技巧（工作流、自动化）
- 支柱3：AI行业观察（趋势、观点、批判）
- 支柱4：副业变现（用AI赚钱的真实案例）

### 参数配置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 发布频率 | 每日1-2篇 | 新号养号期每日不超过1篇 |
| 正文长度 | 300-800字 | 不超过1000字 |
| 标签数 | 5-8个 | 核心词+流量词 |
| 封面比例 | 3:4 竖版 | 小红书默认首选 |
| 标题长度 | ≤20字 | 含emoji和数字 |
| 最佳发布时间 | 工作日7-9/12-14/18-22，周末10-12/15-17/20-23 | |

### 常见问题

**Q：新号怎么养号？**
A：先互动再发内容。搜索你定位领域的关键词，给20-30篇笔记点赞+收藏+评论，持续3-5天。不要上来就发，平台会判定为营销号。发的时候先发"弱营销"内容（纯分享、无任何引导关注），积累5-10篇后再逐步加入CTA。

**Q：什么时候知道账号被限流了？**
A：正常发布后，如果自然流量（非粉丝浏览量）持续低于50，连续3篇以上，基本可以判定限流。解法：暂停发布2-3天，期间只做互动（点赞、评论），然后用一篇高质量工具型内容重新激活。

**Q：图文和视频怎么选？**
A：小红书的图文形式天然更适合"工具型"和"认知型"内容。如果你的内容是教程、清单、对比，优先图文。视频适合需要演示操作流程或讲述故事的内容。初期建议以图文为主，熟练后再加视频。

### 避坑指南

**坑1：把小红书当朋友圈发**
朋友圈是私人空间，小红书是公共内容平台。发"今天天气真好"没有任何传播力。每一条内容都必须回答用户一个问题：这跟我有什么关系？

**坑2：封面用抽象背景图**
"深色底+强调色+抽象几何"这种封面在小红书上毫无辨识度。封面必须包含文章核心信息的视觉表达——具体物体、具体场景、具体隐喻。

**坑3：正文太长**
800字是小红书正文的天花板。超过这个数，完读率断崖式下跌。如果内容确实多，拆成图文系列（第3.3节详解）。

**坑4：忽略标签策略**
标签是小红书的搜索入口。"#AI工具"是核心词，"效率神器"是流量词，组合使用才能最大化搜索曝光。5-8个标签，不要超过10个。

## 3.2 爆款文案创作

### 原理说明

小红书文案的底层逻辑是**信息密度+情绪价值**的结合。用户刷信息流的速度极快，你的文案必须在第1句就停下来。

爆款文案有一个通用结构：**Hook（钩子）→ Body（正文）→ CTA（互动引导）→ Hashtags（标签）**。

Hook 占全文权重的60%——如果第一句没能让人停下来，后面的内容再好也没用。

### 实操步骤

**第一步：写 Hook**

四种经过验证的 Hook 类型：

| 类型 | 公式 | 示例 |
|------|------|------|
| 数字+痛点 | "N个方法解决XX" | "3个方法解决副业收入为0" |
| 反差+悬念 | "以为是X，结果是Y" | "以为AI会抢饭碗，结果帮我涨了3倍工资" |
| 身份+利益 | "作为XX，分享XX" | "作为前大厂PM，说说AI真正的机会在哪" |
| 时间+结果 | "X天/月从Y到Z" | "30天用AI从0做到月入2万" |

**第二步：写正文**

正文结构：2-4个信息点，短句为主，每句≤20字。核心原则：

- 有"我"的视角——"我用它替代了3个外包"比"AI Agent可以提升效率"强10倍
- 有具体数据——"省了2000块/月"比"成本显著降低"有说服力
- 有情绪——吐槽、感叹、不确定，这些是"人味"的来源
- 去AI痕迹——不能用"赋能/闭环/底层逻辑/打法"，不能有"值得注意的是/此外/与此同时"

**第三步：写 CTA**

结尾三件套：
- 💬 互动引导："你们觉得呢？评论区聊聊～"
- ⭐ 收藏引导："码住！以后肯定用得上"
- 👋 关注引导："关注我，每天分享AI实战经验"

**第四步：选标签**

5-8个标签，按"2-3个核心词+2-3个流量词+1个趋势词"分配。

核心词：与你定位直接相关（#AI工具 #副业变现）
流量词：用户会搜索的热词（#大学生 #搞钱）
趋势词：当前热点（#2026趋势 #AI创业）

### 参数配置

| 参数 | 标准 | 检查方式 |
|------|------|---------|
| Hook 停留率 | ≥60% | 前10字是否有数字/冲突/悬念 |
| 平均句长 | ≤20字 | 数一数，超过就拆 |
| "我"的使用频率 | 每300字≥1次 | 搜索"我"字 |
| AI痕迹词 | 0个 | 搜索黑名单词汇 |
| 正文总字数 | 300-800字 | 不超过1000字 |

### 常见问题

**Q：标题写多长最合适？**
A：≤20字（含emoji）。超过20字在信息流里会被截断，用户看不到完整信息。精炼到15字以内最理想。

**Q：emoji 要不要用？**
A：适量。开头用1个emoji建立视觉锚点，正文中每个信息点前可以用emoji做分隔，但不要超过5个。太多emoji显得廉价。

**Q：如何判断文案有没有AI味？**
A：用第11章的 humanizer 方法。核心检查：有没有"赋能/闭环/底层逻辑/打法/值得注意的是/此外"这些词？每句话是不是都是差不多的长度？有没有"我"的视角？如果有问题，重写。

### 避坑指南

**坑1：开头说"今天给大家分享..."**
这是小红书上最常见的低效开头。用户不关心你想分享什么，用户关心的是"这跟我有什么关系"。直接上结论或冲突。

**坑2：正文全是信息罗列**
"第一...第二...第三...第四..."这种罗列读起来像说明书。在罗列中加入个人判断："这个我用了一周才搞明白，但一旦搞懂，效率直接翻倍"。

**坑3：结尾没有CTA**
没有CTA的笔记，互动率比有CTA的低30%以上。用户需要被告诉"下一步做什么"——不是每个人都会主动评论。

### 示例

**标题**：🔥 21岁用AI赚了第一个10万，说点真话

**正文**：
大三那年，我靠AI代写赚了人生第一个10万。
不是什么高端操作，就是把3个工具串起来用。

工具1：Cursor（AI编程）
接了20多个小程序的单子，以前要2周的代码，AI辅助下3天搞定。

工具2：Midjourney（AI绘图）
给淘宝卖家做主图，一张收200，一天出10张。

工具3：ChatGPT（文案）
帮本地商家写公众号，一篇收500。

说点真话：
- 这不体面，但来钱快
- 竞争越来越激烈，现在价格已经卷到原来的1/3
- 最大的壁垒不是技术，是获客渠道

我现在转向了AI教学，因为"教人赚钱"比"帮人赚钱"的复利高得多。

你们觉得AI副业还能做多久？评论区聊聊～

#AI副业 #大学生搞钱 #AI赚钱 #副业收入 #AI工具 #2026创业 #真实经历

## 3.3 图文系列内容生成

### 原理说明

小红书的图文系列（多图笔记）是该平台最高互动的内容形式之一。一篇5-8张的图文笔记，其收藏率通常比单图笔记高出2-3倍。

图文系列不是"一篇文章拆成几张图"。它是一套完整的视觉叙事系统——每张图都是一个独立的信息单元，合在一起构成完整的价值传递。

图文系列有两个自由维度：

| 维度 | 控制什么 | 选项 |
|------|---------|------|
| **风格** | 视觉美感 | cute、fresh、warm、bold、minimal、retro、pop、notion、chalkboard、study-notes |
| **布局** | 信息密度 | sparse、balanced、dense、list、comparison、flow、mindmap、quadrant |

风格决定"长什么样"，布局决定"信息怎么排"。两个维度可以自由组合。

### 实操步骤

**第一步：选择内容大纲策略**

三种策略对应三种内容目标：

| 策略 | 核心思路 | 适合内容 | 推荐风格 |
|------|---------|---------|---------|
| A 故事驱动型 | 个人经历为主线，情感共鸣优先 | 测评、个人分享、转变故事 | warm、cute、fresh |
| B 信息密集型 | 价值优先，高效传递 | 教程、对比、产品测评、清单 | notion、minimal、chalkboard |
| C 视觉优先型 | 视觉冲击为核心，文字最少 | 高颜值产品、生活方式、情绪内容 | bold、pop、retro |

**第二步：拆分图片内容**

| 位置 | 类型 | 信息量 | 布局建议 |
|------|------|--------|---------|
| 第1张 | 封面 | 低（一个核心观点） | sparse |
| 中间 | 内容 | 高（每张1-3个信息点） | balanced / dense / list |
| 最后1张 | 结尾/CTA | 低（引导收藏/关注） | sparse / balanced |

**第三步：保持视觉一致性**

关键方法：**参考图像链（Reference Image Chain）**

1. 先生成第1张（封面）——不用参考图
2. 用第1张作为参考图，生成后续所有图片
3. 这样可以锚定角色设计、色彩风格和插画风格

**第四步：生成与审核**

逐张生成后检查：
- 所有图片风格是否一致？（色调、线条、装饰元素）
- 信息密度是否合理？（封面留白多，内容页信息密）
- 文字是否可读？（字号够大，对比度够高）

### 参数配置

**内容类型 × 风格自动匹配**

| 内容信号 | 推荐风格 | 推荐布局 |
|---------|---------|---------|
| 美妆、穿搭、可爱 | cute | sparse/balanced |
| 健康、自然、清新 | fresh | balanced/flow |
| 生活、情感、温暖 | warm | balanced |
| 警告、重要、必须知道 | bold | list/comparison |
| 专业、商务、极简 | minimal | sparse/balanced |
| 知识、概念、效率 | notion | dense/list |
| 教育、教程、课堂 | chalkboard | balanced/dense |

**图片数量建议**

| 内容类型 | 建议图片数 |
|---------|-----------|
| 单点干货 | 3-5张 |
| 清单/对比 | 5-8张 |
| 完整教程 | 6-10张 |
| 情感故事 | 4-6张 |

### 常见问题

**Q：图文系列和短视频哪个互动率更高？**
A：在工具型、教程型内容上，图文系列的收藏率显著高于视频。在情绪型、故事型内容上，视频更优。选择标准：你的内容是"要存下来用"还是"看完就走"。

**Q：图片之间的风格差异太大怎么办？**
A：严格使用参考图像链——所有后续图片都以第1张为参考。如果风格偏差仍然大，在提示词中增加风格约束词（如"same watercolor illustration style, same color palette"）。

**Q：图文系列的封面需要和单图封面一样精心设计吗？**
A：甚至更重要。图文系列在信息流里只显示第1张，封面质量直接决定点击率。封面遵循第2章的方法，用视觉隐喻+3秒传达核心观点。

### 避坑指南

**坑1：每张图塞太多文字**
单张图的文字不超过50个字。超过这个数，在小红书的小图尺寸下根本看不清。如果信息量大，增加图片数量而不是增加单图文字。

**坑2：所有图片用相同布局**
即使信息密度相同，每张图也应该有不同的视觉节奏。封面sparse→内容balanced→高潮dense→结尾sparse，有张有弛才不会审美疲劳。

**坑3：忽略内容与风格的匹配**
"AI行业趋势分析"用cute（甜美可爱）风格，视觉信号和内容信号严重冲突。风格必须服务于内容调性。

## 3.4 竞品分析与热点研究

### 原理说明

小红书运营的效率提升，一半来自内容创作能力，一半来自**选题能力**。选题决定了你是在跟100个同行抢同一块流量，还是在一个蓝海里独享。

竞品分析和热点研究不是"看看别人在发什么"，而是系统化地回答三个问题：

1. 我的赛道里，什么样的内容数据最好？
2. 这些高数据内容有什么共性规律？
3. 我能在哪个角度做出差异化？

### 实操步骤

**第一步：用 CLI 工具批量采集竞品数据**

```bash
# 搜索关键词相关笔记
redbook search "AI工具推荐" --json

# 分析爆款笔记结构
redbook analyze-viral <note-url> --json

# 提取多篇爆款的内容模板
redbook viral-template <url1> <url2> --json

# 创作者完整分析（profile + posts）
redbook analyze <userId> --json
```

**第二步：分析互动数据模式**

对采集到的笔记计算三个关键比率：

| 指标 | 公式 | 含义 |
|------|------|------|
| 收藏/点赞比 | collected_count / liked_count | 内容实用性强弱 |
| 评论/点赞比 | comment_count / liked_count | 内容讨论性强弱 |
| 分享/点赞比 | share_count / liked_count | 内容社交货币属性 |

收藏/点赞比 >40% 的笔记值得重点研究——它们的结构就是你的模板。

**第三步：用搜索做热点研究**

```bash
cd ~/.openclaw/skills/brave-search
./search.js "小红书 AI 热点 2026" -n 5 --content
./search.js "抖音 AI 热门话题 2026" -n 5 --content
```

把搜索结果和小红书 CLI 数据交叉验证：如果两个来源都指向同一话题，说明是真正的热点而不是平台噪音。

**第四步：用7角度竞争分析法做差异化选题**

对每个候选选题，从7个角度评估竞争程度：
1. 这个角度有没有人写过？
2. 写的人多不多？
3. 最近7天有没有新内容？
4. 高互动内容的共性是什么？
5. 我能不能提供不同的视角？
6. 我能不能提供更具体的数据？
7. 我能不能结合个人经历增强真实性？

7个角度中，如果第5-7个有2个以上能答"是"，这个选题就值得做。

### 参数配置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 竞品分析频率 | 每周1次 | 跟踪赛道变化 |
| 搜索关键词数 | 5-10个/次 | 覆盖核心词+长尾词 |
| 采集笔记数 | 20-30篇/次 | 保证统计意义 |
| 高互动阈值 | 收藏/点赞 >30% | 低于此值不值得研究 |
| 选题评分维度 | 5个 | 点击欲望/信息密度/清晰度/差异化/正文匹配度 |

### 常见问题

**Q：竞品分析做多了会不会被平台判定为爬虫？**
A：redbook CLI 使用模拟浏览器方式访问，频率控制在合理范围内（每次分析间隔30秒以上）基本没有风险。不要一次性采集超过100篇笔记。

**Q：热点追还是不追？**
A：追，但要追"你赛道里的热点"。AI工具赛道的"Cursor更新了XX功能"值得追，"某某明星结婚了"跟你没关系。热点不是目的，借热点的流量推你的内容才是目的。

### 避坑指南

**坑1：只看点赞数不看收藏数**
点赞数可能是刷出来的，但收藏数很难刷。收藏/点赞比才是内容真实价值的指标。

**坑2：完全复制竞品的选题和结构**
分析竞品是为了找到规律，不是为了复制。如果你的选题跟竞品前10名的笔记一模一样，平台会判定为重复内容，限流。

**坑3：选题太多太散**
每个账号的内容支柱不超过5个。支柱越多，账号定位越模糊，算法越不知道该把你的内容推给谁。

## 3.5 每日内容生产 SOP

### 原理说明

自媒体运营的核心矛盾是**内容质量和产出速度不可兼得**。SOP 的作用是把"高质量创作"变成"可重复执行的流程"，降低每次创作的决策成本。

一个成熟的每日内容 SOP 应该做到：选题有方法论、创作有模板、封面有公式、质量有检查清单、发布有标准流程。

### 实操步骤

**Step 1：选题（15分钟）**

1. brave-search 搜索 AI 热点关键词
2. 7角度竞争分析法，选出3个差异化选题
3. 5维评分法（点击欲望/信息密度/清晰度/差异化/正文匹配度），每个选题生成12个标题选Top1

**Step 2：内容创作（30分钟）**

按3.2节的爆款文案结构写3篇正文。每篇流程：
1. 写Hook（1分钟）——4种公式选一种
2. 写正文（10分钟）——2-4个信息点，短句，有"我"的视角
3. 写CTA（1分钟）——结尾三件套
4. 选标签（2分钟）——核心词+流量词+趋势词

**Step 3：去AI痕迹（10分钟）**

用 humanizer 处理所有初稿。核心检查：
- 有没有黑名单词汇？（赋能/闭环/底层逻辑/打法）
- 有没有"值得注意的是/此外/与此同时"？
- 每句话是不是都是差不多的长度？
- 有没有"我"的视角和真实经历？

**Step 4：封面生成（15分钟）**

每篇封面遵循第2章的方法：
1. 提炼核心观点（一句话）
2. 转化为视觉隐喻
3. 选择色彩情绪
4. 生成提示词：`[主体场景] + [视觉隐喻] + [色彩情绪] + [风格] + [格式]`
5. 生成封面：3:4 竖版，1k 分辨率

**Step 5：质量检查（10分钟）**

逐项对照检查清单：
- [ ] 标题 ≤20字
- [ ] 正文 300-800字
- [ ] AI痕迹 <5%
- [ ] 开头有emoji+悬念/数字/冲突
- [ ] 短句为主，有"我"的视角
- [ ] 标签5-8个
- [ ] 封面跟内容强关联
- [ ] 结尾有互动引导
- [ ] 无违禁词

**Step 6：保存与发布**

保存到 `~/clawd/docs/daily-content/{YYYY-MM-DD}/xhs/`，然后发布：

```bash
python3 ~/clawd/skills/xhs-publisher/scripts/publish.py \
  --article ~/clawd/docs/daily-content/2026-04-15/xhs/article.md \
  --cover ~/clawd/docs/daily-content/2026-04-15/xhs/cover.jpg \
  --decision draft
```

### 参数配置

| 步骤 | 时间预算 | 关键工具 |
|------|---------|---------|
| 选题 | 15分钟 | brave-search + 竞争分析法 |
| 内容创作 | 30分钟/篇 | 爆款模板 + humanizer |
| 封面生成 | 15分钟/篇 | relay-image-gen |
| 质量检查 | 10分钟/篇 | 检查清单 |
| **总计** | **~3小时/3篇** | |

### 常见问题

**Q：每天3篇会不会太多？**
A：初期1篇/天，熟练后2-3篇。关键是持续，不是单次数量。连续30天每天1篇，比3天发10篇然后消失效果更好。

**Q：选题枯竭了怎么办？**
A：回到竞品分析。看赛道里最近30天的高互动内容，你会发现至少有10个角度你还没写过。选题枯竭不是因为没有选题，是因为你的视角太窄了。

## 本章小结

**3.1节**的核心：小红书是消费决策引擎，收藏率是最重要的信号。做"工具型"内容，用户会收藏你的内容以后用。

**3.2节**的核心：爆款文案 = Hook（60%权重）+ 正文（短句+我视角+数据） + CTA（三件套） + 标签（5-8个）。标题≤20字，正文300-800字。

**3.3节**的核心：图文系列是高收藏率内容形式。按故事驱动/信息密集/视觉优先三种策略选择大纲，用参考图像链保持风格一致。

**3.4节**的核心：竞品分析用 CLI 工具批量采集，收藏/点赞比>30%的笔记值得研究。7角度竞争分析法做差异化选题。

**3.5节**的核心：每日SOP = 选题15分钟 + 创作30分钟/篇 + 去AI痕10分钟 + 封面15分钟/篇 + 检查10分钟。3篇约3小时。

**小红书运营的最终标准：用户看完你的内容之后，有没有产生"收藏"或"分享"的冲动。如果没有，内容再好看也没用。**
