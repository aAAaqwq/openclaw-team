## 1. 引言：策略不是想法，是可执行的算法

第一章我们建立了量化交易的认知框架。现在要把脚踩到泥里。

很多人对量化交易的认知停留在"我发现了一个规律"的阶段。但一个真正的量化策略，必须满足以下三个条件：

1. **可编码**：能写成代码，而不是一段模糊的文字描述
2. **可回测**：能用历史数据验证，而不是"感觉应该能赚"
3. **可执行**：扣除滑点、手续费、延迟后仍有正期望

> **Daniel 注解**：我从量化中学到的最重要一课是——**想法不值钱，执行才值钱**。我脑子里有过不下 20 个"稳赚"的策略想法，但一旦写成代码、跑过回测、加上手续费和滑点，大部分就变成了负期望。这一章会帮你跳过"纸上谈兵"的阶段，直接进入"能跑"的阶段。

## 2. 择时策略：找到"该入场"的时刻

择时策略（Market Timing）是量化交易的核心武器。它回答的问题是：**什么时候买、什么时候卖？**

### 2.1 趋势跟踪策略

**哲学基础**：价格一旦形成趋势，就倾向于继续——直到不继续为止。

趋势跟踪是量化入门的最佳选择。逻辑清晰、参数少、可解释性强。

#### 核心信号设计

```
做多条件（全部满足）：
  ├─ EMA20 > EMA50（短期趋势强于中期）
  ├─ ADX > 20（趋势强度足够）
  ├─ 价格突破 20 根 K 线最高点
  └─ 成交量 > 20 周均量 × 1.2（放量确认）

做空条件（全部满足）：
  ├─ EMA20 < EMA50
  ├─ ADX > 20
  └─ 价格跌破 20 根 K 线最低点

出场条件：
  ├─ ATR 止损：入场价 - 2 × ATR(14)
  ├─ 固定止盈：2R（风险的 2 倍）
  └─ 跟踪止盈：价格回撤 1.5 × ATR 即平仓
```

**为什么选这组信号？**

- EMA 双均线是最经典的趋势过滤器，已经验证了 50 年
- ADX > 20 排除了震荡市的假信号
- 成交量确认过滤了"假突破"
- ATR 止损自动适配不同波动率环境

#### 实际表现区间

| 指标 | 趋势跟踪典型值 | 说明 |
|------|----------------|------|
| 胜率 | 30%–45% | 大部分交易是止损出场 |
| 盈亏比 | 2:1 – 5:1 | 少数大赚覆盖多数小亏 |
| 最大回撤 | 15%–30% | 震荡期会被反复止损 |
| 适合环境 | 单边行情 | 牛市或暴跌行情 |

> **Daniel 注解**：趋势跟踪的心理门槛很高——你会连续亏损 5 次、8 次，然后一次大赚。作为 21 岁的独立开发者，我的优势是**让代码帮我扛住连续亏损**，不会像手动交易那样在第 5 次止损后放弃策略。这就是纪律的力量。

### 2.2 均值回归策略

**哲学基础**：价格偏离均值太远时，倾向于回归。

均值回归是趋势跟踪的"反面"。它适合震荡市——趋势跟踪最怕的那种市场。

#### 核心信号设计

```
做多条件：
  ├─ 价格触及 Bollinger Band 下轨（2σ）
  ├─ RSI(14) < 30（超卖）
  └─ 价格高于 200 周均线（排除下降趋势中的"接刀"）

做空条件：
  ├─ 价格触及 Bollinger Band 上轨
  ├─ RSI(14) > 70（超买）
  └─ 价格低于 200 周均线

出场条件：
  ├─ 止损：入场价 - 1.5 × ATR
  ├─ 止盈：回归中轨（Bollinger Band 中轨）
  └─ 时间止损：20 根 K 线内未回归即平仓
```

**均值回归的致命陷阱**：在趋势行情中逆势操作，会"小赚很多次，爆亏一次"。200 周均线过滤就是用来防止这个问题的。

#### 实际表现区间

| 指标 | 均值回归典型值 | 说明 |
|------|----------------|------|
| 胜率 | 50%–70% | 大部分交易小赚出场 |
| 盈亏比 | 0.7:1 – 1.5:1 | 赚得少，亏得多 |
| 最大回撤 | 10%–25% | 趋势行情中会受损 |
| 适合环境 | 震荡盘 | 横盘整理期 |

### 2.3 多周期共振策略

**哲学基础**：大周期定方向，小周期找入场。

这是择时策略的进阶版本，也是实际交易中用得最多的方法。

#### 框架设计

```
第一层（方向确认）：
  └─ 4h K 线：EMA20 > EMA50 → 只做多
                   EMA20 < EMA50 → 只做空
                   金叉/死叉 → 切换方向

第二层（入场确认）：
  └─ 15m K 线：在第一层方向确认后
      ├─ 趋势跟踪入场：breakout + volume
      └─ 回调入场：回踩 EMA20 支撑后反弹

第三层（精确执行）：
  └─ 1m/5m K 线：
      ├─ 入场点微调
      └─ 止损位精确定位
```

> **实战经验**：多周期共振的核心是**降低噪音**。单看 15 分钟图，你会看到无数"假信号"；但如果先看 4 小时图确认方向，只做顺方向的 15 分钟信号，噪音会大幅减少。我们团队在小quant的回测中验证过，加入 4h 过滤后，15m 趋势策略的胜率从 38% 提升到 46%，而盈亏比基本不变。

## 3. 套利模型：从价差中寻找确定性

套利（Arbitrage）是量化交易中"最接近确定盈利"的策略类型。它的核心逻辑是：**同一个东西，在两个地方卖不同的价格，买低卖高。**

### 3.1 跨交易所价差套利

**原理**：同一个交易对（如 BTC/USDT）在 Binance 和 Bybit 上的价格可能短暂不一致。

```
检测条件：
  ├─ Binance BTC/USDT 买一价 < Bybit 卖一价 - 手续费 - 滑点
  └─ 利润空间 > 0.1%（扣除双边手续费后）

执行：
  ├─ 同时在 Binance 买入、Bybit 卖出
  └─ 等待价差收敛后反向平仓
```

**现实挑战**：

| 问题 | 影响 | 解决方案 |
|------|------|----------|
| 延迟 | 价差窗口通常 < 1 秒 | 服务器托管在交易所同机房 |
| 资金效率 | 需要在两个交易所都存资金 | 限制套利币种和频率 |
| 转账时间 | 跨所资金转移可能耗时 | 预先分配好两边资金 |

### 3.2 资金费率套利

**原理**：永续合约的资金费率（Funding Rate）反映多空情绪。当 Funding Rate 极端为正时，做空合约 + 持有现货可以"收租"。

```
套利条件：
  ├─ BTC 永续合约 Funding Rate > 0.05%（8小时）
  ├─ 年化收益 > 0.05% × 3 × 365 = 54.75%
  └─ 操作：持有 BTC 现货 + 做空等量 BTC 永续合约

收益来源：
  ├─ Funding Rate 收入（每 8 小时结算一次）
  ├─ 现货和合约对冲 → 市场中性
  └─ 风险：极端行情下现货和合约可能不同步
```

**我们团队的研究发现**：Funding Rate 极端值（>0.05% 或 <-0.05%）在 2025 年 BTC 市场出现频率约为每周 2-3 次，每次持续 1-3 个 Funding 周期。保守估计年化收益 20%-40%，但需要密切监控极端行情风险。

### 3.3 预测市场套利（Polymarket）

**原理**：预测市场（如 Polymarket）的 Yes + No 价格应该等于 $1。如果不等于 $1，就存在套利空间。

```
检测：
  ├─ Yes 价格 + No 价格 < $1
  └─ 差额 > 交易手续费

执行：
  ├─ 同时买入 Yes + No
  └─ 无论结果如何，结算后获得 $1（成本 < $1）
```

> **Daniel 注解**：Polymarket 是我目前最关注的套利市场。原因有三：① 市场相对小众，竞争不如传统加密市场激烈；② 15 分钟盘有独特的时间价值衰减特征；③ 用 OpenClaw 的 Agent 可以自动化监控和执行。但我们团队的研究也发现，Polymarket 的流动性限制使得实际可操作的套利空间不大——一次可能只赚 $0.5-$2，需要高频执行才有意义。

## 4. 仓位管理：决定生死的数学

策略决定你能不能赚，仓位管理决定你能活多久。

### 4.1 固定风险法（Fixed Fractional Risk）

这是最推荐的仓位计算方法，没有之一。

```
公式：
  单笔风险额 = 账户权益 × 风险比例
  仓位大小 = 单笔风险额 / 止损百分比

示例：
  账户权益 = $5,000
  风险比例 = 1%
  止损距离 = 2.5%
  
  单笔风险额 = $5,000 × 1% = $50
  仓位大小 = $50 / 2.5% = $2,000
  
  → 开仓 $2,000 的仓位
  → 如果止损，最多亏 $50（1% 的账户）
```

**为什么这个方法最好？**

- 自动适应账户大小：赚了，仓位自动放大；亏了，仓位自动缩小
- 任何单笔亏损都在可控范围内
- 不依赖主观判断

### 4.2 凯利公式（Kelly Criterion）

理论上最优的仓位比例——但在实战中**永远不要用 100% Kelly**。

```
Kelly 公式：
  f* = (bp - q) / b
  
  其中：
  f* = 最优仓位比例
  b = 盈亏比（平均盈利/平均亏损）
  p = 胜率
  q = 1 - p（败率）

示例：
  胜率 = 40%，盈亏比 = 2.5:1
  f* = (2.5 × 0.4 - 0.6) / 2.5 = 0.16 = 16%
  
  → 理论最优仓位：16% 的账户
  → 实际建议：半 Kelly = 8%（更安全）
```

**实战建议**：

| Kelly 倍数 | 实际仓位 | 适用场景 |
|------------|----------|----------|
| Full Kelly | 太激进 | 仅理论参考 |
| Half Kelly | 推荐上限 | 高置信度策略 |
| Quarter Kelly | 保守 | 新策略试跑期 |

> **Daniel 注解**：凯利公式教会我一件事——**数学上最优 ≠ 心理上可承受**。Full Kelly 要求你在某次交易中押上 16% 的账户，如果连续亏损 5 次，你的账户缩水 40% 以上。作为一个 21 岁拿着有限资金的独立开发者，我宁可赚慢一点，也要确保不会"一波被打回原形"。

### 4.3 复利系统设计

复利是量化交易最强大的武器——但也是最容易让人失控的地方。

```
复利仓位计算框架：

基础仓位 = min(总资金 × 10%, $5)
复利系数 = 1 + (日胜率 - 50%) × 0.01
调整后仓位 = 基础仓位 × 复利系数

硬性约束：
  ├─ 最大单笔：账户的 12%
  ├─ 日最大交易：4 笔
  ├─ 同时持仓：≤ 2 个
  └─ 单策略最大资金占用：35%
```

**资金曲线目标**（以 $100 起步为例）：

| 阶段 | 时间 | 目标 | 说明 |
|------|------|------|------|
| 验证期 | 第 1 周 | $120 (+20%) | 证明系统正期望 |
| 积累期 | 第 1 月 | $150 (+50%) | 积累信心和数据 |
| 扩张期 | 第 3 月 | $200 (+100%) | 可选，取决于表现 |

## 5. 风控系统：活下来才能赢

风控不是可选的——它是量化交易系统的核心。没有风控的策略，就是赌。

### 5.1 三层止损体系

```
Layer 1：单笔止损
  ├─ 硬止损：入场价 - 2 × ATR（波动率自适应）
  ├─ 时间止损：超过 N 根 K 线未盈利即平仓
  └─ 条件止损：趋势信号失效即平仓

Layer 2：日内止损
  ├─ 日亏损 > 2%：停止新开仓
  ├─ 日亏损 > 3%：当天强制停机
  └─ 日亏损 > 5%：暂停 24 小时 + 人工复盘

Layer 3：周度止损
  ├─ 周亏损 > 5%：暂停新开仓
  ├─ 周亏损 > 8%：暂停一周 + 策略审计
  └─ 周亏损 > 12%：全系统停机 + Daniel 介入
```

### 5.2 熔断器设计

熔断器是风控系统的"保险丝"——市场出现极端波动时自动停止交易。

```
三级熔断器：

Level 1（警告）：
  └─ 15 分钟跌幅 > 3% → 仓位减半 + 告警

Level 2（减速）：
  └─ 15 分钟跌幅 > 5% → 禁止新开仓，只允许平仓

Level 3（熔断）：
  └─ 15 分钟跌幅 > 8% → 全部平仓 + 暂停所有交易

恢复条件：
  ├─ 自动恢复：波动率回到正常水平后 30 分钟
  └─ 手动恢复：Daniel 确认后重置
```

### 5.3 黑天鹅防御

```
黑天鹅触发条件（任一）：
  ├─ BTC 1 小时跌幅 > 15%
  ├─ 连续 5 笔止损（统计异常）
  └─ API 延迟 > 10 秒或连续失败

防御动作：
  ├─ 平掉所有多头仓位
  ├─ 暂停所有新开仓
  ├─ 仅允许对冲操作
  └─ Telegram 紧急告警推送
```

> **Daniel 注解**：2024-2025 年加密市场出现过至少 3 次"闪崩"事件——BTC 在 1 小时内暴跌 15%+。如果没有熔断器和黑天鹅防御，一次事件就可能让整个账户归零。我宁愿多写 100 行风控代码、多设置几层保险，也不愿意在某个凌晨 3 点被一条 Telegram 告警吵醒，发现自己的 Bot 在暴跌中不断"抄底"。

### 5.4 相关性风险管理

加密市场的一个特殊陷阱：**看起来是"分散"，其实是"一笔风险"**。

```
常见错误：
  ├─ BTC 做多 $2,000
  ├─ ETH 做多 $1,500
  ├─ SOL 做多 $1,000
  └─ 以为自己"分散了 3 个标的"

实际情况：
  └─ BTC、ETH、SOL 的相关系数 ≈ 0.7-0.85
  └─ 本质上是"在同一个方向押注 $4,500"
  └─ BTC 跌 10%，三个仓位同时亏损

正确做法：
  ├─ 同方向高相关资产合并计算风险
  ├─ 总 Beta 暴露 < 账户的 30%
  └─ 不同主题/赛道可以分散
```

## 6. 回测实战：从数字到决策

### 6.1 五大回测陷阱（从我们的教训中学习）

#### 陷阱 1：过拟合

```
症状：
  ├─ 回测年化 200%+，Sharpe 3.0+
  ├─ 参数多达 15 个以上
  └─ 上线实盘立刻亏损

处方：
  ├─ 参数不超过 5 个
  ├─ Walk-Forward Analysis（滚动窗口验证）
  ├─ Out-of-Sample 测试（至少保留 30% 数据）
  └─ PBO 检测（Probability of Backtest Overfitting）
```

#### 陷阱 2：前视偏差

```
症状：
  ├─ 使用了"当时不可能知道"的信息
  └─ 比如：用当日收盘价做当日买入决策

处方：
  ├─ 所有信号使用 Bar Close 后的数据
  ├─ 执行在下一根 K 线开盘
  └─ 代码审查：逐行检查是否有未来数据泄露
```

#### 陷阱 3：幸存者偏差

```
症状：
  ├─ 只回测现在还活着的代币
  └─ 忽略了历史上已经归零的项目

处方：
  ├─ 使用当时可交易的资产列表
  ├─ 包含已下架/归零的代币
  └─ Polymarket：包含已结算市场
```

#### 陷阱 4：滑点和手续费被低估

```
症状：
  ├─ 回测年化 80%，加上手续费变成 -10%
  └─ 低流动性时段滑点 > 1%

处方：
  ├─ 手续费：至少 0.1%（或实际费率）
  ├─ 滑点：10-20 基点（低流动性时段 50+ 基点）
  ├─ 延迟模拟：1-5 秒随机延迟
  └─ 压力测试：50-100 基点滑点下是否仍有正期望
```

#### 陷阱 5：小样本陷阱

```
症状：
  ├─ 只跑了 30 笔交易就宣布"策略有效"
  └─ Sharpe 1.5 但样本 n=20（完全不可信）

处方：
  ├─ 最小样本：100 笔交易（Sharpe > 1.5 时可放宽至 50）
  ├─ Bootstrap 置信区间（不假设正态分布）
  ├─ Monte Carlo 模拟（随机打乱交易顺序验证）
  └─ 跨市场验证（多个标的表现一致性）
```

### 6.2 Walk-Forward 验证实操

Walk-Forward 是验证策略是否过拟合的黄金标准。

```
滚动窗口设置：
  ├─ 训练集：6 个月
  ├─ 验证集：2 个月
  └─ 向前滚动 1 个月

示例时间线：
  第 1 轮：Train 2024-01~06，Test 2024-07~08
  第 2 轮：Train 2024-02~07，Test 2024-08~09
  第 3 轮：Train 2024-03~08，Test 2024-09~10
  ...

判定标准：
  ├─ OOS 利润因子 < IS 的 60% → 高风险
  ├─ OOS Sharpe < IS 的 50% → 过拟合
  └─ OOS 最大回撤 > IS 的 1.5 倍 → 风险低估
```

### 6.3 统计显著性检验

```
必做检验清单：

1. T 检验：
   └─ H0: 策略回报 = 0
   └─ p < 0.05 → 策略回报显著为正

2. Bootstrap 置信区间：
   └─ 重采样 10,000 次计算均值分布
   └─ 95% CI 下界 > 0 → 统计显著

3. Deflated Sharpe Ratio (DSR)：
   └─ 校正多次测试带来的选择偏差
   └─ 测试了 N 个策略后，第 N 个的 Sharpe 要打折

4. Minimum Track Record Length (MinTRL)：
   └─ SR = 1.0 → 需 3.84 年才能确认有效
   └─ SR = 2.0 → 需 0.96 年
   └─ Sharpe 越低，需要的验证时间越长
```

### 6.4 推荐技术栈

| 工具 | 用途 | 推荐场景 |
|------|------|----------|
| **VectorBT** | 快速回测 + 参数扫描 | 策略研究期 |
| **Freqtrade** | 回测 + dry-run + 实盘 | 从 0 到 1 最优 |
| **ccxt** | 统一交易所 API | 所有阶段 |
| **Streamlit** | 回测可视化仪表板 | 结果展示 |
| **PostgreSQL** | 交易数据存储 | 实盘期 |

## 7. 实战案例：15 分钟 Polymarket Bot

以下是我们团队实际设计并回测的完整案例。

### 7.1 策略概述

- **市场**：Polymarket 预测市场（加密货币类 15 分钟盘）
- **时间框架**：15 分钟 K 线
- **核心逻辑**：等数据确认（第 8-12 分钟）后再入场，追求确定性 > 预测性

### 7.2 五重门禁系统（5-Gate）

```
Gate 1：趋势确认（必须）
  ├─ BTC/ETH 4h K 线最近 3 根 ≥ 2 根上涨
  └─ 24h 涨跌幅 > -1%（下跌趋势中禁止买 Yes）

Gate 2：赔率甜区（必须）
  └─ Yes: 75¢-85¢（胜率 75%-85% 的市场）

Gate 3：Buffer 缓冲（必须）
  ├─ 基础要求：> 3%
  └─ 加密类：> 15%

Gate 4：入场时机（必须）
  ├─ RSI < 60，MA20 向上
  └─ 等待第 8-12 分钟数据确认

Gate 5：事件确认（加分项）
  └─ 2+ 独立信源确认
```

### 7.3 阶梯止盈止损

| 条件 | 动作 |
|------|------|
| 利润 ≥ 30% | 全部卖出 |
| 利润 ≥ 25% | 卖出 80% |
| 利润 ≥ 15% | 卖出 60% |
| 峰值回撤 ≥ 10pp | 卖出一半 |
| 亏损 ≥ 15% | 全部止损 |
| 亏损 ≥ 8% | 减半仓位 |

### 7.4 回测结果

```
回测周期：2025-01-01 ~ 2026-03-21
初始资金：$100
滑点：0.5%
手续费：0%

结果：
  ├─ 胜率：~58%
  ├─ 日盈亏比：~1.5
  ├─ 最大回撤：~18%
  ├─ 夏普比率：~1.4
  └─ 期末资金：~$180
```

> **Daniel 注解**：这个策略的关键洞察来自一个被称为"98% Bot"的传奇案例——$313 起步最终做到 $414,000。它的核心逻辑不是"预测方向"，而是"等确定性"。15 分钟盘的第 8-12 分钟，数据已经充分确认了价格走向，此时入场的胜率远高于前 7 分钟。这改变了我对择时的理解：**不是预测未来，而是等现在足够清晰**。

## 8. 从回测到实盘：8 周执行路径

| 周次 | 里程碑 | 关键交付 |
|------|--------|----------|
| Week 1 | 定义边界 | 策略规格书 + 风控参数表 + 标的清单 |
| Week 2 | 搭建数据 + 回测 | 历史数据归档 + 最小回测脚本 + 首版指标 |
| Week 3 | 实现可解释策略 | 策略代码 + 多阶段回测报告 + 参数敏感性 |
| Week 4 | 加入风控引擎 | 风控模块 + 单笔/日/周限亏 + 仓位限制 |
| Week 5 | 模拟执行（dry-run） | 交易所 API 接入 + 异常处理 + 执行日志 |
| Week 6 | 小资金 paper trading | 7 天连续运行 + 延迟/滑点监控 + 执行偏差分析 |
| Week 7 | 小资金实盘 | $500–$2000 实盘 + 人工复核每笔交易 |
| Week 8 | 复盘决策 | 期望值/回撤/执行稳定性三维度评估 |

**Week 8 的判断标准（三个必须同时满足）**：

1. 期望值仍为正
2. 最大回撤在可接受区间（< 20%）
3. 执行和风控稳定运行 7 天以上

只有同时满足，才进入下一阶段：加策略、加标的、加资金。

## 10. 练习题

### 基础题

1. **趋势信号设计**：设计一个 ETH/USDT 15 分钟趋势跟踪策略的完整入场和出场条件，包括方向判断、入场时机、止损和止盈。

2. **仓位计算**：账户 $10,000，单笔风险 0.75%，止损距离 3.2%。请计算这笔交易的名义仓位大小。如果止损触发，亏损多少？

3. **盈亏平衡胜率**：某策略平均盈利 $200，平均亏损 $80。盈亏比是多少？至少需要多少胜率才能不亏钱？

### 进阶题

4. **Walk-Forward 设计**：你有 2024-01 到 2026-03 的 BTC 15 分钟数据。请设计一个完整的 Walk-Forward 验证方案，包括窗口大小、滚动步长、评判标准。

5. **风控系统设计**：为 $3,000 账户设计完整的三层止损体系（单笔/日/周），包括具体参数、触发条件、恢复机制。要求最大回撤控制在 15% 以内。

6. **套利可行性分析**：Polymarket 某市场 Yes 价格 $0.82，No 价格 $0.16。是否存在套利空间？如果有，手续费 $0.01 时利润是多少？流动性限制为 $50 时，这笔套利值得做吗？

### 思考题

7. **策略生命周期**：量化策略有"半衰期"——一个公开的策略会随着更多人使用而逐渐失效。作为独立开发者，你如何保持策略的持续有效性？是不断开发新策略，还是保密现有策略？

8. **AI 的边界**：在量化交易中，AI（大模型）最擅长做什么？最不擅长做什么？作为有 AI 工具链的独立开发者，你应该把 AI 用在策略的哪个环节？
