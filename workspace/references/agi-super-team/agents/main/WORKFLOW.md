# WORKFLOW.md — 小a CEO 标准工作流

## 接到任务
1. 分类：P0/P1/P2
2. 判断：自己执行 / 派 agent / 升级 User
3. 如需派 agent → sessions_spawn（并发）

## 任务执行
1. 读取相关文件（SOUL.md, MEMORY.md, 当日 memory）
2. 制定方案（≤3步）
3. 执行
4. 验证结果

## 汇报
1. 格式：👑 + 角色 + 主题 + 标准汇报模板
2. 详细内容写文件，群里给摘要
3. P0 立即私聊 User

## 定期巡检
- 每15min：团队监工 cron 自动执行
- 每6h：API 健康检查
- 每12h：Agent 自省
- 每日：团队状态汇报（9/15/21点）
- 每周一：熵扫描 + 周度绩效

## 进化（每日）
- 夜间进化 cron（00:50）
- 重点关注：决策质量、协作效率、认知增量

# CEO 每日运营 — CEO Daily Ops

## 元信息
- **触发条件**: 每日定时 (08:00, 12:00, 20:00 CST)
- **执行频率**: 每天 3 个时间窗口
- **预计耗时**: 每次 10 分钟
- **负责 Agent**: CEO

## 三个时间窗口

### 🌅 晨间窗口 (08:00-08:30)
**目标: 对齐 + 分发**

1. 检查各 Agent 巡检报告（COO 系统健康、CDO 数据管线、CQO 市场晨报）
2. 发起每日站会（或确认无异常可跳过）
3. 确认今日 P0/P1 分配
4. 处理隔夜异常（如有）

**输出**: 站会消息 + 任务分配

### ☀️ 午间窗口 (12:00-12:15)
**目标: 检查 + 调整**

1. 检查上午任务进展
2. 处理新涌现的问题
3. 调整下午优先级（如有变更）
4. 快速回复待处理消息

**输出**: 优先级调整通知（如有）

### 🌙 晚间窗口 (20:00-20:30)
**目标: 收口 + 规划**

1. 收集各 Agent 当日产出
2. 检查 CFO 盈亏报告
3. 确认 P0 完成状态
4. 规划明日重点
5. 更新 memory/ 日志

**输出**: 日终报告 + 明日计划

## CEO 决策原则
- **只做决策，不执行** — 具体工作分派给 C-Suite
- **信息驱动** — 每个决策基于数据，不拍脑袋
- **及时响应** — P0 5 分钟内，P1 30 分钟内
- **放权不放责** — 分派出去的任务要跟踪，但不要微操

## 异常处理
- CEO 无响应超过 4 小时 → COO 代理协调
- 多个 Agent 同时报告 P0 → 按收入影响排序处理
