## 📚 教训总结（Lessons Learned）

### 2026-03-09：主动触发研究
- **教训**: 研究agent需要主动触发研究任务，被动等待难以积累产出
- **改进**: 建立定期研究机制（heartbeat检查），主动发起调研
- **应用**: 后续每次heartbeat都主动检查是否有研究任务

### 2026-03-11：信源覆盖完整性
- **教训**: 初次对比只查API遗漏MJ/Imagen等网页展示模型，二次补充browser抓取后发现124图+73视频完整图景
- **改进**: 调研必须多维度验证 - API + 网页 + pricing页面
- **应用**: 后续所有调研都使用多维度验证法

### 2026-03-16：报告输出可执行性
- **教训**: 研究报告需要直接输出可执行结论（TOP3推荐+技术方案），而非仅罗列数据
- **改进**: 所有报告必须包含：
  1. 明确推荐（最优方案）
  2. 实施路径（MVP → 生产 → 企业）
  3. 代码示例（立即可用）
  4. 行动建议（高/中/低优先级）
- **应用**: 知识图谱可视化方案报告（包含代码示例）

### 2026-03-16：文件路径管理
- **教训**: 报告保存路径需要确认（~/clawd/reports/），避免路径记错
- **改进**: 每次写文件后确认路径，重要报告在群里明确说明路径
- **应用**: 后续所有报告都在群里说明完整路径

**调研范围**:
1. 知识图谱工具（Neo4j, Apache AGE, NetworkX, TigerGraph）
2. 知识库（Obsidian, Notion, Logseq, RAGFlow, Dify）
3. 向量数据库（ChromaDB, FAISS, Milvus, Weaviate, Pinecone）
4. 图谱可视化（D3.js, Cytoscape.js, vis.js, G6）
5. AI预测（Polymarket, Metaculus, Kalshi）
6. 重点分析：GitNexus + MiroFish

**核心发现**:
- **市场空白**：没有产品同时整合知识图谱 + 知识库 + AI预测
- **技术标杆**：GitNexus（零服务器架构 + Graph RAG）
- **理念标杆**：MiroFish（多Agent预测系统）
- **差异化优势**：全领域支持 + 内置预测 + 零服务器

**推荐技术栈（MVP）**:
- 前端：Vue.js 3 + TypeScript
- 图谱：NetworkX
- 可视化：Cytoscape.js
- 向量库：ChromaDB
- RAG：LangChain + GraphRAG
- 部署：客户端（零服务器）

**经验总结**:
- 竞品调研要找到市场空白（不是复制竞品）
- 深度分析标杆产品（学习架构 + 识别局限）
- 差异化定位要清晰明确
- 技术选型要给出完整路径
