# Telegram Evidence Rules

## 什么时候 Telegram 截图有价值

只有当截图能回答下面至少一个问题时，才值得加：

1. 它能证明这件事真的发生过吗？
2. 它能证明这个工作流真的在跑吗？
3. 它能证明这篇文章的判断来自真实协作，而不是空想吗？
4. 它能比文字更快建立信任吗？

如果四个问题都回答不了，别加。

## 默认脱敏等级

默认用 **C：强脱敏**。

### 强脱敏要做的事
- 只截有效区域
- 去掉账号身份特征
- 去掉无关用户名、头像、时间、会话列表
- 保留关键任务、关键结论、关键动作

### 强脱敏后仍然要保留的东西
- 能看出是“真实对话”的界面质感
- 关键问答 / 指令 / 结果
- 与正文论点直接相关的那一段

## 不要怎么截

### 错误 1：整屏搬运
把整个聊天列表、侧栏、头像、历史消息全带进去。

### 错误 2：截到隐私但没信息量
看起来很真，但对文章判断没有帮助。

### 错误 3：一篇文章塞太多聊天图
会让文章失去主轴，看起来像聊天记录整理。

## 推荐做法

### 做法 1：一图一证据
每张图只证明一个结论。

### 做法 2：配图前先写用途
先写清：这张图是为了证明什么，再去截。

### 做法 3：只在需要时使用
OpenClaw / Agent / 自动化工作流类文章高频使用。
工具横评类文章低频使用。

## 用途示例

### 适合截图的场景
- Agent 接到任务并执行
- 系统状态或结果回传
- 真实迭代过程
- 人机协作关键决策

### 不适合截图的场景
- 闲聊
- 重复确认
- 无关上下文
- 无法增强论点的对话
