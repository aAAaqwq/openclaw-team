# Article Image Playbooks

## 1. OpenClaw / Agent 技巧文

### 目标
证明这不是概念，而是已经在真实运行的技巧。

### 推荐配图顺序
1. 开头：Telegram 对话证据图
2. 中段：后台执行 / 配置 / 结果截图
3. 结尾：流程总结图或结果卡片

### 适合的标题类型
- OpenClaw 技巧
- Agent 协作 SOP
- 自动化发布流程
- cron / workflow / browser / node 实战

### 不适合的图
- 无关品牌图
- 纯概念 AI 插画
- 不能提供证据的新奇视觉图

## 2. AI 工具横评 / 产品实测文

### 目标
让读者知道你真的比较过工具，而不是只在复述参数。

### 推荐配图顺序
1. 开头：工具总览图 / 对比图
2. 中段：2-3 张工具界面图或官网图
3. 中后段：1-2 张真实使用截图（必要时可含 Telegram 对话证据图）
4. 结尾：总结卡 / 推荐分工图

### 适合的标题类型
- 6个AI编程工具实测
- Claude / Cursor / Copilot 怎么选
- 哪个 AI 工具更适合真实项目

### 推荐原则
- 工具图为主
- 真实聊天证据图为辅
- 如果聊天图与结论没有直接关系，就不要硬插

## 3. 观点 / 趋势 / 方法论文

### 目标
帮助读者理解判断结构，而不是证明某个工具真的存在。

### 推荐配图顺序
1. 开头：概念图 / 结论图
2. 中段：对比图 / 流程图 / 分类图
3. 结尾：一张结论卡

### 不适合的图
- 产品官网图（除非文章就在分析该产品）
- 无上下文的聊天截图

## 4. 平台运营 / 内容方法文

### 目标
证明策略是从实操中提炼出来的。

### 推荐配图顺序
1. 开头：平台后台或结果图
2. 中段：案例图 / 内容对比图
3. 必要时：1 张 Telegram 对话图，证明真实协作过程

## 5. 默认输出格式示例

```markdown
🧩 文章类型：AI 工具横评 / 产品实测
🎯 主证据类型：工具界面 + 少量真实案例
🖼️ 配图方案：
1. 工具总览图 - 建立主题 - 开头导语后
2. 工具界面图 - 支撑产品判断 - 第二章节中段
3. 工具界面图 - 支撑产品判断 - 第二章节后段
4. 强脱敏 Telegram 图 - 证明真实使用 - 第三章节后
5. 总结卡图 - 收束观点 - 结尾前
🔐 脱敏策略：C 强脱敏
⚠️ 不要加入的图：无关抽象插画、重复截图、与结论无关的聊天记录
```
