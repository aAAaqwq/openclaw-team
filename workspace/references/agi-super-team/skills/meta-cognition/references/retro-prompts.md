# Retro Prompts

Use these prompts after significant work, especially when something drifted, failed, or required repeated intervention.

## Lightweight retro

Answer in 3 bullets:
- What happened?
- Why did it happen?
- What single change would reduce recurrence?

## Full retro

Answer:
1. What was the original signal?
2. What did we first believe?
3. What turned out to be true?
4. Where did routing, judgment, or verification fail?
5. What should become durable?
   - memory
   - SOP/checklist
   - skill improvement
   - cron/monitoring change
   - nothing

## Memory filter

Only store durable items such as:
- stable decisions
- repeated failure patterns
- explicit user preferences
- architecture choices
- verified operating rules

Do not store:
- transient logs
- noisy one-off outputs
- speculative ideas with no decision

## Meta-cognitive questions

Use these when the main failure was judgment rather than tooling:
- Did we mistake status for closure?
- Did we route to the wrong owner?
- Did we skip verification because the answer sounded plausible?
- Did we fail to distinguish real issue vs stale state?
- Did we use a premium model where a cheaper one was sufficient, or vice versa?
