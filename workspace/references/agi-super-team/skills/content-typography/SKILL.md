# 中文内容视觉排版

这不是“选个字体” skill。
这是把一句标题排成**更好读、更有冲击、更像成熟内容团队做的版式**。

## 核心目标

解决 5 个问题：
1. **读不清**：字太小、太密、背景打架
2. **不聚焦**：所有字一样重，没有主次
3. **不耐看**：分行乱、对齐飘、留白脏
4. **不转化**：重点词没被打出来，3 秒内抓不住人
5. **不适配平台**：XHS、公众号、抖音的视觉逻辑混用

## 先判断：你在排什么

先把需求分成 3 类：

### A. 冲击型封面
适用：小红书、抖音图文封面、B站缩略图
目标：**3 秒抓眼 + 一眼看懂主题**
特点：
- 文案短
- 主标题权重极高
- 允许更强对比和更激进分行

### B. 编辑型封面
适用：公众号首图、掘金封面、知识星球横幅
目标：**专业、清晰、可信**
特点：
- 字数可稍多
- 重信息秩序
- 留白和对齐比“炸裂”更重要

### C. 海报型排版
适用：活动海报、课程卡片、知识总结长图头图
目标：**层级完整 + 结构稳定**
特点：
- 通常有标题、副标题、标签、时间/身份信息
- 需要严格层级，不要只会把标题放大

## 排版工作流

```
输入：原始标题 + 核心观点 + 平台 + 画面背景
  ↓
Step 1: 压缩标题（砍废字，只留钩子）
  ↓
Step 2: 标出主关键词（1-3个）
  ↓
Step 3: 决定信息层级（主标题 / 副标题 / 标签）
  ↓
Step 4: 设计分行（按意思断，不按字数硬切）
  ↓
Step 5: 选字重 / 对齐 / 留白 / 强调色
  ↓
Step 6: 按平台规格微调
```

## Step 1：先压缩标题，不要直接排

### 原则
- 能删的虚词全删
- 一张封面只讲 **1 个主判断**
- 标题尽量保留“冲突 / 利益 / 反常识 / 结果”之一

### 例子

原句：
`试了 6 个 AI 编程工具之后，说几句大实话`

压缩后可变成：
- `6个AI编程工具实测`
- `6个AI编程工具，说点真话`
- `AI编程工具，别再瞎吹了`

原句：
`为什么 90% 的 AI Agent 其实都是噱头`

压缩后可变成：
- `90% AI Agent是噱头`
- `AI Agent，大多是包装`
- `别再神化AI Agent了`

## Step 2：找主关键词，只打 1-3 个

不要每个词都强调。

优先强调：
- 数字：`90%`、`6个`
- 冲突词：`封禁`、`噱头`、`真相`、`别再`
- 赛道词：`AI Agent`、`AI编程`、`小红书`

### 强调规则
- 强调词最多 **1-3 个块**
- 同一行里最好只有 **1 个视觉重心**
- 如果已经用颜色强调，就别再给这个词加描边、投影、emoji 三连击

## Step 3：信息层级

默认只用 3 层：

### L1 主标题
- 画面里最先被看到
- 通常占 60-80% 视觉注意力
- 可以 1-2 行

### L2 补充信息
- 用来补语义，不抢主标题
- 例如：`实测 / 避坑 / 2026 / 一线经验`

### L3 标签 / 身份 / 场景说明
- 最弱一层
- 例如：`内容创业`、`AI工具`、`亲测`

**铁律：不要出现 4 个以上同级信息块。**
一旦同级元素太多，画面会散。

## Step 4：分行规则

### 1. 按意思断，不按字数断

好：
`90% AI Agent`
`是噱头`

差：
`90% AI`
`Agent是`
`噱头`

### 2. 第一行放“抓人词”
优先把最强的信息放在第一行左上或视觉中心。

### 3. 行数优先级
- 冲击型封面：**2 行最佳，3 行上限**
- 编辑型封面：**1-2 行最佳**
- 海报型：视结构而定，但主标题仍应控制在 2-3 行内

### 4. 避免这些断法
- 数字和量词拆开：`6` / `个`
- 专有名词拆开：`AI` / `Agent`
- 情绪短语拆开：`别再` / `被骗了`

## Step 5：字体与字重

## 默认建议（中文内容封面）
- 中文标题：**思源黑体 / 阿里巴巴普惠体 / HarmonyOS Sans / 苹方 / 优设标题黑** 同类风格都行
- 原则：**先选中性、清晰、粗壮的无衬线体**，别一上来玩花字

### 字重规则
- 主标题：Bold / Heavy
- 副标题：Regular / Medium
- 标签：Regular / Medium

### 一个版面里最多两种字重策略
推荐：
- 主标题 Heavy
- 其他全部 Medium

不推荐：
- 一行 Heavy，一行 Semibold，一行 Bold，一行 Medium —— 很乱

## Step 6：字距、行距、留白

### 字距
中文大标题通常：
- 默认或略微收紧即可
- **不要把中文标题拉太开**，很容易廉价

### 行距
- 冲击型标题：偏紧
- 编辑型标题：正常偏松
- 如果每行字数少，行距可以更紧；如果每行字数多，行距要放开一点

### 留白
标题块四周要有“呼吸区”：
- 不贴边
- 不贴人物脸
- 不压关键主体
- 不同时靠上、靠左、靠图形边缘都很近

一句话：**标题不是塞进去，是摆进去。**

## Step 7：对齐规则

默认优先级：
1. **左对齐**：最稳，最适合中文信息流封面
2. 居中：只适合对称、仪式感、海报感强的画面
3. 右对齐：少用，容易飘

### 什么时候左对齐最强
- 小红书封面
- 公众号编辑型首图
- 有人物/主体在右侧时

### 居中什么时候可用
- 画面主体居中
- 标题很短
- 想做“海报感”而不是“信息卡”

## Step 8：强调方式，只选 1-2 种

可用强调：
- 改颜色
- 加底色条/色块
- 放大字号
- 改字重

不要同时：
- 改颜色 + 描边 + 阴影 + 发光 + 斜切 + 下划线

### 最稳的强调组合
- **主标题放大 + 关键词换色**
- 或 **主标题纯白 / 黄字 + 关键词加色块**

## 平台适配

## 小红书封面
目标：停留、点击

规则：
- 标题尽量短，最好 8-16 字核心表达
- 关键词必须前置
- 适合高对比：白/黄/红 对 深色背景
- 可以更像“结论卡”

推荐结构：
- 主标题 2 行
- 右上/左上 1 个小标签
- 背景不要过花

## 公众号首图
目标：专业感、可信度

规则：
- 不要做成短视频封面味
- 字不要太爆，重秩序和质感
- 行数少，留白足
- 通常不需要太多情绪词

推荐结构：
- 1 行主标题 或 2 行平稳标题
- 一行小副标题/栏目名
- 更克制的配色

## 抖音图文封面
目标：快速扫过也能抓住

规则：
- 比公众号更狠，比小红书更直白
- 词要短、判断要猛
- 适合超大标题 + 小副标

## 输出格式

当用户要“优化排版”时，输出以下内容：

```markdown
## 排版方案
- 类型：冲击型 / 编辑型 / 海报型
- 主标题：
- 副标题：
- 标签：

## 分行建议
第1行：...
第2行：...
第3行：...（如需）

## 视觉层级
- 最大：...
- 次级：...
- 最弱：...

## 字体建议
- 标题字重：...
- 副标题字重：...
- 对齐：左对齐 / 居中
- 强调方式：换色 / 色块 / 放大

## 风险提醒
- 哪些字太多
- 哪些词不该强调
- 哪些地方会显乱
```

## 快速判断清单

- [ ] 3 秒内能看懂吗？
- [ ] 第一眼有明确重心吗？
- [ ] 标题是不是还能再删 20% 废字？
- [ ] 关键词有没有只打 1-3 个？
- [ ] 是否只用了 1-2 种强调方式？
- [ ] 分行是不是按语义，而不是硬切？
- [ ] 对齐是否稳定？
- [ ] 留白够不够？
