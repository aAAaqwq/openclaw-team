# 掘金内容风格指南

> 专为 AI/科技类技术文章定制
> 核心原则：专业、实用、可复现

---

## 一、标题公式

### 技术实战类
`{技术栈} {成果}：{具体实现方案}`
- "OpenClaw 自动发布多平台内容：Playwright 实战方案"

### 工具测评类
`{数字}款{工具}深度测评：{场景}最佳选择是{结论}`
- "5款AI编程助手深度测评：日常开发最佳选择是Cursor"

### 原理分析类
`深入理解{技术}：{核心机制}原理解析`
- "深入理解OpenClaw Skill系统：从零搭建Agent技能"

### 入门教程类
`{技术}入门指南：从{起点}到{终点}的完整路线`
- "AI Agent开发入门指南：从概念到第一个可用Agent"

### 踩坑记录类
`{技术}踩坑记录：{数字}个真实问题及解决方案`
- "Playwright自动发布踩坑记录：7个真实问题及解决方案"

## 三、掘金平台特色

| 特点 | 说明 |
|------|------|
| 受众 | 开发者、技术人 |
| 偏好 | 代码多、截图多、步骤完整 |
| 流量 | 搜索流量占比高（长尾效应） |
| 排版 | Markdown原生支持，代码块高亮 |
| SEO | 标题含关键词，标签选精准 |

---

## 四、标签选择策略

掘金文章设置 2-3 个标签：

**精准技术标签（选1个）：**
- `人工智能` `前端` `后端` `工具` `开源`

**场景标签（选1个）：**
- `Playwright` `Node.js` `Python` `自动化`

**流量标签（可选1个）：**
- `效率工具` `实战` `教程`

---

## 五、代码块规范

| 要素 | 规范 |
|------|------|
| 语言标注 | 始终注明语言：`js`, `python`, `bash`, `json` |
| 注释 | 关键行注释说明 |
| 截断 | 超过50行的代码截断，展示核心部分 |
| 完整代码 | 文末附GitHub链接或"完整代码见xxx" |

---

## 六、图片规范

| 类型 | 要求 |
|------|------|
| 截图 | 标注重点区域（红框/箭头） |
| GIF | 控制在3MB以内，≤10秒 |
| 架构图 | 用清晰的示意图 |
| 效果图 | before/after 对比最佳 |

## 八、掘金与其他平台的差异化

| 平台 | 深度 | 代码量 | 受众 | 核心价值 |
|------|------|--------|------|---------|
| 小红书 | 浅 | 无 | 泛人群 | 认知/种草 |
| 公众号 | 中 | 少 | 关注者 | 深度理解 |
| **掘金** | **深** | **多** | **开发者** | **可复现的实战方案** |
| 知识星球 | 最深 | 多 | 付费用户 | 独家/内幕 |

掘金文章的关键词：**可复现**。
读者看完应该能照着做出来，所有步骤、代码、配置都必须完整。
