# 🕊️ 蓝血菁英·小红书内容审核标准 v1.0

> 每条笔记发布前必须过这套审核。
> AI味检测不是看字眼，是看"这像不像一个人躺在床上刷手机时能写出来的东西"。

---

## 一、8项必查清单（每条一字不落跑完）

### 1. AI味扫描：3秒扫读法

**测试方法**：把正文随便选3段，遮住前后文，每段单独读一遍。
**判断标准**：
- ❌ 每段读起来像"正确的答案" → AI味
- ✅ 每段读起来像"随口说出来的话" → 合格

**典型AI信号（看到任何一个就重写）：**
```
- 句子长度均匀（每句都在15-25字之间）
- 语法结构完整（主谓宾齐全）
- 每段都有"总结句"（给人"写完了"的感觉）
- 用词太"正确"（不是口语而是书面语）
- 段落之间的逻辑过渡词（"所以""因此""然而""总之"）
```

### 2. 口语化解剖

逐句检查，至少70%的句子要能通过这个测试：

**❌ AI句（以下是第一篇稿子里出的问题）：**
```
"大部分人问'AI能帮我赚什么钱'是在问一个错误的问题"
→ 太完整了，像教科书引言
```

```
"正确的问题是：'我做什么事最耗时，AI能不能帮我把这部分省掉？'"
→ 自问自答，典型的AI回答结构
```

```
"因为工具本身不做任何事，你把工具放对位置它才做事"
→ 工整得不像人说话
```

**✅ 合格版本（人话版本）：**
```
"大多数人上手就问AI能赚什么钱
其实问反了

换成：我每天最烦什么活？"
```

```
"工具嘛，下载谁都会
关键是你放对位置没有"
```

### 3. 每段长度审查

| 规则 | 检查点 |
|------|--------|
| 全文最长段落 | ≤50字（约2-3行手机屏） |
| 全文平均段落 | 15-30字（约1-2行） |
| 最短段落 | 可以只有1-3个字（"对。" "真的假的。" "你细品。"） |

**❌ 有问题的段落（摘自原稿）：**
"大部分人问'AI能帮我赚什么钱'是在问一个错误的问题。正确的问题是：'我做什么事最耗时，AI能不能帮我把这部分省掉？'"
→ 两句话连在一起，30多字，像一篇文章

**✅ 正确的分节：**
"大部分人上来就问「AI能帮我赚什么钱」
其实问错了方向

换成——
我每天哪个活最烦？"

### 4. 开头黄金30字测试

**标准**：「一个正常人刷到这条笔记，看到前30字，会不会停住」

**❌ 我的问题（原稿开场）：**
"用AI搞钱之前，先搞懂这3个底层逻辑"
→ 太像"好的我来告诉你"，演讲稿开场

**✅ 需要改成：**
"你知道为什么下载了十几个AI工具还是赚不到钱吗？
不知道的话，这篇文章就是写给你的"

或者更短：
"用AI搞不到钱
真不是工具的问题"

### 5. 表情符号审查

| 规则 | ✅ 可以 | ❌ 不要 |
|------|---------|---------|
| 数量 | 全篇≤3个 | 每段一个、句句带 |
| 位置 | 自然嵌入，隔行出现 | 和文字紧贴、替换标点 |
| 类型 | ❶❷❸→ 丨◆→ 💡 | 🎉🤩🥰🔥(小红书用烂的) |

### 6. 互动钩子检查

**❌ 不合格：**
"哪个逻辑你觉得最难理解？评论区告诉我"
"评论区聊聊"
"写在评论区"

**✅ 合格：**
要么非常具体 → "你卡在第几个逻辑了？我帮你拆"
要么非常有态度 → "看完这篇还觉得工具是问题的，评论区来吵"
要么非常有惊喜 → "选一个方向，我下篇专门写"

### 7. 自己读一遍出声

**最残忍但最有效的测试**：
把写好的正文自己读一遍，用微信语音录下来，放给自己听。

- 如果读的时候觉得"不像我会说的话" → 重写
- 如果读的时候自然得像在和朋友聊天 → 合格

### 8. 朱雀交叉验证（上线前执行）

用去AI味工具（去i迹/嘎嘎降AI）过一遍检测，确保AI率 < 20%。

## 三、当前第一周稿的问题修正清单（直接从周二篇开刀）

### 原文问题（逐句过）：

>"用AI搞钱之前，先搞懂这3个底层逻辑"
→ 太正。改成：**"用AI搞钱搞不动？问题不在工具在你。"**

>"大部分人问'AI能帮我赚什么钱'是在问一个错误的问题"
→ 太书面。改成：**"大多数人的第一个问题就问歪了"**

>"正确的问题是：'我做什么事最耗时...'"
→ 自问自答，AI标志。改成：**"换成问自己：每天哪个活让我最烦？"**

>"因为工具本身不做任何事，你把工具放对位置它才做事"
→ 工整。改成：**"工具就是个扳手，你得先找到螺丝"**

### 全文重写（周二篇·审核通过版）：

```
用AI搞钱搞不动？
问题不在工具，在你。


❶ 搞钱不是问AI"我该怎么赚钱"
是问自己"哪个活最费时间"

代写文案能赚钱
不是AI写的多好
是你30分钟搞定了别人3小时的工作量
那2个半小时的差距 就是你的钱


❷ 工具不值钱，怎么串起来才值钱

下载10个工具不如搭通一个流程
你要研究的是流程
不是一个个工具怎么用

工具多用一个不会让你多赚一分钱
流程跑通了 每天自动出活


❸ 选方向别问你喜不喜欢
问AI哪里不行

AI搞不定判断
AI搞不定信任

所以你往这两个方向走就对了：
做好"AI做不好的事"
然后让AI给你打下手


三个逻辑 其实就一句话：
你的价值 = 你帮别人省的时间

不是AI代替你 是你用AI代替了别人。


看懂了吗。没看懂的在评论区问，我回。
```

**这个版本对比原版的变化：**
- 无"底层逻辑""正确的问题""因为...所以"等AI高频词
- 句子有断有续，有重复（"工具""流程"出现了两遍）
- 开头直接反转，不是"大家好我是"的结构
- 有口语化省略（"没看懂的在评论区问"——缺少主语）
- 金句收尾但不煽情

## 四、内容质检流程（每次发布前跑一遍）

```
□ 第①关：AI味扫描（3秒扫读3段，有"正确感"就退稿）
□ 第②关：口语化解剖（逐句测试"这像不像人说的话"）
□ 第③关：段落长度（最长≤50字，有1-3字短段）
□ 第④关：开头测试（前30字让人停住）
□ 第⑤关：表情审核（≤3个，不用小红书流行款）
□ 第⑥关：互动钩子（具体/有态度/有惊喜）
□ 第⑦关：出声朗读（不像自己说的就重写）
□ 第⑧关：朱雀AIGC检测（AI率<20%）

8关全过 → 发布
任意一关不过 → 退回重写
```

## 五、日常避坑清单（写的时候随时对照）

| 看到自己写了这种字眼 | 立刻删 |
|---------------------|--------|
| "首先/其次/再次/最后" | 用1.2.3.或者❶❷❸ |
| "总的来说/总而言之" | 直接删掉 |
| "以下X个方法" | 变成"我试下来有用的就X个" |
| "是因为..." | 变成"说白了" |
| "给大家分享" | 删掉，直接上干货 |
| "我们" | 改成"我" |
| "如果你也XX" | 改成"你也是？我前几个月也一样" |
| 排比句 | 拆成短句，长短交错 |
| 完整的因果关系 | 打破它，留白 |
