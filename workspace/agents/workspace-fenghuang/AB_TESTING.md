# 【凤凰】内容A/B测试框架 v1.0

> 目的：每一篇内容不是"发布"就完事，而是通过A/B测试持续优化标题、封面、开头段落、CTA。
> 原则：数据驱动，不靠感觉。每次测试只变一个变量。

| 维度 | 版本A | 版本B | 测试平台 |

| **数字vs问题** | "5个让你…的方法" | "为什么你总是…？" | 公众号/知乎 |

## 二、测试流程

```
Step 1: 确定测试假设（一次只变一个变量）
  例：假设"数据前置"比"金句前置"能提升20%打开率

Step 2: 创建A/B版本（使用humanizer + brand-voice）
  → 两个版本仅开头的第1段不同，其余完全相同

Step 3: 分配发布平台
  → 版本A → 公众号
  → 版本B → 知乎（同时段发布）
  (或：版本A → 小红书主号 / 版本B → 小红书小号)

Step 4: 数据采集（24小时后）
  → 打开率 / 完读率 / 互动率 / 转发率

Step 5: 结论归档
  → 记录胜出版本+数据+学习点
  → 胜出版本的策略固化到品牌语调指南

Step 6: 下一轮测试
  → 换一个变量继续
```

## 四、A/B测试执行SOP（与humanizer配合）

```
1. humanizer 生成内容正文（品牌语调标准版）
2. 复制正文 → 创建2个版本
3. 版本A: 使用"数据前置"开头（调用web_search查最新数据）
4. 版本B: 使用"故事/金句前置"开头
5. 两版本正文完全相同
6. 在同一时间发布到不同平台
7. 24小时后对比数据
```
