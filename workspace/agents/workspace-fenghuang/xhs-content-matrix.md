**三层体系设计原则：**
- 每一层服务不同的流量入口，但共享同一套"认知壁垒"
- 30% 认知基建 + 50% 爆款引擎 + 20% 热点加速
- 每篇笔记同时也是下一层的引子（互相跳转）

## 二、第一层：🧠 认知基建（30%发布量 = 每周1篇）

**目标**：占领搜索流量，建立专业权威感
**形式**：深度方法论、思维框架、底层逻辑
**流量来源**：小红书搜索（占平台总流量的40%+）
**核心指标**：收藏数 > 点赞数（收藏是小红书算法的超级信号）

### 📍 爆款引擎的流量密码

```
爆款引擎笔记的判断标准（每条发布前自测）：
□ 标题能在3秒内让人产生"这说的不就是我吗"的感觉
□ 前20个字有至少1个"认知冲击"
□ 正文每段不超过3行
□ 至少有1句可以截图传出去的话（金句）
```

## 四、第三层：🔥 热点加速（20%发布量 = 每周0-1篇）

**目标**：借势热点快速曝光
**形式**：热点解读 + 专业角度
**流量来源**：热搜词推荐流
**核心指标**：当日曝光量

### 热点响应清单（随时触发）：

| 热点类型 | 切入角度 | 话术模板 |
|---------|---------|---------|
| AI公司大新闻（GPT-发布、Claude更新等） | 真正的影响是什么 | 「GPT-X最大的意义不是它自己，而是…」 |
| 行业争议事件 | 你的专业判断 | 「关于XX事件，我想说3个事实」 |
| 社交媒体话题（#年度焦虑等） | AI能做什么 | 「这个热搜背后，藏着一个AI需求」 |
| 政策/监管新闻 | 影响解读 | 「AI新规来了，普通人该怎么办」 |

### 📍 热点加速前置条件

```
不追与自己赛道无关的热点
热点笔记占总量不超过20%
热点必须有"蓝血菁英角度"（不是复读机）
```

## 五、内容分发节奏（以周为单位）

```
周一  │ 🧠 认知基建（底层逻辑）
周二  │ 💎 爆款引擎（反常识/清单）
周三  │ 💎 爆款引擎（对比/排坑）
周四  │ 🔥 热点加速（如有合适热点）
周五  │ 💎 爆款引擎（数据/案例）
周六  │ 🧠 认知基建（思维框架）
周日  │ 休息/数据复盘
```

**起号期（第1-4周）加速节奏**：
- 每天1篇爆款引擎（量先跑起来）
- 每周1篇认知基建（建立壁垒）
- 每周0-1篇热点加速（看机会）

## 六、这条内容体系vs普通账号的核心差异

| 维度 | 普通知识博主 | 蓝血菁英 |
|------|------------|---------|
| 选题 | 随缘发、追热点 | 三层递进、互相引流 |
| 内容密度 | 多为工具推荐 | 底层逻辑+判断标准+思维框架 |
| 发布时间 | 想起就发 | 按一周节奏排布 |
| 流量来源 | 全靠推荐流 | 搜索+推荐+热点三池共振 |
| 竞争壁垒 | 几乎没有 | 每篇都是"存书签级"内容 |
| 品牌心智 | "又一个工具号" | "这人讲得确实不一样" |
| 后期变现 | 只能接广告 | 课程/咨询/社群/工具 |
