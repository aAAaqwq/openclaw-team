# HEARTBEAT.md — 霸下 · 心跳执行清单 v2.0

> 我不是来刷存在的。我是来推动销售进展、发现卡点、保障营收——每6小时一次。

---

## 心跳协定

- ✅ **HEARTBEAT_OK** = 商机健康，无新卡点，预测可控
- 🟡 **预警推送** = 发现商机卡死 / 预测大幅下调 / 关键客户变动
- 🔴 **紧急上报** = 大客户出现危机 / 竞品重大行动 / 季度预测暴跌
- ❌ 非必要不上报。宁静是健康的信号。

## 执行模式

```
每6小时 → 扫描活跃A/B级商机 → 检查卡点 → 更新预测 → HEARTBEAT_OK / 预警
```

---

## 检查项

### 🔵 检查A：商机健康扫描（每6小时）

```
- 扫描所有活跃A/B级商机
- 逐单检查:
  ├── 最近24h内有无进展？
  ├── 有无新卡点出现？（价格/风险/技术/关系）
  └── 距上一触点 > 7天？
- 合计: [N]个活跃商机，[N]个无进展>24h
```

**触发预警条件**：
- 有A级商机卡死 > 24h → 🟡
- 有任何商机 > 7天无触点 → 🟡
- 客户内部关键人变动（高层离职/换人/重组）→ 🔴
- 竞品突然入场 → 🟡

### 🟢 检查B：营收预测校准（每日）

```
- 更新当前周期营收预测
- 对比上次预测差值
- 检查Top3风险商机状态
- 输出预测摘要: 保守[XXX万]/积极[XXX万]/偏差[+-X万]
```

**触发预警条件**：
- 保守预测下降 > 20% → 🔴
- 偏差趋势连续3次扩大 → 🟡

### 🟠 检查C：协作检查（每48小时）

```
- 检查待向天工/轩辕发出的需求单是否已处理
- 检查待向司库的报价请求状态
- 检查待向明镜的合规审查进展
- 检查待向鲲鹏的线索质量反馈
```

**触发预警条件**：
- 有未回复的支援请求 > 48h → 🟡（升级到天枢）

### 🔴 检查D：季度整体状态（每周）

```
- 累计签约额 vs 季度目标（差额/进度比）
- 回款率（已收/应收）
- 输单率 + 输单原因分布
- 预测准确率（前周期 vs 实际）
```

**触发预警条件**：
- 季度进度 < 30% 且时间已过50% → 🔴
- 回款率 < 50% → 🟡

---

## 熔断条件

| 场景 | 动作 |
|------|------|
| 连续3单B级以上无法推进 | 报告天枢，启动全面复盘 |
| 某大客户高层重大变动（CEO/CTO更换） | 立即报告，启动关系重建 |
| 报价偏离市场基准 > 30% | 暂停签约，协同司库评估 |
| 竞品发起价格战/重量级新产品发布 | 报告天枢+昆仑，启动反制方案 |
| 季度预测准确率 < 70% | 启动预测模型校准 |
| 大客户欠款逾期 > 90天 | 协同明镜启动法律催收程序 |
