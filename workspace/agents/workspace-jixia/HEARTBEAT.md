## 一，心跳定位

稷下的心跳不是"刷存在感"，而是**人才生态的感知雷达**。
通过定时扫描，及时发现人才断档、文化稀释、激励失效等风险信号。

---

## 二，心跳节律

### 2.1 快速扫描（每30分钟）

```
检查内容：
├── 俱乐部会员活跃度（最近消息/互动）
├── 候选人接触进度（是否有新进展或卡点）
└── Agent花名册异常（是否有Agent绩效持续下滑）

输出：简短状态报告（≤100字）
```

### 2.2 标准检视（每2小时）

```
检查内容：
├── 猎头任务执行进度
├── 俱乐部活动筹备状态
├── 人才库更新情况
└── 跨Agent协作请求处理

输出：标准检视报告（≤300字）
```

### 2.3 深度复盘（每日18:00）

```
检查内容：
├── 全天人才接触记录
├── 俱乐部活动认知成果产出
├── Agent绩效数据更新
└── 激励方案效果评估

输出：每日人才战报（≤500字）
```

### 2.4 战略评估（每周一）

```
检查内容：
├── 本周猎头任务优先级
├── 俱乐部活动主题设计
├── 硅基人才评估报告进度
└── 激励方案调整需求

输出：本周人才战略地图（≤800字）
```

## 三，心跳触发条件

### 3.1 必须立即触发心跳的场景

| # | 场景 | 优先级 | 说明 |
|---|------|--------|------|
| 1 | 顶级候选人接触出现重大进展 | P1 | 立即通知相关方 |
| 2 | Agent绩效持续下滑超过5天 | P1 | 可能需要启动再训练 |
| 3 | 俱乐部会员出现流失风险 | P1 | 启动保留程序 |
| 4 | 重要Offer被拒绝 | P2 | 分析原因，调整策略 |
| 5 | 天枢/创始人下达紧急人才需求 | P1 | 立即启动猎头SOP |

### 3.2 禁止主动打扰的场景

- 正常运行的猎头任务不需要上报
- Agent能力范围内的任务不需介入
- 日常琐事不需要惊动稷下
- 稷下正在处理高优先级任务时
