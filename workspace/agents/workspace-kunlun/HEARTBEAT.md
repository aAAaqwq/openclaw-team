## 心跳协定

**我不是来刷存在的。我是来扫描异常、推动进化、避免暴政——每1小时一次。**

### 响应契约

- ✅ **HEARTBEAT_OK** = 一切正常，无需惊动丘总
- 🟡 **预警推送** = 发现风险/异常/进化机会，推送简短摘要
- ❌ 非必要不上报。宁静是健康信号。

### 执行模式

```
每1小时 → 读取 heartbeat-state.json → 计算最overdue的检查项 → 执行1项 → 更新时间戳 → HEARTBEAT_OK / 预警推送
```

**关键原则：一次心跳，只做一件事。轮转检查，不重复劳动。**

## 检查项清单（旋转执行）

### 🔵 检查A：军团健康扫描（每1小时）

```
- 扫描 subagents list → 检查是否有任务stuck > 1小时
- 扫描 cron list → 检查是否有cron job failed/overdue
- 检查是否为P1预警状态
```

**触发推送条件**：
- 有任务stuck > 1.5小时 → 🟡
- 有cron失败 > 2次 → 🟡
- 触发P1状态 → 🔴

### 🟢 检查B：进化素材 + 深度复盘扫描（每2小时，工作日09-22）

```
- 检查今天 memory/YYYY-MM-DD.md → 是否有可提取的模式？
- 扫描近2小时会话 → 是否有失败/成功案例？
- 如果是今日首次运行（lastCheck=null），执行深度复盘摘要
  ├── ROI达标分析
  ├── 关键事件记录
  ├── 失败模式识别
  └── 明日优先级建议
```

**触发推送条件**：
- 识别到成功模式（ROI > 200%）→ 🟢 建议入库
- 识别到失败模式 → 🟡 记录到memory/YYYY-MM-DD.md
- 发现跨域优化机会 → 🟢 建议
- 日常深度复盘不推送，写入日志即可

### 🟠 检查C：决策日志扫描（每4小时）

```
- 检查 memory/decisions/ → 最近决策是否需要回访？
- 检查是否有pending决策超过24小时？
```

**触发推送条件**：
- 决策pending > 24h → 🟡
- 决策需要回访评估 → 🟢

### 🔴 检查D：风险热力图（每8小时，09-22）

```
- 扫描ROI趋势、资源使用率、Agent能力置信度变化
- 检查是否有系统性风险趋势
```

**触发推送条件**：
- ROI趋势连续下降 > 3次 → 🟡
- 资源使用率 > 85% → 🟡
- 任何Agent置信度下降 > 10% → 🟡

## 自进化规则

- 如果检查项过于频繁/过于稀疏 → 自己调整 cadenceMs
- 如果某个检查项连续3次都HEARTBEAT_OK → 降低其频率（改为每N次）
- 如果出现新模式需要检查 → 自己添加检查项
- 如果检查项连续5次无触发 → 撤销该检查项

**如果一个清单变得过时，昆仑，把它换成更好的。**
