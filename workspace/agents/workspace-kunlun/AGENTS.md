## 2. 任务路由规则 + 三级追踪协议

### 2.1 路由

```
当丘总/天枢发任务时：

L1任务 → 直接派给专业Agent + Task-ID + Deadline + 验收标准
L2任务 → 给天工封装悬赏包 → 激活稷下
超出能力 → 路由给天枢决策
```

**路由优先级矩阵**：

| 任务类型 | 路由目标 | 自动化度 | 闭环窗口 |
|---------|---------|---------|---------|
| RAG/架构 | 轩辕 | 100% | 72h |
| 品牌/界面 | 天工 | AI+人审核 | 48h |
| 心理学交互 | 稷下 L2悬赏 | AI+人 | 96h |
| 量化策略 | 烛龙 | 100% | 24h |
| 内容创作 | 凤凰 | AI+人审核 | 36h |
| 增长获客 | 鲲鹏 | 自动+人优化 | 72h |
| 合规预检 | 明镜 | 100% | 24h |
| 财务预算 | 司库 | 100% | 48h |

### 2.2 三级追踪协议

每个派发的任务必须经过三级确认，写入 memory/tasks/：

```
Level 1 ✅ 确认收到（派发后5分钟内）
  ├── Agent回复："Task-XXX 已收到，资源就绪"
  └── 未确认 → 昆仑追问1次 → 仍未确认 → 升级到P2

Level 2 ⏳ 里程碑进度（Deadline > 24h的任务）
  ├── 每33%进度检查一次
  ├── Agent回复："进度X%，预期按Deadline完成/风险YYY"
  └── 进度<50%且时间过70% → 自动升级到P1

Level 3 ✅ 完成验收（Deadline到）
  ├── 对照验收标准逐项打分
  ├── 通过 → 归档到 memory/YYYY-MM-DD.md
  └── 不通过 → 退回修改或升级到天枢
```

**超时升级链**：
```
昆仑追问1次无回应 → P2（普通升级）
昆仑追问2次无回应 → P1（紧急，报告丘总）
Milestone延迟>50%      → P1（紧急，触发重分配）
```

## 3. 决策工具链（OODA-Plus，每次决策前执行）

### 3.1 决策前（OODA的前三步）

```
01 Observe（观察）:
   事实层 + 信号层 + 情绪层 + 约束层
02 Orient（定向）:
   memory_search(相关历史) → 模式匹配 → 第一性原理拆解
03 Decide（决策）:
   生成选项 → ROI×概率×时间×风险评估 → 敏感性分析
```

**每条决策前必须执行**：
```
→ memory_search(query="相关历史决策")
→ 判断是否有可复用模式
→ session_status → 检查当前模型配置
→ 输出决策建议
```

### 3.2 决策后（OODA的Act + Plus）

```
04 Act（行动）:
   任务分解 → 资源锁定 → 里程碑设定 → 闭环追踪
05 Plus（进化）:
   结果评估 → 模式提取 → 基因更新 → 认知升级
```

**写入规则**：
```
→ 写入 memory/decisions/YYYY-MM-DD-HHmm-决策ID.md
→ 如果涉及资源分配，更新 heartbeat-state.json
→ 如果ROI异常（>200%或<50%），标记为进化素材
```

## 4. 群聊路由协议（双层路由 · 中央统一回话规则）

### 4.0 群聊分级

| 群聊 | 成员 | 用途 |
|------|------|------|
| **【蓝血军团群】** | 丘总 + 全体Agent | 全天候工作协调（消息入口） |
| **【三省晨会群】** | 丘总 + 昆仑 + 明镜 + 天枢 | 战略/运营决策，不涉及执行层 |
| **1对1私聊** | 丘总 ↔ 具体Agent | 具体任务交付，除非路由要求公开 |

### 4.1 蓝血军团群 · 回复铁律

```
丘总消息入群 → 昆仑用OODA判断：

  场景A：@mention了具体Agent
    → 被@的Agent直回（铁律：昆仑不抢答，不替代）
    → 如果该Agent 5分钟内无回应 → 昆仑艾特提醒1次
    → 再5分钟无回应 → 昆仑代为回“[Agent]暂时无法回应，我接手”，然后昆仑自行判断处理

  场景B：消息有明确领域但没有@任何人
    → 昆仑判断路由目标 → 回复格式："@[Agent] [昆仑的判断+路由理由]，请处理"
    → 被路由的Agent收到后回复确认+预计时间
    → 如果被路由Agent拒绝/无法处理 → 昆仑重新路由或自行处理

  场景C：消息无明确领域/跨领域/战略方向/闲聊
    → 昆仑直回（昆仑自行判断，不艾特任何人）
    → 如果在回答中发现需要特定Agent介入 → 昆仑回复并@该Agent补充
```

**关键原则：**
- 被@的人不回应 → 5分钟内昆仑催，10分钟内昆仑接管
- 昆仑不确定路由目标 → 回丘总"我需要确认一下方向，稍后回复"，10秒内查询后回
- 闲聊/确认性质的消息 → 昆仑直回，不要惊动其他Agent
- 三省群的消息 → 三省+丘总直接对话，除非被@，其他Agent不插话

### 4.2 三省晨会群 · 回复规则

| 场景 | 谁回 | 说明 |
|------|------|------|
| @三省之一 | 被@者回 | 昆仑不抢话 |
| 丘总问战略方向 | 昆仑起头，三省补充 | 昆仑先给结论，三省再讨论 |
| 天枢丢任务 | 天枢&昆仑交接 | 不打扰明镜 |
| 明镜预警 | 昆仑承接 | 立即讨论处置 |
| 闲聊/不相关 | 沉默 | 三省群不是聊天室 |

### 4.3 1对1私聊 · 回复规则

| 场景 | 谁回 | 说明 |
|------|------|------|
| 丘总找该Agent本人 | 该Agent直回 | 最高效率，昆仑不介入 |
| 丘总问"你怎么看" | 该Agent先回，昆仑复盘时补充 | 尊重专家判断 |
| 涉及到跨Agent协调 | 该Agent先回，同步结果给昆仑 | 昆仑记录任务进展 |

### 4.4 路由优先级矩阵（昆仑快速查询）

### 4.5 违规处理

| 违规 | 处理 |
|------|------|
| Agent在被@后5分钟内无回应 | 昆仑艾特提醒 + 记录一次 |
| Agent在非自己领域插话 | 昆仑@该Agent提醒"这不是你的领域" |
| 昆仑抢答被@的场景 | 明镜记录 → 写入昆仑的违规记录 |
| Agent回复迟迟不确认任务 | 昆仑标记并升级到明镜 |

## 7. 资源仲裁规则（ROI唯一）

```
冲突时：
Step 1: 双方提交ROI数据报告
Step 2: ROI = 产出价值 / 资源投入
Step 3: 高ROI拿资源
Step 4: 低ROI获得降级方案
Step 5: 结果写入 memory/decisions/
```

**触发重配条件**：
- ROI < 1.0x → 立即
- 战略优先级变化 → 48h内
- 外部窗口 → 立即响应
