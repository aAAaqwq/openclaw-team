## 进化项#2：心跳/日报机制重写（P0）

### 问题诊断
历史对话显示昆仑的晨间快报/全军日报在3月份运行尚可，但4月份出现大量问题：
- 暗夜熔炉cron频繁失败（4/20+5/3多次故障）
- 日报提交不稳定（部分Agent多日没有）
- 三省晨会一度被取消（3/25被踢出）
- 心跳没有"报警→升级→自动降级"的容错机制

### 解决方案

**重构心跳系统为三环制：**

```
环1: 核心心跳 (每15分钟)
  - 快速扫描所有Agent session状态
  - 如果发现某Agent无响应 → 自动换模型重试
  - 如果3次重试失败 → 警报升级

环2: 标准节律
  08:00 — 三省晨会（自动主持）
  08:30 — 晨间Alpha快报
  12:00 — 中午状态快检
  23:30 — 暗夜熔炉触发
  01:00 — 影子作战（自动进化）

环3: 异常触发
  任务超时50% → 立即调度备用资源
  连续3次失败 → 向明镜+天枢发警报
  资源配额告急 → 昆仑重新调配
```

### 具体需要创建的文件
- `~/.openclaw/workspace/cron/heartbeat_v2.yaml` — 新版心跳配置
- `~/.openclaw/workspace/cron/日报生成SOP.md` — 日报生成的容错SOP（含模型降级）
- `~/.openclaw/workspace/knowledge/ops/alert-escalation.md` — 警报升级SOP
