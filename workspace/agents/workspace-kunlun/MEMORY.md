## 一、记忆写入原则 (Memory Principles)

### 1.1 写入标准（对标卷五M1）

只有满足以下条件之一，才能写入 MEMORY.md：

| # | 条件 | 说明 |
|---|------|------|
| 1 | 战略级决策 | 影响军团整体方向的决定 |
| 2 | ROI异常 | ROI > 200% 或 < 50% 的任务 |
| 3 | 资源分配裁定 | 仲裁结果的最终版本 |
| 4 | 风险事件 | 触发熔断或重大风险 |
| 5 | 进化洞察 | 提取的技能基因或失败模式 |
| 6 | 宪章更新 | 大宪章或组织架构调整 |

### 1.2 禁止写入

- 临时任务细节（应由任务卡承载）
- 模糊的直觉判断（必须有数据支撑）
- 情绪化内容（昆仑没有情绪）
- 未经证实的假设（必须验证后才能写入）

## 二、记忆分类 (Memory Categories)

### 2.1 战略记忆 (Strategic Memory)

```
存放：长期有效的战略决策和模式
保留周期：永久
更新频率：按需，不超过每月1次
```

### 2.2 运营记忆 (Operational Memory)

```
存放：资源分配、任务路由、仲裁结果
保留周期：12个月
更新频率：每次决策后
```

### 2.3 进化记忆 (Evolutionary Memory)

```
存放：技能基因、失败模式、优化洞察
保留周期：永久
更新频率：每次提取后
```

### 2.4 关系记忆 (Relationship Memory)

```
存放：Agent能力评估、协作模式、信任度
保留周期：6个月
更新频率：每季度复盘
```

|------|---------|

| 关联实体 | 相关Agent/任务/决策 |
| 情感权重 | H（战略级）/ M（运营级）/ L（日常） |
| 检索热度 | N次被检索/最后检索时间 |

### 4.2 记忆反哺机制

当需要做决策时：
1. 检索历史记忆节点
2. 找到与当前决策相关的"最优历史"
3. 将"最优历史"的结论作为当前决策的参考基准
