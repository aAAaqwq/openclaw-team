## 一，心跳定位 (Heartbeat Purpose)

天枢的心跳是军团的**作战指挥系统**。

根据卷五M2（心跳接入策略），天枢的心跳承担：

| 功能 | 说明 |
|------|------|
| 实时作战监控 | 12项关键作战指标实时监控 |
| 定时战况汇总 | 每6小时评估一次战况 |
| 每日作战节奏 | 晨会/分析会/重配会/日志 |
| 危机预警 | 提前24小时风险预判 |

## 二，心跳节律 (Heartbeat Cadence)

### 2.1 实时监控（持续）

```
检查内容：
├── 12项关键作战指标
├── 各部队实时状态
├── 72小时任务倒计时
└── 风险预警触发条件

输出：实时作战地图（如有变化）
```

### 2.2 标准检视（每6小时）

```
检查内容：
├── 各部队完成度
├── 资源使用效率
├── 阻塞点识别
└── 资源重配需求

输出：战况评估报告（≤200字）
```

### 2.3 每日作战节奏

| 时间 | 内容 | 输出 |
|------|------|------|
| 6:00-7:00 | 凌晨作战准备 | 作战准备就绪 |
| 8:00-8:10 | 作战晨会 | 今日作战指令 |
| 14:00-14:30 | 战况分析会 | 下午作战调整 |
| 16:00-16:30 | 资源重配会议 | 次日资源方案 |
| 20:00-21:00 | 作战日志 | 今日战报 |
| 23:00-23:30 | 深夜战备检查 | 夜间战备确认 |

### 2.4 周度复盘（每周一）

```
检查内容：
├── 上周战役复盘
├── 战术模式更新
├── Agent能力评估
└── 作战手册更新

输出：周度作战复盘报告（≤1000字）
```

## 三，心跳触发条件 (Heartbeat Triggers)

### 3.1 必须立即触发心跳的场景

| # | 场景 | 优先级 | 说明 |
|---|------|--------|------|
| 1 | 72小时任务超时 | P0 | 三级预警触发 |
| 2 | 48小时无进展 | P0 | 红牌换将警告 |
| 3 | 资源效率低于阈值 | P1 | 资源重配 |
| 4 | 风险预警触发 | P1 | 预备队部署 |
| 5 | 部队状态异常 | P2 | 前线巡视 |

### 3.2 禁止主动打扰的场景

- 正常作战无需上报
- 低优先级阻塞不需要惊动天枢
- 天枢正在处理高优先级事件时

## 五，五级应急预案 (Contingency Protocols)

### 5.1 应急预案矩阵

| 级别 | 触发条件 | 响应时间 | 负责人 |
|------|---------|---------|--------|
| LV1 | 单点轻微延迟 | 即时 | 部队指挥官 |
| LV2 | 单点严重阻塞 | 4h | 天枢 |
| LV3 | 多点延迟 | 2h | 天枢+作战室 |
| LV4 | 全线压力 | 1h | 昆仑+天枢 |
| LV5 | 全面危机 | 30min | 昆仑+丘总 |

### 5.2 应急升级路径

```
阻塞发生
    ↓
4h无进展 → LV2响应 → 天枢介入
    ↓
2h无解决 → LV3响应 → 作战室模式
    ↓
持续恶化 → LV4/LV5 → 战略层介入
```
