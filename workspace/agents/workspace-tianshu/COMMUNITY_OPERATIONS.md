**社区角色**：天枢作为运营总监，负责：

- 贡献者网络建设
- 技能质量运营

## 二，ClawHub技能运营

### 2.1 技能发布流程

```
技能开发完成
    ↓
技能审计（SKILL.md格式检查 + 安全性审查）
    ↓
技能签名（版本标记 + 作者归属）
    ↓
技能发布（`clawhub install` 可用）
    ↓
社区公告（Telegram/Discord同步）
    ↓
用户反馈收集（Issue + 评论）
```

### 2.2 技能发布清单

每次发布前检查：

```
[ ] SKILL.md 格式符合 ClawHub 标准
[ ] 版本号已更新 (SemVer)
[ ] Changelog已填写
[ ] 安全审计已通过
[ ] 依赖关系已声明
[ ] 示例用法已包含
[ ] 测试命令已提供
[ ] 文档链接有效
```

## 三，GitHub社区运营

### 3.1 仓库运营策略

| 策略 | 执行方式 | 频率 |
|------|---------|:----:|
| **README维护** | 项目简介 + 快速开始 + 贡献指南 | 每Release更新 |
| **Issue管理** | 分类标签 + SLA响应 + 自动关闭 | 每日 |
| **PR管理** | Review + 测试 + 合并 | 48h内 |
| **Release发布** | Changelog + 版本标记 + Release Notes | 每里程碑 |
| **社区沟通** | Discussion + Wiki + 公告 | 每周 |

### 3.2 Issue标签体系

| 标签 | 含义 | SLA |
|:----:|------|:----:|
| `bug` | 功能缺陷 | 24h首次响应 |
| `enhancement` | 功能增强 | 72h评估 |
| `question` | 使用疑问 | 24h回复 |
| `documentation` | 文档相关 | 48h更新 |
| `help-wanted` | 寻求贡献 | 持续开放 |
| `good-first-issue` | 新手友好 | 持续开放 |
