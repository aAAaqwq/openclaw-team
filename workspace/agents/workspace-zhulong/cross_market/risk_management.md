## 二、CVaR(风险条件价值)计算系统

### 2.1 CVaR核心定义

```
CVaR(α) = 在 α 置信水平下，超过 VaR 部分的平均损失

常用设置：
  · α = 95% → CVaR(95%)   （日常风控使用）
  · α = 99% → CVaR(99%)   （极端情况评估）
  
CVaR 比 VaR 更严格：
  · VaR只回答"最坏5%的分位点是多少"
  · CVaR回答"最坏5%的平均损失是多少"
  · CVaR 永远 ≥ VaR
```

### 2.2 CVaR计算实现

```python
import numpy as np

def calculate_cvar(returns, alpha=0.95):
    """
    计算CVaR
    
    参数:
        returns: 收益率序列（numpy array）
        alpha: 置信水平，默认95%
    
    返回:
        VaR: 风险价值（正值表示风险）
        CVaR: 条件风险价值
    """
    # 排序收益率
    sorted_returns = np.sort(returns)
    
    # 计算VaR（alpha分位点）
    var_index = int(len(sorted_returns) * (1 - alpha))
    var = -sorted_returns[var_index]
    
    # 计算CVaR（超过VaR部分的均值）
    tail_returns = sorted_returns[:var_index]
    cvar = -np.mean(tail_returns)
    
    return var, cvar
```

### 2.3 CVaR阈值体系

| 阀门等级 | CVaR(95%)占用 | 对应熔断等级 | 动作 |
|---------|---------------|-------------|------|
| 安全区 | < 30% | 🟢 绿灯 | 正常运行 |
| 关注区 | 30-50% | 🟢 绿灯 | 注意风险敞口 |
| 预警区 | 50-75% | 🟡 黄牌 | 减仓50%，暂停新开仓 |
| 危险区 | 75-90% | 🔴 红牌 | 逐步平仓 |
| 极限区 | > 90% | ⚫ 物理熔断 | 一键清仓+锁仓72h |

### 2.4 实时CVaR监控逻辑

```
每5分钟自动执行：
  1. 获取所有持仓策略的实时收益率序列（30天滚动窗口）
  2. 合并计算全局CVaR(95%)
  3. 检查CVaR占用百分比
  4. 如超过当前等级阀值，触发对应熔断
  5. 记录CVaR时间序列（用于后续分析）
  6. 输出当前CVaR占用到监控界面

特别说明：
  · 加密市场（7x24）需额外计算CVaR(99%)应对尾部风险
  · A股市场（非交易时间）CVaR冻结计算，使用最近交易日数据
  · 跨市场组合需考虑相关性变化对CVaR的影响
```

## 三、仓位管理矩阵

### 3.1 仓位权重计算

```
仓位权重 = f(夏普, 回撤, VIX, 相关性, 费率, 流动性) × 目标仓位

详细计算步骤：

Step 1: 计算各维度评分
  夏普评分:
    SR > 3.0 → 10分
    SR 2.0-3.0 → 8分
    SR 1.5-2.0 → 6分
    SR 1.0-1.5 → 4分
    SR < 1.0 → 2分（应减仓）

  回撤评分:
    MDD < 5% → 10分
    MDD 5-10% → 8分
    MDD 10-15% → 6分
    MDD 15-20% → 4分
    MDD > 20% → 2分（应减仓）

  VIX评分:
    VIX < 20 → 10分
    VIX 20-25 → 8分
    VIX 25-30 → 6分
    VIX 30-40 → 4分（黄牌区）
    VIX > 40 → 2分（红牌区）

  相关性评分（策略间相关性均值）:
    ρ < 0.2 → 10分
    ρ 0.2-0.4 → 8分
    ρ 0.4-0.6 → 6分
    ρ 0.6-0.8 → 4分（黄牌区）
    ρ > 0.8 → 2分（危险区）

  流动性评分:
    优秀（大盘蓝筹/主流币）→ 10分
    良好（中等市值）→ 8分
    一般（小市值/冷门币）→ 6分
    差（极小市值）→ 4分
    极差（垃圾币）→ 2分

Step 2: 加权求和
  总评分 = 夏普×0.25 + 回撤×0.25 + VIX×0.20 + 相关性×0.15 + 流动性×0.15

Step 3: 映射仓位
  总评分 9-10分 → 满仓 (90%)
  总评分 7-9分 → 重仓 (60%)
  总评分 5-7分 → 半仓 (30%)
  总评分 3-5分 → 轻仓 (15%)
  总评分 < 3分 → 空仓/清仓 (0%)
```

## 四、安全措施协议

### 4.1 API密钥安全管理

```
📦 密钥存储规范：
  1. 密钥存储在环境变量或加密文件中
  2. 绝对不与策略代码放在同一目录
  3. 每个交易所建一个独立的密钥文件

🔑 权限控制：
  1. 仅开启交易权限（不开提币权限）
  2. 仅允许指定IP地址访问
  3. 设置API密钥的每日/每次交易限额
  4. 限制最大杠杆倍数（默认3x上限）

🔄 密钥轮换：
  1. 每月更换一次API密钥
  2. 任何安全事件后立即更换
  3. 更换密钥后验证新密钥可用

🛡️ 紧急吊销：
  1. 检测到疑似泄露 → 立即吊销全部密钥
  2. 手动吊销方案已准备（一键执行脚本）
  3. 备用密钥存储于离线环境
```

### 4.2 资金安全铁则

```
⚡ 资金隔离原则：
  1. 永远不把全部资金放在1个交易所
  2. 永远不把全部资金放在1个策略
  3. 永远保留至少30%现金/稳定币缓冲
  
💧 流动性原则：
  1. 单策略使用资金不超过该品种每日成交量的10%
  2. 避免在流动性差的时段/市场重仓
  3. 极端行情下优先平流动性好的头寸

🕌 最大损失原则：
  1. 单笔交易最大损失 ≤ 账户资金的1%
  2. 单日最大损失 ≤ 账户资金的3%
  3. 单策略最大损失 ≤ 账户资金的10%
  4. 全局最大损失 ≤ 账户资金的15%（月度重置）
```

### 4.3 交易操作安全

```
审核机制：
  1. 所有新策略上线需要风控独立审核
  2. 任何止损/止盈修改需要日志记录
  3. 大额交易（>账户资金5%）需要双重确认

容错机制：
  1. 下单失败 → 重试3次（间隔100/500/1000ms）
  2. 连续5次失败 → 停止该策略所有操作
  3. 交易所连接中断 → 自动切换到备选交易所
  4. 全所中断 → 停止所有交易+保留现金

日志记录：
  1. 每笔交易记录到独立日志文件
  2. 日志包含：时间、策略、方向、数量、价格、延迟
  3. 每日生成交易汇总日志
  4. 每周归档日志到独立存储
```

## 五、紧急预案

### 5.1 情景应对矩阵

| 情景 | 触发条件 | 应对措施 | 恢复条件 |
|------|---------|---------|---------|
| 单一交易所中断 | 连接超时/API报错 | 切换到备份交易所 | 恢复连接无异常 |
| 多所同步中断 | 2+交易所同时异常 | 暂停所有交易+锁定密钥 | 逐一恢复确认 |
| 市价剧烈波动 | 单品种5分钟内波动>10% | 暂停该品种+加宽滑点容忍 | 波动率回落<5%/h |
| 未知市场事件 | 检测到异常统计信号 | 降仓50%+启动人工审查 | 查明原因+回测验证 |
| 密钥泄露 | 异常交易行为 | 吊销密钥+冻结账户 | 启用新密钥+审计完成 |
| 网络攻击 | 检测到DDoS/异常请求 | 断开公网+使用备用线路 | 安全团队确认 |
