# 三账本体系 v2.0（国际顶级版）

> **文件定位**：烛龙宪法级文档 #2  
> **角色**：回测账本 → 虚拟盘 → 实盘的完整进化路径  
> **核心用途**：确保策略在进入真金白银之前经过三层验证  
> **初始资金**：虚拟盘 ¥100万 · 实盘逐步激活

## 一、三账本体系总览

### 1.1 三账本进化路径

```
┌──────────────────────────────────────────────────────────────────┐
│                    三账本进化路径                                 │
│                                                                   │
│  回测账本            虚拟盘              实盘                    │
│  │                   │                   │                       │
│  │ 练习市场判断      │ 练习交易经验      │ 真实资金              │
│  │ 练习指标把握      │ 策略验证          │ 收益产生              │
│  │ 因子验证          │ 风控测试          │ 复利增长              │
│  │                   │                   │                       │
│  └───→────→──→────→──┴───→────→──→────→──┘                       │
│       通过回测          虚拟盘验证通过                            │
│       夏普>1.5          2周正向收益                               │
│       回撤<20%          10笔以上交易                              │
└──────────────────────────────────────────────────────────────────┘

核心原则：
  1. 不通过回测 → 不进虚拟盘（逻辑门 + 统计门）
  2. 不通过虚拟盘 → 不上实盘（风险门 + 执行门）
  3. 低资金实盘验证通过 → 逐步加仓（组合门）
  4. 任何阶段的策略回撤 > 20% → 退回上一级重新验证
```

### 1.2 三账本核心对比

| 维度 | 回测账本 | 虚拟盘 | 实盘 |
|------|---------|--------|------|
| **目的** | 验证策略逻辑/因子有效性 | 验证执行力/模拟心态 | 产生真实收益 |
| **资金** | 理论资金（无上限） | 虚拟资金100万 | 真实资金 |
| **市场** | 全市场历史数据 | 实时行情（不真成交） | 实时成交 |
| **风险** | 无（纯计算） | 无（不影响资金） | 真实亏损风险 |
| **周期** | 每晚1:00-5:00 | 全时段（同步实盘） | 市场开放时间 |
| **输出** | 回测报告/因子IC | 模拟交易日志 | 收益/归因/风控 |

### 2.3 回测通过标准

```
✅ 通关条件（缺一不可）：
  ① 夏普比率 > 1.5（目标 > 2.0）
  ② 最大回撤 < 20%（目标 < 15%）
  ③ 交易次数 > 200次（统计显著性）
  ④ 压力测试全部通过（08金融/20熔断/22LUNA）
  ⑤ 核心因子IC > 0.02（年化）

⚡ 加速通关条件（满足任三即可提前进虚拟盘）：
  ① 夏普 > 3.0
  ② 最大回撤 < 8%
  ③ 样本外结果与样本内一致
  ④ 与现有策略相关性 < 0.3
  ⑤ 有清晰经济学解释
```

### 2.4 每日回测流水线

```
每晚 01:00 准时启动：

线程A：因子IC更新（每日）
  扫描当前因子库中所有活跃因子的日IC值
  标记IC < 0.02的因子 → 黄牌警告
  标记IC持续下降的因子 → 半衰期检测

线程B：候选因子测试（每日3-5个）
  从10个候选因子中选3-5个做全量回测
  输出因子回测报告
  统计显著（p<0.05）+ IC>0.03 → 入候选池

线程C：策略组合优化（每周）
  遍历所有活跃策略的相关性矩阵
  用风险平价/均值方差计算最优权重
  检查组合CVaR是否超标

线程D：新领域学习 + 因子挖掘
  每周精读1个领域的研究论文/策略
  提取3-5个新因子概念
  安排到下周回测队列
```

## 三、虚拟盘（第二关）

### 3.1 虚拟盘账户映射

| 虚拟盘 | 初始资金 | 映射实盘 | 目的 |
|--------|---------|---------|------|
| **A股虚拟盘** | ¥200,000 | 华泰证券 | 模拟A股T+1/涨跌停规则 |
| **加密虚拟盘** | ¥400,000 | Bitget/币安 | 模拟7×24/杠杆/费率 |
| **Polymarket** | ¥400,000 | Polymarket | 模拟事件驱动/概率交易 |

> **资金分配策略**：20%/40%/40%，符合盈亏平衡分析——加密和预测市场的高风险对应高权重。

### 3.2 虚拟盘运行规则

```
一、基础规则
  1. 回测通不过 → 不进虚拟盘
  
  2. 虚拟盘策略上线时，初始仓位不得超过该账户资金的10%
     满仓需要至少2周正向收益验证 + 12笔交易记录
  
  3. 虚拟盘出现以下情况 → 策略强制下线：
     · 单策略最大回撤 > 20%
     · 连续5笔亏损
     · 策略夏普(30d) < 1.0

  4. 虚拟盘通过标准：
     · 运行满2周
     · 至少10笔交易
     · 净值 > 初始资金（正向收益）
     · 最大回撤 < 回测MDD × 1.2（容忍略高）

二、状态管理
  running → 策略正在虚拟盘中运行
  paused  → 暂停交易，保留持仓
  closed  → 策略下线，平掉所有仓位
  review  → 审核中（出现异常信号）
  promoted → 升级至实盘
```

### 3.3 虚拟盘状态追踪

```
┌──────────────┬──────────────┬─────────────┬────────────────┐
│ 策略名称     │ 当前状态     │ 运行天数     │ 累计收益       │
├──────────────┼──────────────┼─────────────┼────────────────┤
│ [策略A]      │ running      │ 7天         │ +5.2%          │
│ [策略B]      │ running      │ 4天         │ +2.1%          │
│ [策略C]      │ review       │ 2天         │ -3.8%          │
│ [策略D]      │ promoted → E │ 14天        │ +8.6% ✅       │
└──────────────┴──────────────┴─────────────┴────────────────┘

注：promoted为升级入实盘的策略，标记E便于追踪从虚拟盘来的策略
```

### 3.4 回测→虚拟盘升级流程

```
策略通过回测 → 候选池（可同时5个候选）
              │
       进入虚拟盘（初始化仓位10%）
              │
         监控7天（或10笔交易）
              │
     ┌────────┼────────┐
     │        │        │
   正向     中性     负向
     │        │        │
  再观察   优化后    下线/淘汰
   7天     重测
     │
  增仓至30%
     │
  再观察7天
     │
  增仓至60%
     │
  运行2周以上，净值>初始 → 可升实盘
```

## 四、实盘（第三关）

### 4.1 实盘账户映射

| 实盘账户 | 市场 | 当前状态 | 策略并行数 |
|---------|------|---------|-----------|
| **华泰证券** | 🇨🇳 A股 | ⚡ 待激活 | 3-5条（需API对接） |
| **Bitget** | ₿ 加密(主) | ⚡ 待激活 | 3-5条（需API对接） |
| **币安** | ₿ 加密(备) | ⚡ 待激活 | 辅助/备用 |
| **Polymarket** | 🎯 预测市场 | ⚡ 待激活 | 2-3条（概率偏差） |

> 每个实盘账户设**日亏损上限**（总资金1.5%），达到后自动暂停所有策略。

### 4.2 虚拟盘→实盘升级条件

```
☑ 虚拟盘运行满 2 周
☑ 至少 10 笔交易记录
☑ 净值 > 初始资金（正向收益）
☑ 最大回撤 < 回测MDD × 1.2
☑ 与现有实盘策略相关性 < 0.5
☑ 策略有完整的风控止损逻辑
☑ 有对应的回测报告存档
☑ API对接测试通过（模拟下单）
☑ 熔断逻辑部署完毕
☑ 丘总审批通过

✅ 全部通过 → 可以上实盘
```

### 4.3 实盘上线流程

```
策略通过虚拟盘验证
       ↓
第一阶段：微量实盘（仓位 5% 目标资金）
  运行3个完整交易日
  监控：滑点、执行延迟、实际vs预期差异
       ↓
第二阶段：增仓至30%
  运行1-2周
  检查：夏普、回撤、收益是否与虚拟盘一致
       ↓
第三阶段：目标仓位
  根据仓位矩阵评分确定最终仓位
  最大不超过该策略分配上限（总资金20%）
       ↓
持续监控
  每日归因分析
  月度策略审查
  超额回撤 → 降仓/下线
```

### 4.4 实盘风控即时触发

```
⚠️ 单策略单日亏损 > 2%  →  暂停该策略 → 人工/自动审查
⚠️ 总账户单日亏损 > 1.5% →  黄牌预警 → 减仓50%
🚨 总账户单日亏损 > 5%   →  红牌熔断 → 全部新单暂停
🔴 总账户单日亏损 > 8%   →  物理熔断 → 一键清仓+锁仓72h
```

## 七、补充规则

### 7.1 紧急下线与恢复

```
紧急下线触发条件：
  · 策略连续5笔亏损
  · 策略单日亏损 > 2%
  · 策略最大回撤 > 20%
  · 相关交易所出现异常
  · API连接中断

恢复上线条件：
  · 紧急下线原因已查明并修复
  · 重新通过回测验证
  · 回虚拟盘重新跑3天以上
  · 丘总审批
```

### 7.2 10倍增长路线图

```
最终目标：年度10倍增长（+1000%）

前三季度（9个月）：
  · Q1: 建立基础设施 + 策略库 + 稳定月化1-2%
  · Q2: 多市场叠加 + 月化2-3%
  · Q3: 复利加速 + 月化3-5%

第四季度爆发：
  · 杠杆 + 复利双重效应
  · 更高风险策略在可控范围内
  · 全年达到10X

注意：10倍不是线性增长，必须通过复利+高风险策略的叠加。
但风险必须在可控范围内——亏20%需要赚25%才能回本，
亏50%需要赚100%才能回本。
```
