### 2.1 五大策略类型

| # | 策略类型 | 风险等级 | 年化预期 | 频率 | 资金上限 | 难度 |
|---|---------|---------|---------|------|---------|------|
| 1 | **数学套利 (YES+NO<1)** | 🟢 无风险 | 5-15% | 高频 | 30% | ⭐ |
| 2 | **高概率债券** | 🟢 低风险 | 8-20% | 中频 | 40% | ⭐ |
| 3 | **恐慌错杀捡漏** | 🟡 中风险 | 20-50% | 事件驱动 | 15% | ⭐⭐ |

| 5 | **事件驱动交易** | 🟡 中风险 | 15-40% | 事件驱动 | 5% | ⭐⭐⭐ |

### 2.2 详细策略定义

#### 策略1：数学套利 (YES+NO<1)

```
条件：当市场激烈博弈时，YES合约 + NO合约 < 1（例如 0.45 + 0.45 = 0.90）
操作：同时买入 YES 和 NO
目标：价格回归到 >= 1 时平仓
风险：接近零（只输时间不输钱）
收益：每笔 3-10%（50%资金利用率 × 2-5次/周 ≈ 5-15%/月）
最佳时机：重大事件前24小时（选举/CPI/FOMC/体育决赛）
```

#### 策略2：高概率债券策略

```
条件：结果基本确定的合约，概率 > 92%
操作：买入 YES（或 NO）做"穷人的债券"
目标：持有到期获得确定收益
收益：年化 8-20%（取决于锁定期和概率）
筛选标准：
  • 概率 ≥ 93%（最佳）
  • 概率 85-92%（可接受，需检查流动性）
  • 概率 < 85%（放弃，风险回报比不够）
注意事项：7月后非核心市场不再投入新资金
```

#### 策略3：恐慌错杀捡漏

```
条件：突发事件引发恐慌，优质概率被系统性低估
典型场景：
  • 体育赛事爆冷 → 衍生市场过度反应
  • 选举民调剧烈波动 → 特定候选人概率崩盘
  • 突发新闻 → 市场过度解读
操作：买入被错杀概率
退出：回归理性后平仓
风险控制：
  • 恐慌当天不抄底
  • 等2次确认（24小时后+1个独立信息源验证）
  • 止损设在-30%
```

#### 策略4：信息优势套利（核心引擎）

```
这是Polymarket最核心、收益最高的策略。

流程：
① W1-W52每周选定一个领域 → 高强度学习
② 跟踪领域内核心信息源（专家/媒体/数据）
③ 发现概率偏差 > 15%（市场错了）
④ 下单建仓
⑤ 信息落地后获利了结

关键指标：
  • 概率偏差阈值：≥15%
  • 仓位：单一领域 ≤ 信息优势套利总预算的 20%
  • 止损：-25%（判断错了要认）
  • 频率：每周至少1次

52周领域推进表 → 参见《polymarket_52_week_plan.md》
```

#### 策略5：事件驱动交易

```
条件：已知事件即将发生（Fed决议/NFP/CPI/财报/大选）
操作：事件前2-48小时布局，事件后快速平仓
收益来源：市场在事件前的过度博弈
退出规则：
  • 结果公布后立即平仓（无论盈亏）
  • 绝不持有过夜（事件后市场反转概率极高）
  • 设置-15%止损，5分钟内执行
适用事件：
  • FOMC利率决议
  • 非农数据（NFP）发布
  • CPI/PPI数据
  • 公司财报（加密/科技/能源）
  • 选举投票日
  • 重大政策宣布
```

## 六、风控总则

### 6.1 仓位铁律

```
┌──────────────────────────────────────────────┐
│  📋 仓位铁律（硬编码级）                       │
├──────────────────────────────────────────────┤
│  1. 单一市场 ≤ 总Polymarket资金 20%           │
│  2. 信息优势套利 ≤ 总预算 50%                 │
│  3. 数学套利 ≤ 总预算 30%                     │
│  4. 高概率债券 ≤ 总预算 40%                   │
│  5. 恐慌错杀 ≤ 总预算 15%                     │
│  6. 事件驱动 ≤ 总预算 5%                      │
│  7. 单日最大亏损 ≤ 总预算 5% → 熔断        │
│  8. 单一领域止损 ≤ 该领域仓位 30%              │
└──────────────────────────────────────────────┘
```

### 6.2 安全架构

| 层级 | 规则 |
|------|------|
| **L0: 主钱包** | 绝不连接Polymarket，冷存储 |
| **L1: 热钱包(操作层)** | 仅存放操作资金，定期从主钱包补 |
| **L2: 策略隔离** | 每个策略独立子钱包，互不影响 |
| **L3: 熔断机制** | 单日亏损5% → 停止所有交易24小时 |
| **L4: 物理熔断** | 连续3日亏损 → 清仓 → 72小时后复盘 |

### 6.3 止损规则表

| 策略类型 | 止损线 | 触发动作 |
|---------|--------|---------|
| 数学套利 | 持有至到期（不设止损） | — |
| 高概率债券 | 概率跌破80% | 立即平仓 |
| 恐慌错杀 | 浮亏-30% | 强制平仓+复盘 |
| 信息优势套利 | 浮亏-25% | 强制平仓+复盘 |
| 事件驱动 | -15% 或 事件后30分钟 | 无条件平仓 |

## 七、复盘KPI体系

### 7.1 每日KPI

| 指标 | 优秀 | 一般 | 需要改进 |
|------|------|------|---------|
| 交易次数 | ≥3次 | 1-2次 | 0次 |
| 胜率 | ≥65% | 50-65% | <50% |
| 单笔平均收益 | >5% | 2-5% | <2% |
| 单日PnL | >总资金1% | 0-1% | 亏损 |
| 概率偏差命中 | >50% | 30-50% | <30% |

### 7.2 每周KPI

| 指标 | 目标 |
|------|------|
| 新领域学习 | 1个（完成52周计划） |
| 信息源新增 | ≥5个 |
| 领域专家卡 | 更新或新建1张 |
| 总交易次数 | ≥15次 |
| 周总收益 | >总资金3% |
| 最大回撤 | <5% |

### 7.3 每月复盘

```
每月1日做：

1. 月度收益归因（哪个领域/策略赚钱）
2. 概率偏差命中率统计
3. 52周计划进度检查
4. 领域专家卡内容更新
5. 策略权重再平衡
6. 信息源有效性评估（哪些信息源产生过命中）
7. 安全审计（密钥/钱包/API状态）
8. 下月领域重点计划
```
