```javascript
/* 资金费率套利策略模板 - FMZ平台 */
var config = {
    exchange: "Bitget",
    symbol: "BTC/USDT:USDT",  // 永续合约
    spotSymbol: "BTC/USDT",   // 现货
    minFundingRate: 0.01,     // 最低费率阈值(%)
    targetLeverage: 3,        // 目标杠杆
    hedgeRatio: 1.0,          // 对冲比例
    maxPosition: 1.0,         // 最大仓位BTC
    stopLoss: 0.05            // 止损5%
};

function main() {
    var state = "idle";
    while (true) {
        var funding = exchange.GetFundingRate();
        var pos = exchange.GetPosition();
        
        if (funding.rate > config.minFundingRate && state == "idle") {
            // 开仓：做多现货 + 做空永续
            exchange.Spot.Buy(config.hedgeRatio);
            exchange.Futures.Sell(config.hedgeRatio);
            state = "hedged";
        }
        
        if (state == "hedged") {
            // 监控费率是否下降
            if (funding.rate < 0.005) {
                // 平仓
                exchange.Spot.Sell(config.hedgeRatio);
                exchange.Futures.Buy(config.hedgeRatio);
                state = "idle";
            }
            // 监控止损
            if (funding.rate < -0.02) {
                // 紧急平仓
                exchange.Spot.Sell(config.hedgeRatio);
                exchange.Futures.Buy(config.hedgeRatio);
                state = "emergency_close";
            }
        }
        Sleep(60 * 60 * 1000); // 每小时检查一次
    }
}
```

        result = api.funding_time(symbol)
        return result["data"]
    except BitgetAPIException as e:

        return None

# 获取合约持仓
def get_positions(symbol="BTCUSDT"):
    try:
        result = api.position(symbol)
        return result["data"]
    except BitgetAPIException as e:
        Log("API错误:", e.message)
        return None

            order = client.futures_create_order(

                type=ORDER_TYPE_MARKET,
                quantity=quantity

## 四、资金费率套利部署方案

### 4.1 策略原理

**核心逻辑**：
```
永续合约资金费率 = 多头支付给空头或多头收到的费用
当费率 > 0.01%/8h，做空永续 + 做多现货 → 赚费率
当费率 < -0.01%/8h，做多永续 + 做空现货 → 赚费率
```

**收益计算**：
```
日化收益 = 费率 × 3（每天3次结算）× 杠杆倍数
年化收益 = 日化收益 × 365

示例：
日费率均值 0.015%/8h
3x杠杆
日化 = 0.015% × 3 × 3 = 0.135%
年化 = 0.135% × 365 = 49.3%
```

### 4.3 执行参数

| 参数 | 值 | 说明 |
|------|---|------|
| 开仓费率阈值 | >0.01%/8h | 高于此值开仓 |
| 平仓费率阈值 | <0.005%/8h | 低于此值平仓 |
| 最大杠杆 | 3x | 控制风险 |
| 最大仓位 | 总资金的30% | 防止集中风险 |
| 止损费率 | < -0.02%/8h | 紧急平仓 |
| 监控频率 | 1小时 | 每1小时检查一次 |
| 优先交易所 | Bitget | 主力账户所在 |
| 辅助交易所 | 币安 | 第二选择 |

### 4.4 风险监控

```
资金费率套利的特有风险：
1. 基差风险 — 现货与合约价差扩大
   → 监控基差偏离度，>2%时预警

2. 杠杆清算风险 — 极端行情导致合约爆仓
   → 使用低杠杆(1-3x)，预留充足的保证金

3. 交易所风险 — 提币暂停/维护
   → 分散到至少2个交易所，预留回撤空间

4. 费率反转风险 — 正费率突转为负费率
   → 设置紧急平仓条件

5. 资金占用风险 — 套利资金被长期锁定
   → 设置最长持仓时间（建议<30天）
```

### 6.1 开仓前检查

```
□ 信号置信度 > 最低阈值（趋势策略>60%，套利>70%）
□ 当前CVaR占用 < 75%
□ 单一标的敞口 < 总资金20%

□ 策略间相关系数 < 0.7
□ 流动性检查通过（滑点容忍范围内）
□ 交易所运行正常（无维护/暂停）
□ API连接正常
□ 保证金充足 > 保证金要求的150%
□ 杠杆倍数 < 最大限制（3x）
```

### 6.2 每日执行流程

```
🌅 06:00  晨间检查
  - 检查持仓状态
  - 检查资金费率
  - 更新风控参数

⚡ 08:00  策略信号计算
  - 多因子合成
  - 信号生成
  - 仓位分配

📈 09:00-17:00  执行时段
  - 按信号执行交易
  - 实时风控监控
  - 记录每笔交易

🌙 17:00-21:00  收盘总结
  - 当日收益归因
  - 风控审计
  - 次日计划

🛡️ 全天候  风控守护
  - 资金费率监控
  - 异常波动告警
  - 三色牌熔断
```

### 6.3 熔断机制

```
🟢 绿灯：正常交易
  条件：CVaR < 50%，日亏损 < 2%
  操作：正常执行

🟡 黄牌预警（缩仓50%）：
  条件：单日亏损 > 2%
       或 CVaR占用50-75%
  操作：新交易需双重确认，减仓至半仓

🔴 红牌熔断（一键清仓）：
  条件：单日亏损 > 5%
       或 CVaR占用 > 75%
       或 交易所出现异常
  操作：暂停所有新交易，逐步平仓

⚫ 物理熔断（拔网线）：
  条件：单日亏损 > 8%
       或 CVaR占用 > 90%
       或 多所同步异常
  操作：关闭所有API连接，强制全部平仓
```
