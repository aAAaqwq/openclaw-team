## 二、第1层：智能定投引擎（底仓层）

### 2.1 设计原理

**核心哲学**：不要预测市场底部，用恐惧指数做定投加速器

**定投三原则**：
1. **雷打不动**：每周固定时间买入固定金额（U本位）
2. **恐惧加速**：市场越恐慌，买入越多
3. **贪婪减速**：市场越狂热，买入越少

### 2.2 智能定投公式

```
每周定投金额 = 基础定投 × 恐惧因子

基础定投：总资金的1%/周（年化52%资金投入）
恐惧因子：基于恐惧贪婪指数动态调整

恐惧因子计算：
  恐惧贪婪指数值 = FGI (0-100)
  恐惧因子 = 1.5 - (FGI / 100)
  
  当FGI = 10（极度恐惧）：恐惧因子 = 1.4 → 投入1.4倍
  当FGI = 50（中性）：恐惧因子 = 1.0 → 投入基础量
  当FGI = 90（极度贪婪）：恐惧因子 = 0.6 → 投入0.6倍
```

### 2.3 执行参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 基础定投比例 | 总资金1%/周 | 年化52%投入 |
| 最大周投入 | 基础定投 × 2 | 极端恐惧时 |
| 最小周投入 | 基础定投 × 0.3 | 极端贪婪时 |
| 定投标的 | BTC 60% / ETH 40% | 核心持仓 |
| 定投频率 | 每周一10:00 UTC | 固定时点 |
| 恐惧贪婪数据源 | Alternative.me | 每日更新 |
| 限价单偏离 | 低于市价0.5% | 挂限价单降低滑点 |

### 2.4 阈值触发加仓

除了定期定投外，设置以下条件作为**额外加仓触发**：

| 触发条件 | 加仓比例 | 说明 |
|---------|---------|------|
| BTC周跌幅 > 15% | 基础定投 × 3 | 恐慌性下跌加仓 |
| BTC月跌幅 > 30% | 基础定投 × 5 | 深度熊市重仓 |
| BTC低于200周均线 | 基础定投 × 4 | 历史级底部区域 |
| FGI < 10 | 基础定投 × 2 | 极度恐慌 |
| 交易所储备金异常 | 暂停买入 | 安全优先 |

### 2.5 智能定投流程

```
每周一 09:00 UTC：
  1. 获取FGI恐惧贪婪指数
  2. 计算本周基础定投金额
  3. 计算恐惧因子 → 实际定投金额
  4. 检查阈值触发条件
  5. 如果有触发 → 叠加额外加仓
  6. 总投入 = 定投金额 + 触发加仓
  7. 使用限价单（低于市价0.5%）买入
  8. 记录买入价格/数量/时间

每月底：
  1. 汇总当月定投记录
  2. 计算平均买入价格
  3. 对比一次性购买vs定投效果
  4. 调整定投参数（如需）
```

## 三、第2层：资金费率套利引擎 ★★★ 核心层

### 3.1 核心逻辑

**目标**：在囤币底仓上叠加Delta中性套利，赚取永续合约的资金费率，**增加币数量**

**工作原理**：
```
持有1 BTC现货 + 做空1 BTC永续合约 = Delta中性
当资金费率 > 0时：空头收到资金费
每8小时结算一次费率
年化增币率 = 平均日费率 × 3次结算 × 365天

示例：
  日费率均值：0.015%/8h
  年化增币：0.015% × 3 × 365 = 16.4%
  如果3x杠杆：年化增币 = 16.4% × 3 = 49.2%
```

### 3.2 执行参数

| 参数 | 值 | 说明 |
|------|---|------|
| 最小开仓费率 | > 0.01%/8h | 低于此不划算 |
| 目标杠杆 | 1x-3x | 1x最安全，3x提高收益 |
| 最大敞口 | 总持仓BTC的100% | 不对冲超过持有量 |
| 平仓费率 | < 0.005%/8h | 利润变薄时平仓 |
| 紧急止损费率 | < -0.02%/8h | 费率反转紧急处理 |
| 检查频率 | 每1小时 | 持续监控 |
| 交易所 | Bitget(主力)+币安(备) | 分散风险 |
| 账户类型 | U本位合约 | 便于计算 |

### 3.3 执行流程

```
Step 1：费率监控（每小时）
  for each 交易所:
    get_funding_rate(symbol)
    if rate > 0.01%:
      → 进入开仓决策

Step 2：开仓决策
  检查条件：
  □ 费率 > 0.01%/8h？
  □ 账户余额充足？
  □ 未超过最大敞口？
  □ 交易所状态正常？
  
  如果全部通过 → 执行开仓

Step 3：开仓执行
  1. 确定对冲数量 = min(现货持仓量, 最大敞口)
  2. 做空永续合约 (对冲数量)
  3. 记录开仓信息：时间/数量/费率/价格

Step 4：持仓监控（每小时）
  1. 检查当前费率
  2. 如果费率 < 0.005% → 准备平仓
  3. 如果费率 < -0.02% → 紧急平仓
  4. 检查基差偏离
  5. 检查杠杆率

Step 5：平仓执行
  1. 平仓永续合约空头
  2. 计算本次收益：(累计收到的费率 - 交易成本)
  3. 记录平仓信息
  4. 等待下一次开仓机会
```

### 3.4 收益预估

| 场景 | 日费率均值 | 年化增币(1x) | 年化增币(3x) | 概率 |
|------|-----------|-------------|-------------|------|
| 牛市高峰 | 0.05%/8h | 54.75% | 164.25% | 10% |
| 正常牛市 | 0.02%/8h | 21.90% | 65.70% | 30% |
| 震荡市 | 0.015%/8h | 16.43% | 49.28% | 40% |
| 熊市/低波动 | 0.005%/8h | 5.48% | 16.43% | 20% |

**保守预估年化增币率（持续运行3x杠杆）**：**25-50%**

### 3.5 风险管理

```
资金费率套利的特有风险：

1. 基差风险
   当现货价格与永续价格出现巨大偏离时的损失
   缓解：基差 > 2% 时暂停操作

2. 杠杆清算风险
   极端行情导致空头合约被清算
   缓解：使用低杠杆(1-3x)，预留300%保证金

3. 费率反转风险
   正费率突变为负费率，空头开始支付费用
   缓解：设置紧急平仓条件，费率 < 0 立即平仓

4. 交易所风险
   交易所暂停提币/维护/故障
   缓解：分散交易所，预留备用方案

5. 资金效率风险
   资金被套利占用，错失更好的收益机会
   缓解：设置资金使用上限（总资金50%）
```

## 四、第3层：动态仓位管理

### 4.1 仓位管理原则

```
总资金分配：
  ┌──────────────────────────────────┐
  │  底仓（长期持有）: 40-60%        │ ← 智能定投积累不卖出
  │  套利仓位（Delta中性）: 20-30%   │ ← 费率套利用
  │  灵活仓位（趋势交易）: 10-20%    │ ← 趋势择时增强
  │  现金储备（防御）: 10-20%        │ ← 现金/U本位
  └──────────────────────────────────┘
```

### 4.2 波动率自适应调整

| 波动率环境 | 底仓 | 套利仓 | 灵活仓 | 现金 |
|-----------|------|--------|--------|------|
| 低波动（BTC 30d波动<30%） | 50% | 30% | 10% | 10% |
| 中等波动（30-60%） | 45% | 25% | 15% | 15% |
| 高波动（60-100%） | 40% | 20% | 15% | 25% |
| 极端波动（>100%） | 30% | 10% | 10% | 50% |

### 4.3 仓位再平衡规则

```
再平衡触发条件（任一满足触发）：
1. 时间触发：每月1日自动再平衡
2. 偏离触发：某个仓位比例偏离目标 > 10%
3. 波动触发：BTC 30天波动率跨越阈值

再平衡执行：
1. 计算当前各仓位比例
2. 计算目标比例（基于当前波动率环境）
3. 优先通过套利仓调整（不影响底仓）
4. 如果必须调整底仓 → 通过定投节奏微调（不要卖出底仓）
5. 记录再平衡操作
```

**两套择时系统**：

```
趋势择时系统A：长期趋势过滤器
  使用：BTC 200日均线 + 200周均线（STH-MVRV）
  目标：判断大周期牛熊
  信号：
    价格 > 200日均线 → 牛市模式 → 正常/加码定投
    价格 < 200日均线 → 熊市模式 → 减量/只定投

趋势择时系统B：中期趋势增强
  使用：SuperTrend (ATR倍数=3) + EMA50/200交叉
  目标：在牛市中的回调加仓，在熊市反弹中减仓保护
  信号：
    SuperTrend转为多头 + EMA50>EMA200 → 趋势增强
    SuperTrend转为空头 → 减灵活仓位
```

## 六、第5层：物理熔断保护层

### 6.1 熔断级别

```
🟢 正常状态
条件：
  - BTC持仓低于总资产的60%
  - 费率套利正常运行
  - 日内回撤 < 3%
操作：
  - 正常执行所有层级

🟡 预警状态
条件（任一）：
  - BTC价格跌破 200日均线 - 20%
  - 全市场恐慌指数 < 10
  - 交易所出现异常（暂停提币）
  - 单日回撤 > 5%
操作：
  - 暂停趋势择时层（关闭灵活仓）
  - 费率套利降低至1x
  - 定投仅执行基础量

🔴 熔断状态
条件（任一）：
  - BTC价格崩盘 > 30%（单日）
  - 多所同步异常
  - 检测到系统性风险
  - 单日回撤 > 10%
操作：
  - 立即平仓所有费率套利仓位
  - 关闭所有自动化交易
  - 保留底仓不动（不卖币）
  - 等待72小时评估
```

### 6.2 底仓保护原则

```
底仓保护：
  ✅ 底仓在任何情况下都不卖出
  ✅ 底仓始终存放在个人钱包（非交易所）
  ✅ 只有在物理熔断时才考虑转移
  
套利仓保护：
  ✅ 套利仓在熔断时立即平仓
  ✅ 熔断后资金转回现货/USDT
  
定投保护：
  ✅ 只有在交易所异常时才暂停定投
  ✅ 熔断解除后自动恢复定投
```
