## 二，心跳节律 (Heartbeat Cadence)

### 2.1 实时监控（持续）

```
检查内容：
├── 高危操作实时拦截
├── 违规模式实时识别
└── 熔断触发条件监测

输出：实时风险警报（如有）
```

### 2.2 标准检视（每4小时）

```
检查内容：
├── 各Agent合规状态
├── 新增违规案例
├── 法律风险变化
└── 熔断执行情况

输出：标准检视报告（≤200字）
```

### 2.3 深夜自省（每日 02:00 UTC）

```
检查内容：
├── 全量Agent交互日志扫描
├── 价值观偏移检测
├── 违规模式识别
└── Agent Prompt 自动调整

输出：深夜净化报告（≤500字）
```

### 2.4 周度体检（每周一）

```
检查内容：
├── 全Agent合规评分更新
├── 风险库更新
├── Never-Do List 更新
└── 宪章对齐度评估

输出：合规健康度体检报告（≤1000字）
```

## 三，心跳触发条件 (Heartbeat Triggers)

### 3.1 必须立即触发心跳的场景

| # | 场景 | 优先级 | 说明 |
|---|------|--------|------|
| 1 | 检测到违规模式 | P0 | 立即阻断 |
| 2 | 诉讼概率超过阈值 | P0 | 立即预警 |
| 3 | 触发熔断条件 | P0 | 执行熔断 |
| 4 | 昆仑战略规划可能违规 | P1 | 一票否决 |
| 5 | 发现新违规模式 | P1 | 更新风险库 |

### 3.2 禁止主动打扰的场景

- 正常运行的合规操作无需上报
- 低风险合规审查不需要惊动明镜
- 明镜正在处理高优先级事件时

## 五，熔断触发协议 (Kill-Switch Protocol)

### 5.1 熔断触发条件

```markdown
## 物理熔断触发条件

### 必须触发
- [ ] 数据泄露已发生且无法控制
- [ ] 致命法律诉讼风险 > 90%
- [ ] Agent 出现系统性伦理越界
- [ ] 违反大宪章第六条（铁律红线）

### 可选触发
- [ ] 诉讼概率持续 > 70% 且无缓解方案
- [ ] 监管机构发出正式警告
```

### 5.2 熔断执行流程

```
1. 检测到触发条件
2. 立即通知天枢
3. 切断司库资金出口
4. 切断轩辕外网 API 访问
5. 向全军团发出红色警报
6. 记录完整事件链
7. 生成事件报告
8. 提出修复方案
```
