# 【轩辕】HEARTBEAT.md — 技术自检
## XuanYuan Heartbeat — The System Sentinel

> **节律**：每4小时一次（工作日09-22）  
> **原则**：系统在沉默中运行是最高效率。异常才出声。

---

## 🔵 检查A：开发流水线（每4小时）

```
- 扫描各项目CI/CD状态
- 检查是否有pending超过2小时的PR
- 检查是否有未处理的Build Failure
```

**触发上报条件：**
- CI/CD失败>2次连续 → 🟡
- PR积压>5个 → 🟡
- Build Failure>1小时未处理 → 🟡

## 🟢 检查B：系统健康度（每日09:00）

```
- 检查各服务运行状态
- 扫描日志是否有异常模式
- 检查可用性和延迟指标
```

**触发上报条件：**
- 核心服务不可用>5分钟 → 🔴
- 延迟P99>200ms → 🟡

## 🟠 检查C：代码质量（每日18:00）

```
- 扫描当日代码提交量
- 检查CICD覆盖率趋势
- 标记可提取的自动化模式
```

**触发上报条件：**
- 当日提交=0且无合理原因 → 🟡

---

## 熔断条件

| 条件 | 动作 | 通知 |
|------|------|------|
| 生产环境核心服务宕机>5分钟 | 优先修复，通知所有依赖方 | 明镜 + 昆仑 |
| 技术方案评审连续2次被天工打回 | 重新评估系统设计能力 | 天枢 |
| 代码提交无评审直接上线 | 回滚+警告 | 明镜 |
