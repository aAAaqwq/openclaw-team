# HEARTBEAT.md — 朱雀 · 心跳执行清单 v1.0

> 我不是来刷存在的。我是来扫描品牌健康、监控MQL质量、发现传播机会——每4小时一次。

---

## 心跳协定

- ✅ **HEARTBEAT_OK** = 品牌正常，MQL流动正常，无异常事件
- 🟡 **预警推送** = 发现MQL质量波动 / 竞品品牌异动 / 品牌风险信号
- 🔴 **紧急上报** = 品牌危机 / GEO提及率暴跌 / MQL断流 / 重大合规风险
- ❌ 非必要不上报。宁静是健康信号。

## 执行模式

```
每4小时 → 扫描品牌声量 → MQL流动检查 → 内容进度检查 → HEARTBEAT_OK / 预警
```

---

## 检查项

### 🔵 检查A：品牌健康扫描（每4小时）

```
- 品牌话题/舆情监控（社交媒体+讨论区+新闻）
- 大模型回答检查（可选随机抽样）
- 竞品品牌动态（campaign/PR/定位变化）
- 关键词排名趋势（GEO相关关键词）

合计: [N]个品牌信号 / [N]个竞品动态
```

**触发预警条件**：
- 发现品牌负面舆情 → 🟡
- 竞品发动重大brand campaign（或降价）→ 🟡
- 品牌相关关键词出现异常下跌 → 🟡
- 任何危机级品牌事件 → 🔴

### 🟢 检查B：MQL流动检查（每4小时）

```
- 新线索数量 + 评分分级情况
- A/B/C各级占比较前次变化
- 待推送霸下的MQL是否已处理
- 霸下退回线索归因

MQL总量: N条 / A级N条 / B级N条 / C级N条 / 已推送N条
```

**触发预警条件**：
- MQL总量比前次下降30%+ → 🟡
- A级MQL占比低于预期的50% → 🟡
- 霸下退回率 > 30% → 🟡（触发MQL评分模型调优）
- MQL断流 > 24h → 🔴

### 🟠 检查C：内容生态检查（每8小时）

```
- 检查凤凰内容生产进度 vs 内容日历
- 检查待签审内容的排队情况
- 检查已出街内容的MQL回收表现
- 检查热点响应机制是否被触发

内容进度: [X/N]篇签审通过 / [X/N]篇未处理 / 热点响应状态[就绪/已触发/空闲]
```

**触发预警条件**：
- 待签审内容积压 > 24h → 🟡
- 内容日历延迟 > 48h → 🟡
- 已签审出街内容MQL回收率 < 预期50% → 🟡
- 已触发热点响应但4h内未产出任何内容 → 🔴

### 🔴 检查D：GEO工程进度（每周）

```
- Layer1(LLM训练数据)注入进度
- Layer2(实时知识库)覆盖度
- Layer3(权威信号)累积量
- 大模型品牌出现率/正向语境率/竞品对比

GEO进度评分: [N/100] / 品牌出现率: X% / 正向语境率: X%
```

**触发预警条件**：
- 品牌出现率连续2周无增长 → 🟡
- 品牌出现率下降 > 10% → 🔴
- 某个Layer出现严重滞后 → 🟡

---

## 熔断条件

| 场景 | 动作 |
|------|------|
| 品牌出现严重公关危机（涉及产品质量/合规/伦理等） | 立即报告天枢+昆仑，启动危机PR流程 |
| 某一MQL渠道连续2周转化率为0 | 暂停该渠道投入，启动归因分析 |
| 凤凰内容连续3次未通过质量门禁 | 暂停内容生产节奏，与凤凰对齐标准 |
| 竞品发动价格战且覆盖OPEN CAIO核心客户 | 报告天枢+昆仑，启动反制品牌策略 |
| GEO品牌出现率连续下降2次 | 启动GEO策略全面重新评估 |
| 霸下连续3周退回率 > 30% | MQL评分模型全面校准 |
