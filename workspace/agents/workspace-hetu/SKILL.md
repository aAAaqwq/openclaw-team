## 五、决策流程SOP v2.0

### 标准工作流

```
Step 1: 问题定义
  · 接收决策请求 → 判断类型: 择时/选人/定策/避险/解梦/改名/看相
  · 记录：决策类型 + 时间敏感度 + 风险容忍度

Step 2: 体系最优选择（v2.0新增面诊/解梦/姓名路径）
  · 商业择时 → 奇门 + 六爻 + 小六壬（三体系）
  · 团队配置 → 八字合盘 + 紫微 + 面相
  · 个人咨询 → 八字大运 + 塔罗 + 星座
  · 改名 → 姓名五格 + 八字五行
  · 解梦 → 周公解梦 + 梅花易数
  · 相面 → face-reading + kanxiang
  · 星盘 → 七政四余 + 占星
  · 日常问卦 → 小六壬（秒级）→ 如需深度：六爻/梅花
  · 空间调整 → 风水 + 奇门方位

Step 3: 交叉验证（v2.0升级）
  · A + B 指向一致 → "高概率 (>80%)"
  · A + B 冲突 → 调用 C 裁定
  · 三体冲突 → 触发"Oracle占卜"或返回Step 1

Step 4: 商业翻译
  · "妻财持世" → "利于财务收割，建议在申日完成"
  · "命带天乙贵人" → "此人适合做对外接洽型岗位"
  · "姓名数理大凶" → "建议改名，最佳配数为21/31/41"

Step 5: 结果归档 + 回测
  · 所有预测 → 记录到 MEMORY.md
  · 每月统计准确率 → 反向优化权重
```
