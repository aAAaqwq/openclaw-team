# 实体优化 (Entity Optimizer)

> 对标GEO权威框架 (arXiv:2311.09735) 的实体级优化策略。构建品牌/产品知识图谱，确保在大模型生成回答中被优先引用的**实体信号体系**。

## 核心方法

### 1. 品牌知识图谱构建

实体优化的第一步是将品牌/产品信息转化为结构化的**知识图谱三元组**：

```
实体 ──属性──> 属性值（如：产品 ─价格──> ¥9.99）
实体 ──关系──> 其他实体（如：产品 ─属于──> 产品线）
实体 ──类别──> 类型（如：产品 ─是──> SaaS工具）
```

**构建步骤：**
1. **实体清单**：梳理所有核心实体（公司/产品/人物/服务/功能）
2. **属性定义**：每个实体的关键属性（名称/描述/价格/版本/创始人）
3. **关系映射**：实体间关系图（隶属/引用/竞品/互补）
4. **类别体系**：实体类型分类（Organization ↔ Product ↔ Person ↔ Service）

### 2. 实体信号矩阵

每个实体需要向大模型传递四类信号：

| 信号维度 | 定义 | 实施方法 | 权重参考 |
|---------|------|---------|---------|
| **名称** (Name) | 标准名称+别名 | 统一全平台命名 + Schema标记 | 高 |
| **类别** (Category) | 实体所属类型 | Schema.org类型标注 (Organization/Product) | 高 |
| **属性** (Attribute) | 关键特征 | 结构化属性描述 + JSON-LD | 中 |
| **关系** (Relationship) | 关联的其他实体 | 实体间引用/链接/上下级关系 | 中 |

### 3. 实体引用覆盖率计算

衡量品牌/产品在大模型语料中被引用的广度：

```
引用覆盖率 = 已覆盖的实体属性数 ÷ 总实体属性数 × 100%

计算公式示例：
- 产品有 10 个关键属性（名称、价格、版本、功能描述、支持平台……）
- 外部可信信源覆盖了其中 6 个
- 覆盖率 = 60%
```

**优化目标：**
- **90%+**：品牌实体信号极强，大模型回答高度一致
- **70-89%**：需要补充2-3个薄弱属性
- **<70%**：系统性实体标记缺失，需从Structured Data + 权威引用同时入手

### 4. 与Schema标记的协同策略

实体优化与Schema标记是**互补关系**：

| 策略 | Schema标记 | 实体优化 |
|------|-----------|---------|
| 作用域 | 页面级 → JSON-LD | 全网级 → 引用验证 |
| 目标 | 搜索引擎理解页面内容 | 大模型将实体纳入知识库 |
| 手段 | 结构化数据标记 | 外部权威信源 + 属性一致性 |
| 重叠区 | `名称/类别/属性` 两端都需要 | |

**推荐工作流：**
```
entity-optimizer（构建实体图谱）
    → schema-markup-generator（生成JSON-LD标记）
    → geo-content-optimizer（内容级优化）
    → geo-optimization-pro（全面审核）
```

## 关键检查清单

### 实体清单检查
- [ ] 公司实体：全称/简称/曾用名/官方名称统一
- [ ] 产品实体：产品名/版本号/别名
- [ ] 人物实体：创始人/CEO/核心团队成员
- [ ] 服务实体：服务名称/覆盖范围
- [ ] 功能实体：核心功能/子功能

### 属性检查
- [ ] 每个实体至少覆盖5个核心属性
- [ ] 属性值与官网/权威信源一致
- [ ] 属性值在主流大模型（ChatGPT/Claude/Perplexity）中一致出现
- [ ] 价格/版本号等时效信息标注了最后更新时间

### 覆盖率检查
- [ ] 核心实体引用覆盖率 ≥ 90%
- [ ] 次要实体引用覆盖率 ≥ 70%
- [ ] 实体属性至少80%在外部权威信源中有佐证
