# 线索自动化 (Lead Gen Automation)

> 全渠道自动获取并筛选高质量线索的系统：从**多源发现→线索评分→自动跟进序列→CRM集成**，实现"设置即交付"的自动化获客管道。

## 核心方法

### 1. 多源线索获取

| 来源渠道 | 获取方式 | 线索质量 | 单线索成本 | 最适合行业 |
|---------|---------|---------|-----------|-----------|
| **社交媒体** | LinkedIn/X/Twitter精准搜索+自动触达 | ⭐⭐⭐⭐ | 低 | B2B SaaS/咨询 |
| **搜索引擎** | Google/Bing SEO关键词排名+Landing Page | ⭐⭐⭐ | 中 | 高搜索量行业 |
| **冷邮箱** | 邮件验证+个性化模板+自动发送 | ⭐⭐ | 极低 | 大规模触达 |
| **内容营销** | Leadoffer（白皮书/模板/检查清单） | ⭐⭐⭐⭐⭐ | 高（一次性） | 深度决策行业 |
| **社区/论坛** | Reddit/HN/Stack Overflow/Quora自动回复 | ⭐⭐⭐ | 低 | 开发者/技术工具 |
| **活动/Webinar** | Eventbrite+Zoom注册用户列表 | ⭐⭐⭐⭐ | 中 | 企业客户 |
| **推荐/Referral** | 已有客户自动推荐引擎 | ⭐⭐⭐⭐⭐ | 极低 | 高NPS产品 |
| **Apify/Crawl** | 网站+目录+行业列表爬取 | ⭐⭐ | 极低 | 批量初始获客 |

### 2. 线索评分模型

#### BANT模型（经典B2B）

```
B — 预算 (Budget)：8K+/年 → 3pt   5K-8K → 2pt    <5K → 1pt
A — 权限 (Authority)：最终决策者 → 3pt   推荐者 → 2pt   使用者 → 1pt
N — 需求 (Need)：急需解决痛点 → 3pt   有改进意愿 → 2pt   无立即需求 → 1pt
T — 时间 (Timeline)：本月内 → 3pt   3个月内 → 2pt   6+个月 → 1pt

总分 = B + A + N + T（最高12pt）
10-12pt → 热线索 🔥
7-9pt   → 温线索
4-6pt   → 冷线索
< 4pt   → 不在目标客户画像中
```

#### GPCT模型（HubSpot/Chorus改进版）

```
G — 目标 (Goal)：客户的业务目标是什么？
P — 计划 (Plan)：客户打算怎么实现？
C — 挑战 (Challenge)：什么阻碍了客户实现目标？
T — 时间 (Timeline)：以上三个维度的时间表
```

#### CHAMP模型（销售团队常用）

```
C — 挑战 (Challenges)：客户的最大痛点
H — 权限 (Authority)：对接人的决策能力
A — 预算 (Money)：客户预算是否匹配
P — 优先级 (Prioritization)：采购在本公司的优先级
```

### 3. 自动跟进序列设计

**跟进节奏模板（默认5步序列）：**

| 步序 | 触达方式 | 时机 | 内容 | CTA |
|------|---------|------|------|-----|
| Step 1 | 邮件/私信 | D0 | 简短个性化开场 | 约15min电话 |
| Step 2 | 邮件 | D3 | 价值文档/案例分享 | 是否适合？ |
| Step 3 | LinkedIn/X | D7 | 点赞/评论+转发内容 | 建立关系 |
| Step 4 | 邮件 | D10 | 压力+紧迫感（限时/名额） | 最后一次机会 |
| Step 5 | 电话/语音 | D14 | 直接沟通 | 约时间 |

**自动化规则：**
- 线索打开邮件/点击链接 → 提升优先级 + 缩短下一步间隔
- 线索回复"不感兴趣" → 移入培育列表（Nurture，冷却30天）
- 线索回复预约需求 → 立即分配给销售团队
- 全部5步无人回应 → 移入冷线索池（90天后重新激活）

### 4. CRM集成策略

| CRM | 集成要点 | 同步字段 |
|-----|---------|---------|
| **HubSpot** | 双向同步+工作流自动化 | 线索来源/评分/跟进阶段/标签 |
| **Salesforce** | REST API事件驱动同步 | Lead对象+Contact对象+Task |
| **Pipedrive** | Webhook实时同步 | Deal阶段+活动记录 |
| **Notion/Airtable** | 轻量级自建 | 线索状态/备注/负责人 |

**触发自动化的关键事件：**
```
线索评分 > 7（BANT）
  → 自动分配最匹配的销售
  → 创建1st触达任务
  → 发送"新线索通知"Slack消息

表单提交（Demo/免费试用/咨询）
  → 自动发送欢迎序列Step 1
  → 更新CRM线索状态
  → 创建个性化跟进模板

邮件打开率 < 30%（上一批次）
  → 暂停该批次
  → 触发标题/主题内容自动重写
```

## 线索自动化泳道图

```
获取 → 过滤/验证 → 评分 → 序列 → 跟进 ← CRM同步
                                    ↓
                             冷线索池  ← 热线索(sales)
                              (90天)      ↓
                                         成交/流失数据反馈
```
