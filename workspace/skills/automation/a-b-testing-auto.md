# A/B测试自动化 (A/B Testing Automation)

> 全自动化的增长实验框架：从假设提出→实验设计→部署运行→数据监控→统计决策，**全程无需人工干预**。

## 核心方法

### 1. 实验设计模板

每次A/B实验必须包含以下结构：

```
HYPOTHESIS:
  我们相信 [这个变化] 对于 [用户群] 会 [改变行为],
  由 [指标] 验证。
  置信度: [高/中/低]
  背景: [为什么我们觉得这个变化有效]

VARIANTS:
  Control: [当前体验 / Group A]
  Treatment: [新体验 / Group B]  （可扩展A/B/n）

METRICS:
  主指标: [metric_primary] — MDE: [最小可检测效应值]
  辅指标: [metric_secondary1], [metric_secondary2]
  护栏指标: [guardrail_metric]

SAMPLE SIZE:
  每组需要 [N] 个用户 | 预计运行 [D] 天

DECISION RULES:
  赢 (Win): p-value < 0.05, 效应方向为正
  输 (Loss): p-value < 0.05, 效应方向为负
  平 (Flat): 达到样本量后 p-value ≥ 0.05
```

### 2. 统计显著性计算

#### 频率主义方法 (Frequentist)

```
H₀: 对照组和实验组无显著差异
H₁: 对照组和实验组存在显著差异

p-value 阈值: 0.05（默认）
当 p < 0.05 → 拒绝 H₀ → 差异统计显著

Z-test（比例类指标）:
  Z = (p₁ - p₂) / √(p̂(1-p̂)(1/n₁ + 1/n₂))

T-test（均值类指标）:
  t = (x̄₁ - x̄₂) / s_p √(1/n₁ + 1/n₂)
```

#### 贝叶斯方法 (Bayesian)

```
P(实验组优于对照组 | 数据) > 95% → 宣布胜者

优势：
- 支持持续监控（无多次比较惩罚）
- 直接输出"胜率"概率（vs p-value反证）
- 样本不足时可输出"证据不足"，不会误判
```

### 3. 自动化运行流程

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│ 1. 假设   │────>│ 2. 设计   │────>│ 3. 部署   │────>│ 4. 监控   │────>│ 5. 决策   │
│ 生成     │     │ 实验方案  │     │ 流量分配  │     │ 数据采集  │     │ 自动    │
│ (AI驱动)  │     │ (自动)   │     │ (自动)   │     │ (实时)   │     │ 触发    │
└──────────┘     └──────────┘     └──────────┘     └──────────┘     └──────────┘
```

**各阶段自动化程度：**

| 阶段 | 自动化方式 | 人工输入 |
|------|-----------|---------|
| 假设生成 | AI分析用户行为数据+流失路径，生成假设列表 | 确认/修改假设 |
| 方案设计 | 依据模板自动填写ICE评分+样本量计算 | 审核 |
| 部署 | 与A/B测试平台API集成，自动分流 | 无需干预 |
| 流量分配 | 50/50 自动均衡 | 设置最低样本量 |
| 数据监控 | 每小时自动p-value+护栏检查 | 设置预警阈值 |
| 决策触发 | 满足停止条件时自动宣布胜者/跑者/拉平 | 最终确认 |

### 4. 多变量测试 vs A/B/n 测试策略

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| 改变单一元素（按钮颜色/文案） | A/B测试 | 最简单、统计效力最高 |
| 改变多个独立元素 | 多变量测试 (MVT) | 并行测试组合效果 |
| 多个变体对比最佳方案 | A/B/n测试 | 一次比较多个候选方案 |
| 测试页面整体改版 | A/B测试（全量变体） | 保持统计效力 |

**决策规则：**
- MVT需要显著更多流量（变体数×A/B样本量）
- 流量<10K/天 → 只用A/B测试
- 流量>100K/天 → 可用MVT

## 实验运行检查清单

- [ ] 假设ICE评分 >= 5（否则搁置）
- [ ] 样本量计算：至少90%统计效力
- [ ] 主指标定义清晰，可量化
- [ ] 护栏指标设置完成（无负面效应）
- [ ] 流量分配确认（通常50/50）
- [ ] 运行时长：至少覆盖1个完整业务周期（通常7天）
- [ ] 无提前窥探停止规则（除非Bayesian方法）
