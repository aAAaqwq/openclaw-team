# 伙伴验证 (Bounty Partner Verification)

> 增长悬赏体系中，对**交付物验收+伙伴尽职调查+KYC/KYB核查+争议解决**的标准化流程。确保每一笔悬赏奖金都对应真实有效的商业价值。

## 核心方法

### 1. 悬赏交付物验收标准

验收取决于悬赏类型的不同交付物：

| 悬赏类型 | 验收要点 | 最小证据要求 |
|---------|---------|------------|
| **人脉引荐 (L1)** | 引荐邮件/消息截图+对方确认回复 | 引荐截图 + 对方"已收到"确认 |
| **会议安排 (L2)** | 会议确认邀请+会议纪要摘要 | 日历确认 + 30字以上会议内容总结 |
| **深度交流 (L2)** | 60min+会议+跟进计划 | 会议纪要 + 对方后续动作确认 |
| **战略对接 (L3)** | 正式提案机会 | 正式邀请函/日程确认邮件 |
| **信息/情报** | 独家信息+可验证 | 信息源 + 交叉验证（至少2方确认） |

**验收流程：**

```
伙伴提交 → AI初筛（格式+完整性）→ 人工复核（价值判断）→ 验收通过/补充/驳回
```

**验收时间线：**
- 提交后 24h 内：AI初筛完成
- 48h 内：人工复核完成（L1/L2）/ 72h内（L3）
- 通过后 7个工作日内：悬赏金发放到账

### 2. 尽职调查清单

#### 信息核实
- [ ] 交付物中的联系人/组织真实存在（LinkedIn/企查查/官网验证）
- [ ] 信息源可靠且非公开渠道可获取（不能是百度查到的东西）
- [ ] 引荐链路上无中间人虚构（引荐截图需含完整上下文）
- [ ] 会议纪要与对方公开信息不矛盾

#### 违规检查
- [ ] 无利益冲突（伙伴是否同属竞品/利益相关方？）
- [ ] 无贿赂/回扣嫌疑
- [ ] 无虚假抬价（伙伴是否"演双黄"虚构引荐需求？）
- [ ] 无隐私/NDA违规

### 3. KYC/KYB核查流程

#### 个人伙伴 (KYC)

| 检查项 | 要求 | 方式 |
|-------|------|------|
| **身份验证** | 真实姓名+身份证/护照+联系邮箱 | 首次悬赏前一次性提交 |
| **背景核实** | 行业经历+专业能力声明 | 自我声明+LinkedIn验证 |
| **信用记录** | 无恶意违约历史 | 内部信用分系统 |
| **关联关系** | 是否与本企业有关联 | 关联关系声明 |

**KYC触发时机：**
- L1：首次参与时完成基础KYC
- L2：晋升或单次悬赏≥¥2,000需补充背景资料
- L3：单次≥¥10,000需完成完整KYC+签署保密协议

#### 企业伙伴 (KYB)

| 检查项 | 要求 | 方式 |
|-------|------|------|
| **营业执照** | 合法注册 | 企查查/天眼查/对应区域工商查询 |
| **法人代表** | 与对接人一致或授权 | 授权书 |
| **行业资质** | 无负面新闻/行业黑名单 | 公开信息检索 |
| **税务合规** | 可开具合规发票 | 发票样本 + 税务登记确认 |

### 4. 争议解决框架

| 争议类型 | 典型场景 | 处理方式 | 时限 |
|---------|---------|---------|------|
| **交付争议** | 伙伴认为完成但验收方认为不足 | 比对验收标准→双方举证→调解 | 7个工作日内 |
| **效果争议** | 完成了任务但未达成预期商业效果 | 悬赏仅验收"行为"非"结果"→按标准结算 | — |
| **品质争议** | 引荐后对方未跟进或未产生价值 | 按引荐完成标准已达成→正常结算 | — |
| **违规争议** | 一方指控另一方违反规则 | 暂停发放→内部审查→升级至管理者裁决 | 14个工作日内 |
| **重复提交** | 多个伙伴提交同一目标 | 按首次提交时间优先 | 24h内判定 |

**争议处理流程：**

```
1. 受理：收到书面争议声明
2. 暂停：暂停与争议相关的悬赏金发放
3. 调查：双方举证 + 第三方证据（截图/邮件/记录）
4. 裁决：按验收标准+悬赏规则做出判决
5. 执行：支付/拒付/按比例支付
6. 上诉：不服裁决可在3个工作日内提出上诉至管理者
```

## 伙伴验证检查清单

- [ ] 交付物包含所有要求的证据类型
- [ ] 证据真实、完整、可交叉验证
- [ ] 伙伴已完成必要KYC
- [ ] 无利益冲突/违规嫌疑
- [ ] 悬赏类型与验收标准匹配
- [ ] 金额/结算方式已确认
- [ ] 争议条款双方知晓
