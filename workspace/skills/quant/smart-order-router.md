# 智能订单路由

订单执行智能路由系统，最小化市场冲击和交易成本。

## 订单类型与算法

### 隐藏/冰山订单

| 类型 | 原理 | 适用场景 |
|------|------|---------|
| **冰山订单(Iceberg)** | 仅暴露部分订单量，隐藏剩余 | 大额订单，避免市场冲击 |
| **隐藏订单** | 完全不在订单簿显示 | 信息敏感交易 |
| **储备订单(Reserve)** | 自动补充暴露量 | 持续执行的冰山单 |

### TWAP（时间加权平均价格）

将大额订单均匀分配到时间窗口内执行：

```
总订单量: Q
执行时间: T 分钟
切片数量: N = T × 60 / 间隔(秒)
每片大小: q = Q / N
```

| 参数 | 推荐值 |
|------|--------|
| 切片间隔 | 30-60秒 |
| 随机化范围 | ±20% |
| 价格偏离容忍 | 0.1%-0.3% |

### VWAP（成交量加权平均价格）

按历史成交量分布分配订单：

```
时段权重 = 历史该时段成交量 / 日总成交量
时段订单 = 总订单 × 时段权重
```

| 参数 | 推荐值 |
|------|--------|
| 历史窗口 | 20交易日 |
| 更新频率 | 每5分钟 |
| 偏差容忍 | 实际VWAP偏离 < 0.05% |

## 市场冲击模型

### Almgren-Chriss模型

```
总交易成本 = 固定成本 + 暂时冲击 + 永久冲击

暂时冲击: I_g(t) = γ × σ × X_α × (α/VT)^δ
永久冲击: I_p(t) = θ × σ × X_α × (α/VT)^δ

其中:
σ = 波动率
X_α = 订单大小
V = 日均成交量
T = 执行时间
α = 参与率
γ, θ, δ = 经验参数
```

### 参数配置

| 参数 | BTC | ETH | 小币种 |
|------|-----|-----|--------|
| γ | 0.1 | 0.15 | 0.5 |
| θ | 0.01 | 0.015 | 0.05 |
| δ | 0.6 | 0.6 | 0.7 |

## 执行质量分析

### Implementation Shortfall

```
IS = (执行价格 - 决策价格) × 交易量
   = 固定成本 + 延迟成本 + 价格影响 + 机会成本
```

### 分析指标

| 指标 | 计算 | 优秀 |
|------|------|------|
| IS/bps | 短视成本/交易量bps | < 5bps |
| 滑点率 | (成交价-中间价)/中间价 | < 0.05% |
| 参与率 | 交易量/市场成交量 | < 5% |
| VWAP偏离 | 成交VWAP/市场VWAP - 1 | ±0.02% |
