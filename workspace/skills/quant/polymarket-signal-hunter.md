# Polymarket信号猎手

在预测市场中捕捉错误定价和套利机会，结合实时新闻事件驱动策略。

## 预测市场机会发现

### 概率错配类型

| 类型 | 描述 | 信号强度 |
|------|------|---------|
| **基本面错配** | 预测概率与客观概率偏差 > 15% | ⭐⭐⭐ |
| **新闻事件驱动** | 重大消息后概率未及时更新 | ⭐⭐⭐ |
| **多市场价差** | 同一事件在不同市场概率不同 | ⭐⭐⭐⭐ |
| **流动性误定价** | 低流动性市场的极端价格 | ⭐⭐ |
| **过度反应** | 事件发生后概率过度修正 | ⭐⭐⭐ |

### 机会评分模型

```
Score = w₁ × |P_market - P_fundamental|
      + w₂ × (1 - 流动性系数)
      + w₃ × 时间衰减因子
      + w₄ × 事件确定性

where w₁ + w₂ + w₃ + w₄ = 1
```

| 参数 | 权重 | 说明 |
|------|------|------|
| 概率偏差 | 0.5 | 市场价vs基本面价偏差 |
| 流动性 | 0.2 | 深度不足 = 更多套利空间 |
| 时间衰减 | 0.15 | 临近到期 = 价格收敛 |
| 确定性 | 0.15 | 事件结果可验证性 |

## 实时监控扫描器设计

### 架构

```
数据源层: Polymarket API / News API / Twitter API
    ↓
信号层: 概率计算 → 偏差检测 → 评分排序
    ↓
通知层: Telegram推送 / Discord Webhook
```

### 扫描频率

| 数据源 | 频率 | 优先级 |
|--------|------|--------|
| Polymarket订单簿 | 30秒 | 高 |
| 主流新闻RSS | 1分钟 | 中 |
| Twitter/X API | 持续流 | 中 |
| 链上数据 | 5分钟 | 低 |

## Telegram推送机制

### 通知格式

```
🚨 Polymarket信号: [事件名称]

📊 当前概率: 35% | 基本面概率: 55%
📈 偏差: +20% ⭐⭐⭐
💰 流动性: $45,000
⏰ 到期: 2025-01-15
📎 链接: [Polymarket链接]

推荐操作: 买入"是" @ 35c
目标退出: 50c (+43%)
```

### 通知分级

- 🔴 **紧急**(偏差 > 30%): 立即推送+声音提醒
- 🟡 **关注**(偏差 15-30%): 推送静默通知
- 🔵 **信息**(偏差 < 15%): 汇总摘要推送

## 信号置信度评分
