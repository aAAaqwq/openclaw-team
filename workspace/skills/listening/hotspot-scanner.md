# 热点扫描 (Hotspot Scanner)

> 跨平台实时发现正在飙升的热点话题，评估其与品牌/产品的关联度，计算最佳响应时间窗口，确保**在热度达到峰值前完成内容部署**。

## 核心方法

### 1. 多平台热点发现

| 平台 | 信号类型 | 监测方法 | 信号强度 |
|------|---------|---------|---------|
| **Reddit** | 话题帖子+评论 | Subreddit关键词+发帖速度 | ⭐⭐⭐⭐⭐ |
| **Hacker News** | 技术/创业话题 | 首页+New+评论区 | ⭐⭐⭐⭐ |
| **Twitter/X** | 趋势话题+热门推文 | 趋势API+话题标签 | ⭐⭐⭐⭐ |
| **LinkedIn** | 行业讨论+影响者意见 | 关键词+话题标签+互动率 | ⭐⭐⭐ |
| **微信** | 公众号+视频号+搜一搜 | 搜一搜热点+公众号阅读量 | ⭐⭐⭐ |
| **抖音/微博** | 热搜榜单+话题 | 实时热搜API+话题飙升 | ⭐⭐⭐⭐ |
| **Perplexity/Google** | AI搜索趋势 | 相关内容主题飙升 | ⭐⭐⭐（GEO特有） |
| **Google Trends** | 搜索趋势 | 实时搜索+相关查询 | ⭐⭐⭐ |

### 2. 话题飙升检测算法

```
话题热度 = 基础热度 × 趋势速度 × 加速度因子

基础热度     — 该话题的绝对讨论量/搜索量
趋势速度     — 单位时间增量（1h/4h/24h）
加速度因子   — 速度是否在加快（二阶导数）

飙升标识：速度 > 基线均值 + 2σ（标准差）
          且加速度 > 0（仍在加速阶段）
```

**生命周期阶段检测：**

| 阶段 | 特征 | 信号 | 行动建议 |
|------|------|------|---------|
| **萌芽期** | 小范围讨论，加速上升 | 在Reddit/知乎小圈子内发现 | 观察+准备内容框架 |
| **爬升期** | 跨平台扩散，加速明显 | 多平台同时出现+搜索飙升 | 🎯 **最佳进入时机** |
| **峰值期** | 全网讨论，速度放缓 | 主流媒体+社交刷屏 | 确认品牌参与内容已上线 |
| **衰退期** | 讨论减少，加速度为负 | 热度下降+新话题替代 | 不要追，沉淀复盘数据 |

### 3. 热点与品牌/产品关联度评分

关联度评估确保不蹭无效热度：

```
关联度 = 话题相关度 × 品牌契合度 × 受众重叠度

话题相关度（1-10）：热点主题与品牌/产品核心领域的匹配程度
品牌契合度（1-10）：品牌能以自然、真实的方式参与该讨论
受众重叠度（1-10）：热点讨论人群与品牌目标受众的重叠比例
```

**决策矩阵：**

| 关联度 | 热度 | 行动 |
|-------|------|------|
| ≥ 7 | ≥ 7 | **P0 立即行动** — 内容团队4h内出稿 |
| ≥ 5 | ≥ 8 | P1 快速跟进 — 当日内出稿 |
| ≥ 3 | ≥ 8 | P2 低风险热点 — 出简短回应即可 |
| < 3 | 不限 | ❌ 不参与（蹭热度反效果） |

### 4. 响应时间窗口计算

```
内容创建时间 + 审核时间 + 分发时间 ≤ 热度爬升窗

示例：
- 内容创建：2h（AIGC辅助）
- 审核：1h
- 多平台分发：1h
- 总耗时：4h
- 热度爬升窗（萌芽→峰值）：6-24h
- 判定：✅ 可追

如果总耗时 > 爬升窗，判定为无法追：
→ 转向次要关联话题
→ 或放弃
```

## 热点扫描产出模板

```
HOTSPOT REPORT — [YYYY-MM-DD HH:MM]
────────────────────────────────────
1. 话题：#话题名
2. 来源平台：Twitter → 知乎 → Reddit（跨平台）
3. 热度指标：
   - 当前热度：8.5/10
   - 趋势速度：+45%/4h
   - 加速度：↑（仍在加速）
   - 生命周期阶段：爬升期
4. 关联度评分：7.5/10（话题相关度8+品牌契合7+受众重叠7）
5. 响应窗口：4h内需完成 → ✅ 可追
6. 行动建议：P0 — 创建[内容类型]
```
