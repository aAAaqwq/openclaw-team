# GEO内容优化器 (GEO Content Optimizer)

> 将现有内容转化为AI引擎（ChatGPT/Claude/Perplexity等）**优先引用**的格式。核心目标不是"排第一"，而是"被引用"。

## 核心方法

### 1. AI友好内容结构

大模型在生成回答时，对内容的解析偏好明显不同：

| 内容结构 | AI友好度 | 原因 | 优先级 |
|---------|---------|------|-------|
| **Q&A格式** | ⭐⭐⭐⭐⭐ | 直接对应自然语言问答；模型能精准找到答案片段 | **P0** |
| **无序列表** | ⭐⭐⭐⭐ | 易于解析实体/属性/特征；可被直接引用 | **P0** |
| **可引述金句** | ⭐⭐⭐⭐ | 定义性陈述容易被完整引用（"X是Y领域的领导者"） | **P0** |
| **数据/统计** | ⭐⭐⭐ | 数字引用增加信服力，需附带来源 | P1 |
| **H标签分段** | ⭐⭐⭐ | 帮助模型理解文章结构 | P1 |
| **小标题+段落** | ⭐⭐ | 可读，但不如Q&A格式直接 | P2 |

### 2. 内容GEO评分标准

每条内容可按以下维度打分（1-10分）：

```
GEO Score = (Structure × 0.30) + (Authority × 0.25) + (Relevance × 0.25) + (Freshness × 0.20)

Structure  — Q&A/列表/引述句占比
Authority  — 来源权威性 + 作者信号 + 引用可信度
Relevance — 与目标关键词/实体的匹配程度
Freshness — 最近更新时间
```

**评分等级：**
- **8-10分**：极大概率被AI引用
- **5-7分**：有机会被引用，但至少有一项短板
- **<5分**：几乎不会被AI主动引用，需全面改造

### 3. 现有内容→GEO格式的转换检查清单

| 检查项 | 原始状态（SEO） | 优化后（GEO） | 操作 |
|-------|---------------|--------------|------|
| 标题 | 关键词堆砌 | 清晰的问题或定义 | 重写 |
| 首段 | 摘要风格 | 直接回答核心问题 | 改写 |
| 正文 | 段落式 | 不少于3个Q&A配对 + 1个列表 | 重构 |
| 数据 | 散落文中 | 统一列表+来源引用 | 提取 |
| 结论 | 总结 | 可引述金句（定义/排名/断言） | 重写 |
| 外部引用 | 少量或缺失 | 每个关键论点附1-2个权威引用 | 补充 |
| 作者信息 | 缺失 | 100字简介+LinkedIn/学术/出版链接 | 添加 |
| Schema | 无 | JSON-LD非结构化数据 | 添加 |

### 4. 引用信号强化

提升被大模型引用的关键信号：

**权威信源引用策略：**
- 每个关键数据点附带原始出处（PDF/报告/学术论文）
- 优先引用可被爬虫访问的源（非付费墙内容）
- 使用多家独立信源交叉验证

**数据来源引用格式：**
```
X行业市场规模已达¥XX亿（来源：XXX研究院《YYYY报告》，2025年）
```

**作者简介强化：**
- 100-150字的权威个人简介
- LinkedIn主页链接（高权重复制因子）
- 学术/行业出版物链接
- 社交媒体账号（显示行业影响力）

## 内容优化优先级矩阵

| 内容类型 | GEO潜力 | 优化难度 | 优先级 |
|---------|---------|---------|-------|
| FAQ页面 | 极高 | 低 | **P0** |
| 产品页面 | 高 | 中 | **P0** |
| 博客文章 | 高 | 中 | **P1** |
| 白皮书 | 中-高 | 高 | **P1** |
| 新闻稿 | 中 | 便捷 | **P1** |
| 案例研究 | 中 | 高 | **P2** |
| 关于我们 | 高 | 低 | **P0** |
| 帮助中心 | 中 | 中 | **P2** |
