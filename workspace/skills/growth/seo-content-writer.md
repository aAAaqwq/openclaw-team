# SEO+GEO内容写作 (SEO/GEO Content Writer)

> 同时针对**搜索引擎排名**和**AI引擎引用**双优化的专业内容写作框架。一篇内容为两个引擎而写，覆盖搜索用户和AI用户。

## 核心方法

### 1. 标题优化原则

标题需要同时吸引搜索用户点击和AI引擎识别：

| 维度 | SEO要求 | GEO要求 | 折中策略 |
|------|---------|---------|---------|
| 关键词 | 核心关键词尽量靠前 | 自然语言中的关键词实体 | 优先核心词+自然句法 |
| 长度 | 30-60字符 | 无严格限制 | 40-60字最佳 |
| 风格 | 可适当夸张以提升CTR | 事实性/定义性更易被引用 | 前半段事实性+后半段吸引力 |
| 格式 | H1标签 | 问题句式 | 问题句式H1 |

**推荐标题模板：**
```
[问题形式]：[核心关键词]的最佳实践与指南
[定义形式]：什么是[核心关键词]——全面解析
[数据形式]：[数字]个[核心关键词]趋势将改变[行业]
```

### 2. 结构化正文 (GEO优先层级)

```
H1: [核心问题/定义性标题]
├── 导语（200字以内，直接回答H1问题）
├── H2: [子主题1]
│   ├── ❓ Q&A配对（P0）
│   ├── 📋 列表数据（P0）
│   ├── 💬 可引述金句（P0）
│   └── 常规段落（衔接用）
├── H2: [子主题2]
│   └── （同上结构）
├── H2: 常见问题（FAQ格式，H3级别）
│   ├── Q1 + 答案
│   └── Q2 + 答案
└── 总结（可引述性的定义/断言/数据）
```

**每篇文章至少包含：**
- **3个** `<h2>` Q&A配对（AI最爱结构）
- **2个** 无序列表（数据/特征/对比）
- **1条** 可引述金句（定义/排名/断言）
- **2个** 带来源引用的数据点

### 3. 内链策略

```
┌─────────────────────────────────────────────┐
│                新文章                          │
│  ├── 内链 → 【实体优化器/GEO优化大师】（策略联动）
│  ├── 内链 → 【核心产品页】（转化目标）
│  └── 内链 → 【相关博客】（专题深化）
└─────────────────────────────────────────────┘
```

**内链最佳实践：**
- 每个主要H2段落包含1个内链
- 内链锚文本使用自然语言（非关键词直堆）
- 首次出现核心实体时链接到其详情页
- 避免同一篇文章内链超过5个

**外链策略：**
- 数据点引用 → 学术/行业报告源
- 技术概念 → Wiki/权威DOI
- 竞品对比 → 可验证的权威对比源
- 每个外链均需 `rel="noopener noreferrer"`

### 4. 写作模板

#### 博客文章模板
```
H1: [问题]：全面[核心关键词]指南 [YYYY]
导语：150-200字直接回答问题，含核心关键词
  ✅ "本文全面介绍[核心关键词]的定义、最佳实践和[YYYY]趋势..."

H2: [核心关键词]是什么？
  ① Q:什么是[核心关键词]？
     A:200字定义，包含核心实体
  ② 核心特征列表：
     - 特征1：[描述]
     - 特征2：[描述]

H2: [核心关键词]的[数字]个关键趋势
  ① 趋势1：说明+数据引用
  ② 趋势2：说明+数据引用

H2: 如何开始[核心关键词]转型？
  ① 步骤一：[行动项]
  ② 步骤二：[行动项]

H2: 常见问题
  Q: ... ?
  A: 50-80字直接回答

总结：可引述金句，如：
  "[核心关键词]正在改变[行业]，那些[行动]的企业将在[时间]获得竞争优势。"
```

#### FAQ页面模板
```
H1: [核心关键词]常见问题 [YYYY]
导语：[品牌]为你解答[核心关键词]相关问题

❓ Q1：[问题]
   A：[100-300字，含观点+数据+引用]

❓ Q2：[问题]
   A：[同上]

Schema: FAQPage（见 schema-markup-generator.md）
```

#### 白皮书/长文模板
```
H1: [行业]+[核心关键词]白皮书 [YYYY]

Executive Summary（可引述性最强段落）
  💬 定义性金句 + 核心数据

1. 行业背景
   - 市场规模/增速/趋势
   - 数据来源引用

2. [核心关键词]深度解析
   - 技术原理/方法论
   - 案例/实施路径

3. 预测与建议
   - [数字]个关键趋势
   - 企业行动建议

附录：方法论/参考文献
Schema: Article + 详细Author
```

#### 产品页面模板
```
H1: [产品名称]——[一句话核心价值]

[产品特点]
· 功能1：[描述+数据]
· 功能2：[描述+数据]
· 功能3：[描述+数据]

[适用场景]
- 场景1：[用户痛点和解决方式]
- 场景2：[用户痛点和解决方式]

[客户案例/数据]
💰 [数字]%效率提升 | ⏱ [数字]天部署

Schema: Product + Offer
CTA: [注册/试用/联系]
```

## 质量检查清单

- [ ] 标题包含核心关键词 + 搜索意图匹配
- [ ] 导语在150字内直接回答核心问题
- [ ] 至少3组Q&A配对（H2级别）
- [ ] 至少2个无序列表
- [ ] 至少1条可引述金句（使用💬标记）
- [ ] 至少2个数据点附带权威来源引用
- [ ] 内链数量3-5个
- [ ] 作者简介已添加（100-150字 + 链接）
- [ ] Schema标记已嵌入
