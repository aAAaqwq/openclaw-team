# Growth Flywheel Architect

## When to Apply
Use when designing a new growth mechanism, diagnosing stalled growth, or creating a self-sustaining user acquisition engine.

## The Growth Flywheel Framework

### 1. Core Loop Design

Every growth flywheel needs 4 elements:

```
              ┌─────────────┐
              │ 用户获取     │
              │ (Acquire)   │
              └──────┬──────┘
                     │
              ┌──────▼──────┐
              │ 价值交付     │
              │ (Deliver)   │
              └──────┬──────┘
                     │
              ┌──────▼──────┐
              │ 网络效应     │
              │ (Network)   │
              └──────┬──────┘
                     │
              ┌──────▼──────┐
              │ 病毒传播     │
              │ (Viral)     │
              └──────┬──────┘
                     │
                     └──→ 回到顶部
```

### 2. Flywheel Design Checklist

```
□ 用户获得价值后才传播（先交付，再请求分享）
□ 传播机制是产品内置的，不是营销附赠的
□ 每个分享动作给用户带来实际价值（折扣/声望/效率）
□ 病毒系数(K-factor) > 1.0
□ 传播路径短（≤3个点击）
□ 新用户也能快速激活并获得价值（Time-to-Value < 5分钟）
```

### 3. Common Flywheel Types

| Type | How It Works | Example | K-factor |
|------|-------------|---------|----------|
| **Invite** | 用户邀请朋友→双方获奖励 | Dropbox | 1.2-1.5 |
| **Content** | 用户创造内容→吸引新用户 | TikTok | >2.0 |
| **Network** | 用户越多价值越大 | Slack/微信 | >1.5 |
| **Integration** | 用户用A工具→B工具自动推荐 | Shopify | 1.1 |
| **Litigation** | 用户分享→内容上传播→更多观看 | BuzzFeed | 1.5-2.0 |

### 4. Flywheel Audit

```
Step 1: 识别当前的增长瓶颈
  - 获客慢？→ 检查传播路径效率
  - 激活低？→ 检查Time-to-Value
  - 留存低？→ 检查核心价值交付
  - 传播弱？→ 检查病毒机制设计

Step 2: 量化每个环节
  - 获客到激活转化率 XX%
  - 激活到传播转化率 XX%
  - 传播回获客转化率 XX%
  - 当前K-factor: X.X

Step 3: 确定杠杆环节
  - 哪个环节提升1%带来的总增长最大？
  - 哪个环节的优化成本最低？
  - 优先优化：ROI最高的那个环节
```
