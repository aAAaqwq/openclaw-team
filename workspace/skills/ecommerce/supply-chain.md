# 供应链与库存预测引擎

基于AI的库存预测与供应链协同系统，最小化资金占用，最大化库存周转。

## 库存预测模型

### 基础预测公式

```
预测销量(t) = Base(t) × Seasonality(t) × Trend(t) × Promotion(t) + Event(t)

其中:
- Base = 历史销量移动平均
- Seasonality = 周/月/年季节性因子
- Trend = 长周期趋势方向
- Promotion = 促销活动增量系数
- Event = 突发事件修正(差评/舆情/竞品下架)
```

### 分层预测

| 层级 | 粒度 | 预测窗口 | 应用 |
|------|------|---------|------|
| **SKU级** | 单品 | 7-30天 | 补货决策 |
| **品类级** | 二级类目 | 30-90天 | 采购计划 |
| **店铺级** | 全店 | 90-365天 | 预算与备货策略 |

### 安全库存计算

```
安全库存 = Z × σ_d × √(L)

Z = 服务水平对应的z值
  (95% = 1.65, 97% = 1.88, 99% = 2.33)
σ_d = 日需求的标准差
L = 补货提前期(天)

再订货点 = 日均销量 × 提前期 + 安全库存
```

## 库存健康度仪表盘

| 指标 | 健康 | 警示 | 危险 |
|------|------|------|------|
| **库存周转率(ITO)** | >12次/年 | 6-12次 | <6次 |
| **缺货率** | <2% | 2-5% | >5% |
| **滞销占比(>90天)** | <5% | 5-15% | >15% |
| **库存天数(DOI)** | 30-45天 | 45-60天 | >60天/<15天 |

## 供应商管理

### 供应商评估矩阵

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **价格竞争力** | 25% | 同行比对 |
| **质量合格率** | 25% | 退货率 < 2% 满分 |
| **交货准时率** | 20% | > 95% 满分 |
| **账期友好度** | 15% | 越长越好 |
| **配合度** | 15% | 打样速度/定制能力 |

### 补货自动化流程

```
库存检查（每日/实时）
    ↓
安全库存阈值触发
    ↓
检查在途库存+在产库存
    ↓
计算补货量 = 预测消费 - 现有库存 - 在途 + 安全库存
    ↓
生成采购建议单
    ↓
人工确认 → 推送至ERP
    ↓
供应商接单 → 生产/出库
    ↓
入库质检 → 上架销售
```
