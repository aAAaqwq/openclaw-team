# 定价策略设计器 (Pricing Strategy Designer)

> 系统化的SaaS/数字产品定价方法论。涵盖定价模型选择、实验设计、竞品对标和弹性测试，使命是**找到价格与销量的最优平衡点**。

## 核心方法

### 1. SaaS定价模型

| 模型 | 机制 | 适合类型 | 优势 | 风险 |
|------|------|---------|------|------|
| **分层定价** (Tiered) | 2-4个固定套餐 | 功能有明显差异的产品 | 简单、用户易理解 | 用户可能选择过低/过高的套餐 |
| **按用量定价** (Usage-based) | 按API调用/存储/用户数 | 基础设施/PaaS | 低门槛起步、随增长付费 | 收费不可预测 |
| **价值定价** (Value-based) | 按给客户创造的价值 | 高ROI工具/咨询 | 可收高价、利润率高 | 需要强价值证明 |
| **混合定价** (Hybrid) | 基础月费+超额用量 | 快速增长的SaaS | 兼顾稳定性+增长 | 验证复杂度高 |
| **免费试用** (Freemium) | 免费版+付费升级 | 高网络效应产品 | 用户获取快 | 付费转化率低 |
| **阶梯定价** (Per User/Seat) | 按用户数 | 协作型工具 | 随客户规模线性增长 | 大客户可能觉得贵 |

### 2. 定价实验设计

#### Van Westendorp 价格敏感度测试

```
太便宜（Too Cheap）→ 便宜（Cheap）→ 合理（Good Value）→ 贵（Expensive）→ 太贵（Too Expensive）

推荐定价区间 = 合理下限 ~ 合理上限
最佳定价点 = 合理(Good Value)区间分布最密集区域
切换点（Indifference Price Point）= 便宜与贵的交叉点
```

**执行方式：**
```
向 100-200 名目标用户提问：
① 低于多少钱你会觉得"太便宜质量不可信"？
② 低于多少钱你会觉得"便宜但可以接受"？
③ 高于多少钱你会觉得"贵但仍在考虑范围"？
④ 高于多少钱你会觉得"太贵了完全不考虑"？

然后绘制四条曲线，找出：
- IDP (Indifference Price Point)：便宜与贵的交叉
- PMC (Point of Marginal Cheapness)：便宜与合理交叉
- PME (Point of Marginal Expensiveness)：合理与贵的交叉
- OPP (Optimal Price Point)：太便宜与太贵的交叉
```

#### Conjoint Analysis（联合分析）

评估用户对各属性的权衡取舍：

```
属性1：价格（¥99 / ¥199 / ¥399）
属性2：功能数量（基础 / 专业 / 企业）
属性3：存储空间（50G / 200G / 无限）
属性4：支持（邮件 / 聊天 / 专属）

→ 每个用户选择多个组合
→ 用逻辑回归计算各属性的效用值
→ 找出最优组合+价格弹性
```

### 3. 竞品定价对标框架

| 维度 | 评估标准 | 信息源 |
|------|---------|-------|
| **价格区间** | 竞争对手各套餐的价格范围 | 官网/G2/Capterra |
| **功能差异** | 同价位下功能对标的性价比 | 功能对比矩阵 |
| **定位差异** | 高端/中端/低端市场定位 | 品牌/营销素材 |
| **收费方式** | 月付vs年付vs永久许可证 | 定价页面+公开信息 |
| **隐藏成本** | 超用费/集成费/启动费 | 用户评论+论坛 |
| **免费层策略** | 免费版的功能限制 | 免费版注册体验 |

**对标策略选择：**

| 定位 | 定价策略 | 适用情况 |
|------|---------|---------|
| **领先者溢价** | 比竞品高20-50% | 独特功能+品牌强势 |
| **市场均价** | 与主流竞品持平 | 市场已经成熟 |
| **渗透定价** | 比竞品低20-40% | 快速抢占份额 |
| **免费增值** | 免费+付费差异化 | 高网络效应 |

### 4. 价格弹性测试方法

测定用户对价格变化的敏感度：

```
Price Elasticity = %ΔQuantity / %ΔPrice

|弹性| > 1：弹性需求 → 价格敏感
|弹性| < 1：非弹性需求 → 对价格不敏感
```

**测试方法：**

| 方法 | 操作 | 优点 | 风险 |
|------|------|------|------|
| **A/B定价测试** | 同一页面随机显示不同价格 | 直接测试转化差异 | 同一用户前后看到不同价格混淆 |
| **地理定价** | 不同地区显示不同价格 | 自然分割测试 | 区域差异干扰 |
| **时间序列** | 某时段临时调整价格 | 简单快速 | 节假日/季节性影响 |
| **Survey/问卷** | 假设场景价格敏感度 | 低成本 | 行为vs声明差距 |

**测试建议：**
- 首次调价：变动不超过±20%
- 观察周期：至少2个完整计费周期
- 护栏指标：保留率（防止降价吸引→留存差）
- 分用户群：已有用户 vs 新用户分别测试

```
  免费→付费转化率：[X]%

```
