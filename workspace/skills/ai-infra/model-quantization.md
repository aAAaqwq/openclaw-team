# 模型量化

大模型量化压缩技术选型与评估指南。

## 量化方法对比

| 方法 | 精度损失 | 推理加速 | 适用场景 | 格式 |
|------|----------|----------|----------|------|
| GPTQ | 低 | ~2x | GPU推理 | 4bit/8bit |
| AWQ | 最低 | ~2x | GPU推理 | 4bit/8bit |
| GGUF | 中 | ~1.5x | CPU/边缘 | 2-8bit |
| QLoRA | 中 | — | 训练/微调 | NF4 |
| BitsAndBytes | 中 | 无硬加速 | 快速试用 | 4bit/8bit |
| SmoothQuant | 低 | ~2x | INT8推理 | INT8 |
| AQLM | 低 | ~3x | 极致压缩 | 2bit/3bit |

### 推荐选型
```
GPU生产部署：
- 首选 AWQ（质量最好，工具链成熟）
- 次选 GPTQ（生态最广）

CPU/边缘：
- 首选 GGUF（Ollama/llama.cpp原生）

微调：
- QLoRA + NF4（显存需求低）

极致压缩：
- AQLM（牺牲少量质量，换取3x压缩）
```

## 量化精度详解

### W4A16（权重4bit，激活16bit）
- **原理**：权重INT4量化，激活保持FP16
- **效果**：显存减约4x，推理速度提升~2x
- **质量损失**：< 1%（AWQ），~2%（GPTQ）
- **适用**：通用GPU部署

### INT8量化
- **SmoothQuant**：迁移难度到激活，保持精度
- **效果**：显存减半，吞吐量~2x
- **质量损失**：< 0.5%（大模型损失更小）
- **适用**：对精度敏感的正式环境

### FP8（H100原生）
- **原理**：FP8前向计算，FP16存储
- **效果**：FP8 Tensor Core加速
- **适用**：H100/B200部署
- **注意**：需框架支持（TensorRT-LLM/vLLM）

### 低位量化（3bit/2bit）
| 位宽 | 方法 | 压缩比 | 质量损失 |
|------|------|--------|----------|
| 3bit | AQLM | ~8x | 中 |
| 2bit | AQLM/QuIP# | ~12x | 高 |
| 混合（2:4） | SpQR | ~9x | 中 |

> **注意**：2bit以下仅适用于资源极度受限场景。

## 校准数据集构建

### 通用校准数据
- **WikiText-2**：LLM校准标准集
- **C4（Colossal Clean Crawled Corpus）**：多样化文本
- **Pile**：学术论文+代码+书籍

### 领域校准数据
```
构建原则：校准数据应匹配部署场景
- 代码场景 → The Stack / CodeSearchNet
- 中文场景 → MMLU中文 / C-Eval / C-MTEB
- 金融场景 → FinQA / 金融公告数据集
```

### 数据量建议
```
GPTQ: 128-256个样本（每个512 token）
AWQ: 128个样本
量化效果随数据量增加饱和，无需过多。
```

## 量化后评估

### 质量损失评估
| 指标 | 说明 | 测量方法 |
|------|------|----------|
| Perplexity | PPL变化率 | Wiki/PTB数据集 |
| MMLU | 知识推理 | 多任务准确率对比 |
| HumanEval | 代码能力 | pass@1对比 |
| 部署场景 | 业务指标 | 端到端对比测试 |

### 性能提升评估
| 指标 | 说明 |
|------|------|
| Token/s（吞吐） | 每秒钟生成Token数 |
| TTFT（首Token时延） | 用户等待时间 |
| ITL（Token间延迟） | 流式体验 |
| GPU显存占用 | 可承载更大Batch |

### 量化测试流程
```
1. 基线测试：FP16完整模型（PPL/MMLU/推理时延）
2. 量化后测试：相同指标对比
3. 业务场景测试：端到端对话质量人工评估
4. 性能压测：最大Batch/吞吐
5. 验收标准：PPL上升 < 1%，业务质量人工评估无差异
```
